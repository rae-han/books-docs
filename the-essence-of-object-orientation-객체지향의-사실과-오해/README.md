# 객체지향의 사실과 오해 (역할, 책임, 협력 관점에서 본 객체지향)

> 『객체지향의 사실과 오해 — 역할, 책임, 협력 관점에서 본 객체지향』 (조영호 지음, 위키북스, 2015)

객체지향이란 무엇인가라는 원론적인 질문에 대한 저자 나름의 대답. 클래스나 상속이 아닌 **역할·책임·협력**이라는 관점에서 객체지향의 본질을 탐구한다.

## 개요

커피 공화국, 이상한 나라의 앨리스, 런던 지하철 노선도 같은 비유와 사고실험으로 객체지향의 핵심 개념을 설명한다. 객체지향을 다루는 다른 책이나 자료를 읽을 때 미리 알고 있으면 도움이 되는 기본 배경과 관점을 제공하는 책이다.

후속작 《[오브젝트](../objects/README.md)》의 자매편이다 — 이 책이 객체지향의 첫 두 걸음(객체를 바라보는 것, 협력 공동체로 바라보는 것)을 다루고, 오브젝트가 세 번째·네 번째 걸음(역할과 책임 부여, 코드로 표현하는 기술)을 잇는다. 두 책은 독립적으로 읽을 수 있다.

## 핵심 개념 맵

- **객체지향 = 협력하는 객체들의 공동체**: 클래스는 구현 도구일 뿐, 본질은 협력·역할·책임 (Ch1·4)
- **행동이 상태를 결정한다**: 상태 중심이 아니라 행동(책임) 중심으로 객체를 정의해야 협력에 적합해진다 (Ch2)
- **타입 = 행동에 의한 분류**: 같은 메시지에 응답하면 같은 타입 — 구현이 아니라 행동 기준 (Ch3)
- **메시지가 객체를 결정한다**: 협력에 필요한 메시지를 먼저 찾고, 그 메시지를 처리할 객체를 나중에 (Ch5)
- **구조는 안정, 기능은 불안정**: 도메인 모델(구조) 위에 유스케이스(기능)를 배치해야 변경에 견딘다 (Ch6)
- **은유(메타포)**: 현실 객체의 이름을 빌리되 능동적 존재로 재창조 — 의인화 (Ch2·7)
