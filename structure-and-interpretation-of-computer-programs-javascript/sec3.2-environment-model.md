# Section 3.2: The Environment Model of Evaluation (평가의 환경 모델)

## 핵심 질문

대입을 도입하면서 치환 모델이 무력화되었다. 변수 바인딩과 대입을 정확하게 설명하는 새로운 평가 모델은 어떤 구조를 가지는가?

---

## 핵심 개념

| 개념 | 정의 |
|------|------|
| **프레임(Frame)** | 이름-값 바인딩의 집합을 담는 테이블 |
| **환경(Environment)** | 프레임들의 체인 — 각 프레임이 상위 환경(enclosing environment)을 가리킴 |
| **전역 환경(Global environment)** | 최상위 프레임. 내장 함수와 전역 변수를 담음 |

---

## 1. 환경의 구조

### 1.1 프레임과 환경

**환경**은 프레임들의 연결 리스트다:

```
┌──────────────────┐
│  Frame II        │
│  x: 3            │──→ ┌──────────────────┐
│  y: 5            │    │  Frame I (Global) │
└──────────────────┘    │  square: ...      │
                        │  sum: ...         │
                        └──────────────────┘
```

변수를 평가할 때, **현재 프레임부터 시작하여 상위 프레임으로** 체인을 따라 올라가며 바인딩을 찾는다. 이것이 **렉시컬 스코핑**의 메커니즘이다.

### 1.2 함수 객체

함수는 두 부분으로 구성된다:
1. **코드**: 매개변수와 본문
2. **환경**: 함수가 **정의된** 환경을 가리키는 포인터

```javascript
const square = x => x * x;
// square는 (코드: x => x * x, 환경: 전역 환경)
```

---

## 2. 평가 규칙

### 2.1 함수 적용

`square(5)`를 평가할 때:

1. **새 프레임**을 생성한다. 매개변수 `x`를 인수 `5`에 바인딩한다.
2. 새 프레임의 **상위 환경**은 함수 객체가 가리키는 환경(전역)이다.
3. 새 환경에서 함수 본문 `x * x`를 평가한다.

```
┌───────────┐
│  E1        │
│  x: 5     │──→ ┌─────────────┐
└───────────┘    │  Global     │
                  │  square: ●──┼──→ (x => x * x, Global)
                  └─────────────┘
```

### 2.2 대입의 메커니즘

`x = new_value`는 환경에서 `x`의 바인딩을 찾아 값을 **변경**한다. 새 바인딩을 만드는 것이 아니라 기존 바인딩을 수정한다.

### 2.3 클로저의 메커니즘

```javascript
function make_withdraw(balance) {
    return amount => {
               if (balance >= amount) {
                   balance = balance - amount;
                   return balance;
               } else {
                   return "insufficient funds";
               }
           };
}

const W1 = make_withdraw(100);
```

1. `make_withdraw(100)` 호출 → 프레임 E1 생성: `balance: 100`
2. 반환되는 화살표 함수는 (코드, E1)로 구성 — E1을 기억한다
3. `W1(50)` 호출 → 프레임 E2 생성: `amount: 50`, 상위는 E1
4. E2에서 `balance`를 찾으면 E1의 `balance: 100` 발견 → `balance = 50`으로 변경

```
┌────────────────┐
│  E2             │
│  amount: 50    │──→ ┌───────────────┐
└────────────────┘    │  E1            │
                       │  balance: 50  │──→ Global
                       └───────────────┘
```

`W2 = make_withdraw(100)`은 **별도의 E3** 프레임을 생성하므로, `W1`과 `W2`는 독립적인 `balance`를 가진다.

> **핵심 통찰**: 환경 모델은 클로저, 대입, 렉시컬 스코핑을 **하나의 통합된 메커니즘**으로 설명한다. 함수가 정의 시점의 환경을 기억하고(클로저), 그 환경의 바인딩을 변경할 수 있다(대입). 이것이 상태를 가진 객체의 근본 메커니즘이다.

## 계산 모델

### 환경 다이어그램 읽는 법

환경 다이어그램은 SICP 학습의 핵심 도구다:

1. **함수 객체**: 원으로 표시, 코드를 가리키는 화살표 + 환경을 가리키는 화살표
2. **프레임**: 테이블로 표시, 이름-값 바인딩 나열
3. **환경 포인터**: 프레임에서 상위 프레임으로의 화살표
4. **함수 적용**: 새 프레임 생성 → 매개변수 바인딩 → 상위 환경 연결

---

## 요약

- **환경**은 프레임의 체인이며, 변수 탐색은 현재 프레임에서 시작하여 상위로 올라간다.
- **함수 객체**는 코드 + 정의 시점의 환경으로 구성된다 (클로저).
- **함수 적용**은 새 프레임을 생성하고, 상위 환경을 함수 객체의 환경으로 설정한다.
- **대입**은 환경에서 바인딩을 찾아 값을 변경한다.
- 환경 모델은 치환 모델의 한계를 극복하여 대입, 클로저, 렉시컬 스코핑을 통합적으로 설명한다.
