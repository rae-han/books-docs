# Section 4.4: Logic Programming (논리 프로그래밍)

## 핵심 질문

프로그래밍이 **"사실과 규칙을 선언"** 하는 것만으로 이루어질 수 있는가? "어떻게"를 기술하지 않고 "무엇"만 기술하는 프로그래밍은 가능한가?

---

## 핵심 개념

| 개념 | 정의 |
|------|------|
| **질의 언어(Query language)** | 사실과 규칙의 데이터베이스에 질의하는 선언적 언어 |
| **통일(Unification)** | 두 패턴을 일치시키는 변수 바인딩을 찾는 알고리즘 |
| **패턴 매칭(Pattern matching)** | 데이터가 특정 패턴에 일치하는지 검사하는 과정 |
| **규칙(Rule)** | 사실로부터 새로운 사실을 도출하는 논리적 추론 규칙 |

---

## 1. 선언적 프로그래밍

### 1.1 함수적 프로그래밍 vs 논리 프로그래밍

함수적 프로그래밍에서 `append`는 **방향**이 있다:

```javascript
append(list(1, 2), list(3, 4));  // list(1, 2, 3, 4)
```

논리 프로그래밍에서 `append`는 **관계**다:

```
append_to_form((1, 2), (3, 4), ?z)    → z = (1, 2, 3, 4)  // 합치기
append_to_form(?x, (3, 4), (1, 2, 3, 4))  → x = (1, 2)    // 역으로!
append_to_form((1, 2), ?y, (1, 2, 3, 4))  → y = (3, 4)    // 역으로!
append_to_form(?x, ?y, (1, 2, 3))     → 여러 답              // 분해!
```

같은 "규칙"으로 다양한 방향의 질의에 답할 수 있다.

### 1.2 질의 시스템

사실(assertions)의 데이터베이스:

```
(job (Bitdiddle Ben) (computer wizard))
(salary (Bitdiddle Ben) 60000)
(supervisor (Hacker Alyssa P) (Bitdiddle Ben))
```

질의(queries):

```
(job ?x (computer wizard))          // 컴퓨터 마법사인 사람?
(supervisor ?x (Bitdiddle Ben))     // Ben의 부하 직원?
```

규칙(rules):

```
(rule (lives_near ?person_1 ?person_2)
      (and (address ?person_1 (?town . ?rest_1))
           (address ?person_2 (?town . ?rest_2))
           (not (same ?person_1 ?person_2))))
```

> **핵심 통찰**: 논리 프로그래밍은 Section 1.1에서 시작한 "선언적 지식 vs 명령적 지식" 대비의 정점이다. 수학에서 sqrt(x)는 "y² = x인 y"로 **선언적**으로 정의된다. 논리 프로그래밍은 이런 선언적 정의만으로 계산을 수행하는 패러다임이다. 그러나 효율적인 실행을 위해서는 평가 순서(탐색 전략)에 대한 이해가 여전히 필요하다.

---

## 2. 통일 (Unification)

### 2.1 패턴 매칭의 일반화

패턴 매칭은 한쪽이 상수이고 다른 쪽이 패턴인 경우:

```
(?x 2 3)  매칭  (1 2 3)  →  x = 1
```

**통일**은 양쪽 모두 변수를 포함할 수 있는 일반화:

```
(?x ?y)  통일  (?y 3)  →  x = 3, y = 3
```

통일 알고리즘은 질의 시스템과 규칙 기반 추론의 핵심 엔진이다.

---

## 3. 질의 시스템의 구현

질의 시스템은 **또 하나의 평가기**다. Section 4.1의 메타순환 평가기와 유사한 구조:

- `evaluate` → `qeval` (질의 평가)
- 표현식 유형에 따른 분기 (and, or, not, rule 적용)
- 환경 → **프레임 스트림**(가능한 변수 바인딩의 스트림)

```
qeval(질의, 프레임_스트림) → 새로운_프레임_스트림
```

---

## 요약

- **논리 프로그래밍**은 사실과 규칙을 선언하고, 시스템이 자동으로 추론하는 패러다임이다.
- **통일(unification)** 은 패턴 매칭의 일반화로, 양쪽에 변수가 있어도 매칭이 가능하다.
- 질의 시스템은 메타순환 평가기와 유사한 구조를 가지며, **프레임 스트림**을 통해 가능한 답을 표현한다.
- 논리 프로그래밍은 "무엇"과 "어떻게"의 경계를 탐구하지만, 효율성을 위해 절차적 고려가 여전히 필요하다.
