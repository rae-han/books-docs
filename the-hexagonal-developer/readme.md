# 육각형 개발자 (시니어 개발자로 성장하기 위한 10가지 핵심 역량)

> 『육각형 개발자 — 시니어 개발자로 성장하기 위한 10가지 핵심 역량』 (최범균 지음, 한빛미디어, 2023)

스포츠계에서 다양한 능력치가 고루 균등한 선수를 '육각형 선수'라고 부른다. 이 책의 '육각형 개발자'는 **여러 역량을 고루 갖춘 개발자**다. 구현 기술뿐 아니라 설계·품질 관리·업무 관리·공유·리더십과 팔로워십까지 **10가지 핵심 역량**을 다룬다.

## 개요

건강한 몸에 여러 영양소가 필요하듯, 좋은 개발자가 되려면 구현 기술 외에도 다양한 역량을 키워야 한다. 모든 역량을 다 잘하기는 어렵지만 골고루 발전시킬 수는 있다 — 구현 기술만 편식하지 말고 개발에 필요한 다양한 역량을 섭취해서 좋은 개발자로 성장하자는 것이 이 책의 메시지다.

기술 역량(구현·코드 이해·응집도와 결합도·리팩터링·테스트·아키텍처)과 비기술 역량(업무 관리·공유·리더십)을 한 권에서 균형 있게 다루며, 한국 개발 조직의 현실(레거시·일정 압박·주니어 온보딩)을 반영한 저자 경험담이 풍부하다.

## 학습 가이드

1. **Ch1로 전체 지도를 잡고 순서대로** — 챕터가 기술 역량(Ch2~8) → 비기술 역량(Ch9~11) 순으로 확장된다
2. **코드 품질 트랙**: Ch4(코드 이해) → Ch5(응집도·결합도) → Ch6(리팩터링) → Ch7(테스트)
3. **주니어 탈출 트랙**: Ch2(학습법) → Ch9(업무 관리) → Ch10(공유)
4. **리더 준비 트랙**: Ch3(가치와 비용 — 비즈니스 관점) → Ch8(아키텍처 결정) → Ch11(리더와 팔로워)

## 핵심 역량 맵

```
        구현 기술 (Ch 2)
           │
    ┌──────┼──────┐
    │      │      │
코드 이해  응집도·결합도  아키텍처·패턴
 (Ch 4)    (Ch 5)       (Ch 8)
    │      │      │
    └──────┼──────┘
           │
   리팩터링 ── 테스트
    (Ch 6)    (Ch 7)
           │
    ┌──────┼──────┐
    │      │      │
업무 관리  공유    리더·팔로워
 (Ch 9)  (Ch 10)  (Ch 11)
```

- **구현 기술은 출발점일 뿐** — 코드 이해·응집도/결합도·아키텍처가 설계 역량의 삼각대 (Ch2·4·5·8)
- **리팩터링과 테스트는 한 쌍** — 테스트가 있어야 수정 공포 없이 개선할 수 있다 (Ch6·7)
- **가치와 비용이 판단 기준** — 기술 선택·아키텍처 결정 모두 유지보수 비용 절감으로 수렴 (Ch3·8)
- **비기술 역량이 시니어를 가른다** — 업무 나누기·공유·리더십/팔로워십 (Ch9~11)
- **역량 향상은 스스로 책임진다** — 맺음말의 결론
