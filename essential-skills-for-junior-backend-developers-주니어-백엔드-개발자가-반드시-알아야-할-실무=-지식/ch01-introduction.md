# Chapter 1: Introduction (코딩을 할 수 있게 된 것일 뿐)

## 핵심 질문

프레임워크로 API 서버를 만들 수 있게 된 것만으로 서버 개발자가 된 것일까? 실제 서비스를 운영하려면 코딩 외에 어떤 지식이 필요한가?

---

## 1. 신입 개발자 A의 이야기

신입으로 들어온 개발자 A가 있다. 개발자 A는 내부 직원이 사용할 간단한 사이트를 만들고 있었다. 기능 구현은 순조로웠다. 입사 전에 포트폴리오를 만들면서 필요한 구현 기술을 경험했기에 새로운 기술을 익혀야 하는 어려움도 없었다. 개발 PC에 톰캣 서버와 MySQL을 설치하고 제대로 동작하는지 확인도 했다. 기대한 대로 동작한다. 개발 서버에 배포한 뒤 기획팀에 테스트를 요청했다.

### 1.1 첫 번째 문제: 커넥션 누수

얼마 안 가 오류가 발생한다는 연락을 받았다. 개발 주소에 접근하니 DB에 연결할 수 없다는 오류가 발생한다. 개발 DB 자체는 연결이 잘 된다. 일단 개발 서버를 재시작했다. 이제 잘 된다. 기획팀에 다시 해보라고 연락했다.

하지만 제대로 실행되는 시간은 오래가지 않았다. 선배 개발자에게 도움을 요청하자, 선배는 코드를 훑어보더니 "커넥션을 안 닫았네요"라고 말한다. 일부 코드에서 사용이 끝난 DB 커넥션(*Connection - 애플리케이션과 데이터베이스 사이의 연결*)을 닫지 않은 것이다.

```java
// 개발자 A가 작성한 문제의 코드
Connection conn = ds.getConnection(); // 커넥션 풀에서 구함
try {
    // ... DB 작업 수행
} catch (Exception e) {
    // ... 에러 처리
}
// conn.close()가 누락 → 커넥션 풀에 반환하지 않음
```

DB 커넥션을 사용한 뒤 커넥션 풀(*Connection Pool - 미리 생성해둔 DB 커넥션을 재사용하는 기법*)에 반환하지 않았고, 그로 인해 커넥션이 누수되었다. 결국 풀에 있던 모든 커넥션이 소진되었고, 이때부터 풀에서 새로운 커넥션을 가져올 수 없는 오류가 발생했다.

### 1.2 두 번째 문제: 커넥션 최대 유지 시간 미설정

개발자 A는 커넥션 누수가 발생하는 버그를 모두 찾아 수정하고 개발 서버에 배포했다. 시간이 좀 지났는데도 오류가 발생한다는 연락이 오지 않았다. 개발자 A는 안도하며 자신감을 느낀 채 퇴근했다.

그런데 다음 날 문제가 다시 발생했다. DB에 또다시 연결할 수 없었다. 커넥션을 풀에 반환하지 않는 코드가 남아 있는지 다시 살펴봤지만 그런 코드는 없었다. 다시 선배 개발자에게 도움을 요청하자, 선배는 DB 오류 메시지와 커넥션 풀 설정을 보더니 "커넥션의 최대 연결 시간 설정이 없네요"라고 말한다.

DBMS(*Database Management System - 데이터베이스 관리 시스템*) 자체적으로 커넥션 유지 시간이 설정되어 있는데, 풀에 있는 커넥션의 연결 시간이 DBMS의 설정 시간을 초과하면서 발생한 문제였다. 커넥션 풀의 커넥션 최대 유지 시간을 알맞게 설정한 후 다시 개발 서버에 배포하니, 이제 시간이 지나도 DB 연결이 안 되는 문제가 발생하지 않았다.

> **실무 사례**: 개발자 A의 이야기는 가상의 사례지만 실제로 흔히 발생하는 문제 중 하나다. 스프링과 같은 프레임워크를 사용하면서 커넥션을 반환하지 않는 문제는 줄었지만, 커넥션 풀을 잘못 설정해서 DB 연결이 제대로 되지 않는 문제는 지금도 초보자들이 자주 겪는다.

## 2. 코딩만으로는 부족하다

초보 백엔드 개발자가 몰라서 저지르는 실수는 커넥션 풀뿐만이 아니다:

- **보안에 취약한 코드**를 작성하기도 한다
- 처음에는 잘 동작하지만 **시간이 지나면서 급격히 느려지는 시스템**을 만들기도 한다

스프링부트, Go, Node.js를 사용해 API 서버를 만드는 방법을 익혔다 하더라도, 이것은 서버 개발자가 알아야 할 지식의 일부에 지나지 않는다. 서버 개발을 위해 알아야 할 다양한 기초 지식을 모르면 실제 서비스 운영 과정에서 여러 문제가 발생할 수 있다.

> **핵심 통찰**: 프레임워크를 사용해 API 서버를 만드는 것은 서버 개발자가 알아야 할 지식의 일부에 불과하다. 기초 지식이 부족하면 보안 취약점, 성능 저하, 커넥션 관리 실패 등 운영 환경에서 다양한 문제가 발생한다.

## 3. 이 책의 목적

> 필자 역시 기초 지식이 부족해서 많은 실수를 저질렀다. 그러나 경험이 쌓이고 부족했던 지식을 학습하면서 실수를 줄일 수 있었다. 이러한 경험을 바탕으로 이 책을 썼다.

이 책은 저자가 직접 경험하고 학습한 내용 중에서 초보 백엔드 개발자에게 도움이 될 만한 기초적인 지식을 정리한 것이다. 하나의 주제를 깊게 다루기보다는 성능, 외부 연동, 보안 등 초보 백엔드 개발자가 반드시 알아야 할 여러 주제의 기초적인 내용을 다룬다.

## 요약

- 프레임워크로 API 서버를 만들 수 있다는 것은 서버 개발자가 알아야 할 지식의 **일부에 불과**하다
- 커넥션 풀 관리(반환 누락, 최대 유지 시간 미설정)는 초보 개발자가 자주 겪는 대표적인 실수다
- 기초 지식이 부족하면 보안 취약점, 성능 저하 등 운영 환경에서 다양한 문제가 발생한다
- 이 책은 성능, 외부 연동, 보안 등 초보 백엔드 개발자가 반드시 알아야 할 여러 주제의 기초를 다룬다
