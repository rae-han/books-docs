# 주니어 백엔드 개발자가 반드시 알아야 할 실무 지식

> 『주니어 백엔드 개발자가 반드시 알아야 할 실무 지식 — 시행착오를 줄여주는 실무 밀착 백엔드 개발 가이드』 (최범균 지음, 한빛미디어, 2025)

백엔드 개발자로 경력을 쌓다 보면 코드 외에도 다양한 기초 지식이 필요하다는 사실을 알게 된다. 성능부터 네트워크 구성까지, 초보 서버 개발자가 알아두면 좋을 **10가지 주제의 기초**를 다루는 책.

## 책 정보

| 항목 | 내용 |
|------|------|
| **저자** | 최범균 (『육각형 개발자』, 『DDD Start!』 저자) |
| **출판** | 한빛미디어, 2025 |
| **형식** | 주제별 독립 챕터 — 하나를 깊게보다 10가지를 넓게 |
| **대상 독자** | 백엔드 개발을 시작한 지 몇 년 안 된 주니어 개발자 |

## 개요

"느려진 서비스, 어디부터 봐야 할까"처럼 챕터 제목이 실무에서 실제로 부딪히는 질문이다. 성능·DB·외부 연동·비동기·동시성·IO·보안·서버·네트워크·설계 패턴의 10개 주제를 실무 사례와 함께 기초 수준에서 훑는다. 같은 저자의 [육각형 개발자](../the-hexagonal-developer/README.md)가 역량·태도를 다룬다면, 이 책은 백엔드 기술 지식의 지도를 그려 준다.

## 핵심 개념 맵

- **성능 문제는 좁혀 가며 찾는다**: 응답 시간 → 구간 분해 → DB/IO/외부 연동 순의 탐색 (Ch2·3·7)
- **외부는 반드시 실패한다**: 타임아웃·재시도·격리는 옵션이 아니라 기본값 (Ch4~5)
- **동시성은 터지기 전에**: 원자성 깨짐은 재현이 어렵다 — 잠금 전략을 먼저 설계 (Ch6, 부록 C)
- **보안·서버·네트워크는 장애 대응의 어휘**: 몰라도 개발은 되지만 장애 때 답답해진다 (Ch8~10)
- **기초를 넓게, 필요할 때 깊게**: 이 책은 지도 — 각 주제의 심화는 전문 서적으로
