## 2. 인터넷 (Internet) — 컴퓨터의 네트워크

### 2.1 인터넷이란?

**인터넷(Internet)** = 전 세계 컴퓨터를 연결하는 **네트워크들의 네트워크(network of networks)**.

Chapter 5에서 두 아이가 전선으로 통신한 것을 전 세계 규모로 확장한 것이다.

### 2.2 패킷 스위칭 (Packet Switching)

전통적 전화 통신은 **회선 교환(circuit switching)** — 통화 중에 전용 회선을 독점한다.

인터넷은 **패킷 교환(packet switching)** — 데이터를 작은 조각(**패킷**)으로 나눠 독립적으로 전송한다:

```
"Hello, World!" 메시지:

패킷 1: [헤더: 목적지=IP, 순서=1/3] [데이터: "Hello"]
패킷 2: [헤더: 목적지=IP, 순서=2/3] [데이터: ", Wor"]
패킷 3: [헤더: 목적지=IP, 순서=3/3] [데이터: "ld!"]

각 패킷은 다른 경로로 갈 수 있음 → 목적지에서 재조립
```

| 비교 | 회선 교환 | 패킷 교환 |
|------|-----------|-----------|
| 비유 | 전용 고속도로 | 일반 도로 (여러 차 공유) |
| 자원 | 통신 중 독점 | 공유 |
| 효율 | 낮음 (대기 시간도 점유) | 높음 |
| 장애 대응 | 회선 끊기면 통신 끊김 | 다른 경로로 우회 가능 |

### 2.3 TCP/IP 프로토콜

**프로토콜(protocol)** = 통신 규칙의 집합. Chapter 1의 모스 부호도 일종의 프로토콜이었다.

| 계층 | 프로토콜 | 역할 |
|------|----------|------|
| 애플리케이션 | HTTP, HTTPS, FTP, SMTP | 웹, 파일 전송, 이메일 |
| 전송 | **TCP** | 신뢰성 있는 전송 보장 (패킷 순서, 재전송) |
| 인터넷 | **IP** | 패킷 라우팅 (목적지 찾기) |
| 네트워크 | Ethernet, Wi-Fi | 물리적 전송 |

### 2.4 IP 주소

네트워크 상의 각 컴퓨터를 식별하는 고유 번호:

**IPv4**: 32비트 (Chapter 2의 2ⁿ 공식!)
```
32비트 = 2³² = 4,294,967,296 개 주소
표기: 192.168.1.1 (8비트씩 4개 = 32비트)
```

문제: 43억 개로 부족! → IPv6 도입

**IPv6**: 128비트
```
128비트 = 2¹²⁸ ≈ 3.4 × 10³⁸ 개 주소
표기: 2001:0db8:85a3:0000:0000:8a2e:0370:7334
```

### 2.5 DNS (Domain Name System)

사람은 숫자보다 이름을 기억하기 쉽다:

```
www.example.com  →  [DNS 서버]  →  93.184.216.34
```

DNS는 **도메인 이름을 IP 주소로 변환**하는 전화번호부와 같은 시스템이다.

## 3. 월드 와이드 웹 (World Wide Web)

### 3.1 인터넷 ≠ 웹

| | 인터넷 | 웹 |
|---|--------|-----|
| 본질 | **인프라** (네트워크) | **서비스** (인터넷 위의 하나의 애플리케이션) |
| 비유 | 도로 시스템 | 도로 위의 택배 서비스 |
| 탄생 | 1969 (ARPANET) | 1991 (Tim Berners-Lee) |
| 프로토콜 | TCP/IP | HTTP/HTTPS |

### 3.2 Tim Berners-Lee의 발명

1989-1991년, CERN의 물리학자 **Tim Berners-Lee**가 세 가지를 발명:

1. **HTML** — 문서의 구조를 정의하는 마크업 언어
2. **URL** — 문서의 위치를 지정하는 주소 체계
3. **HTTP** — 문서를 요청하고 전달하는 프로토콜

---

## 4. URL (Uniform Resource Locator)

웹 상의 자원 위치를 지정하는 표준 주소:

```
https://www.example.com:443/path/page.html?query=value#section
│       │                │  │              │           │
프로토콜  호스트(도메인)    포트  경로           쿼리 문자열   프래그먼트
```

| 부분 | 예시 | 역할 |
|------|------|------|
| 프로토콜 | https:// | 통신 방법 |
| 호스트 | www.example.com | 서버의 도메인 이름 |
| 포트 | :443 | 서버의 서비스 번호 (보통 생략) |
| 경로 | /path/page.html | 서버 내의 파일 위치 |

## 5. HTTP (HyperText Transfer Protocol)

### 5.1 요청과 응답

HTTP는 **요청-응답(request-response)** 프로토콜이다:

```
[웹 브라우저]                    [웹 서버]
     │                              │
     │ ── HTTP 요청 ──────────────→ │
     │    GET /index.html            │
     │                              │
     │ ←────────── HTTP 응답 ────── │
     │    200 OK                    │
     │    <html>...</html>          │
```

### 5.2 주요 메서드

| 메서드 | 용도 |
|--------|------|
| **GET** | 자원 요청 (페이지 가져오기) |
| **POST** | 데이터 제출 (폼 전송) |
| **PUT** | 자원 갱신 |
| **DELETE** | 자원 삭제 |

### 5.3 상태 코드

| 코드 | 의미 |
|------|------|
| **200** | OK — 성공 |
| **301** | Moved Permanently — 영구 이동 |
| **404** | Not Found — 페이지 없음 |
| **500** | Internal Server Error — 서버 오류 |

## 6. HTML (HyperText Markup Language)

### 6.1 마크업 언어

HTML은 문서의 **구조와 의미**를 태그(tag)로 표시하는 마크업 언어다:

```html
<!DOCTYPE html>
<html>
<head>
    <title>My Page</title>
</head>
<body>
    <h1>Hello, World!</h1>
    <p>This is a <a href="other.html">link</a>.</p>
</body>
</html>
```

| 태그 | 의미 |
|------|------|
| `<html>` | HTML 문서의 루트 |
| `<head>` | 메타 정보 (제목, 스타일 등) |
| `<body>` | 문서의 본문 내용 |
| `<h1>` | 제목 (heading) |
| `<p>` | 문단 (paragraph) |
| `<a href="...">` | 하이퍼링크 — **웹의 핵심!** |

### 6.2 하이퍼텍스트 (Hypertext)

**하이퍼링크(hyperlink)**로 문서들이 연결된 비선형 텍스트 구조:

```
[페이지 A] ──링크──→ [페이지 B] ──링크──→ [페이지 D]
     │                    │
     └──링크──→ [페이지 C] ←──링크──┘
```

이것이 "웹(Web, 거미줄)"이라 불리는 이유다 — 문서들이 링크로 거미줄처럼 얽혀 있다.

### 6.3 CSS와 JavaScript

| 기술 | 역할 | 비유 |
|------|------|------|
| **HTML** | 구조 (내용과 의미) | 건물의 뼈대 |
| **CSS** | 표현 (색상, 레이아웃, 글꼴) | 건물의 외관과 인테리어 |
| **JavaScript** | 동작 (상호작용, 동적 기능) | 건물의 전기/기계 시스템 |

## 7. 이 책의 여정 — 총정리

Chapter 1에서 시작한 여정이 여기서 완결된다:

```
Ch 1:  손전등 ON/OFF ─── 이진 신호의 탄생
Ch 2:  2ⁿ 조합 ──────── 코드의 수학적 기초
Ch 3:  점자 6비트 ────── 고정 길이 이진 코드
Ch 4:  전기 회로 ─────── 전자의 흐름
Ch 5:  전선 통신 ─────── 장거리 전송
Ch 7:  릴레이 ────────── 전기로 전기를 제어
Ch 8:  논리 게이트 ───── AND, OR, NOT
Ch 14: 가산기 ────────── 논리로 산술을
Ch 17: 플립플롭 ─────── 기억의 탄생
Ch 19: RAM ───────────── 대용량 메모리
Ch 21: ALU ───────────── 연산 엔진
Ch 23: CPU ───────────── 자동 실행 기계
Ch 26: OS ────────────── 하드웨어 추상화
Ch 27: 프로그래밍 ────── 인간의 의도를 기계어로
Ch 28: 웹 ────────────── 전 세계 컴퓨터를 연결
```

> **핵심 통찰**: 손전등의 ON/OFF에서 시작하여 월드 와이드 웹에 도달했다. 이 모든 것의 근본은 동일하다 — **두 가지 상태(0과 1)의 조합**. 웹 페이지를 볼 때 화면의 각 픽셀은 비트(RGB 값)이고, 네트워크를 통해 전송되는 데이터는 비트(패킷)이고, 서버의 CPU가 처리하는 명령어도 비트(기계어)다. Chapter 1의 손전등이 깜빡이는 것과 본질적으로 같다 — 차이는 오직 속도와 규모뿐이다.

## 요약

- **인터넷**: 전 세계 컴퓨터를 연결하는 네트워크. 패킷 교환, TCP/IP, IP 주소, DNS
- **웹**: 인터넷 위의 서비스. Tim Berners-Lee가 HTML, URL, HTTP를 발명 (1989-91)
- **URL**: 웹 자원의 주소 (프로토콜://호스트/경로)
- **HTTP**: 요청-응답 프로토콜. GET/POST 메서드, 상태 코드 (200, 404, 500)
- **HTML**: 문서의 구조를 태그로 정의. 하이퍼링크로 문서를 연결 → "웹(거미줄)"
- 이 책의 전체 여정: 손전등(Ch 1) → 전기(Ch 4) → 게이트(Ch 8) → CPU(Ch 23) → OS(Ch 26) → 프로그래밍(Ch 27) → **웹(Ch 28)**. 모든 것은 0과 1의 조합이다.
