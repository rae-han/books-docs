# Chapter 21: The Arithmetic Logic Unit (산술 논리 장치)

## 핵심 질문

> 덧셈, 뺄셈, AND, OR 등 여러 연산을 수행하는 회로들을 어떻게 하나의 장치로 통합하며, ALU의 상태 플래그는 왜 중요한가?

---

## 1. ALU란 무엇인가

**ALU (Arithmetic Logic Unit, 산술 논리 장치)**는 CPU의 핵심 연산 엔진이다. 산술 연산(덧셈, 뺄셈)과 논리 연산(AND, OR, XOR, NOT)을 **하나의 회로**로 통합한 장치다.

```
입력 A (8비트) ───→ ┌─────────┐
                    │         │ ───→ 결과 (8비트)
입력 B (8비트) ───→ │   ALU   │
                    │         │ ───→ 플래그 (Z, C, V, N)
연산 선택 ─────────→ └─────────┘
```

---

## 2. 멀티플렉서 (Multiplexer, MUX)

ALU의 핵심 부품은 **멀티플렉서(MUX)** — 여러 입력 중 하나를 선택하여 출력하는 회로다.

### 2-to-1 MUX

| 선택(S) | 출력 |
|---------|------|
| 0 | 입력 A |
| 1 | 입력 B |

### 4-to-1 MUX

| S₁ | S₀ | 출력 |
|----|-----|------|
| 0 | 0 | 입력 0 |
| 0 | 1 | 입력 1 |
| 1 | 0 | 입력 2 |
| 1 | 1 | 입력 3 |

ALU에서는 **모든 연산을 동시에 계산**하고, MUX로 원하는 연산의 결과만 **선택**하여 출력한다:

```
A,B → [가산기]  → 결과1 ─┐
A,B → [AND]    → 결과2 ─┤
A,B → [OR]     → 결과3 ─┼→ [4-to-1 MUX] → 최종 결과
A,B → [XOR]    → 결과4 ─┘
                          ↑
                     연산 선택 (2비트)
```

> **핵심 통찰**: ALU는 "각 연산마다 별도 회로를 켜고 끄는" 것이 아니다. **모든 연산이 항상 동시에 실행**되고, MUX가 원하는 결과만 골라낸다. 사용되지 않는 연산의 결과는 그냥 버려진다. 이것이 하드웨어의 병렬성이다.

---

## 3. ALU의 연산 목록

### 산술 연산

| 연산 | 기능 | 회로 |
|------|------|------|
| ADD | A + B | 리플 캐리 가산기 (Ch 14) |
| SUB | A - B | 가산기 + 2의 보수 (Ch 16) |
| INC | A + 1 | 가산기 (B=1) |
| DEC | A - 1 | 가산기 (B의 보수, Cin=0) |

### 논리 연산

| 연산 | 기능 | 회로 |
|------|------|------|
| AND | A AND B | AND 게이트 ×8 |
| OR | A OR B | OR 게이트 ×8 |
| XOR | A XOR B | XOR 게이트 ×8 |
| NOT | NOT A | NOT 게이트 ×8 |

### 시프트 연산

| 연산 | 기능 | 효과 |
|------|------|------|
| SHL | 왼쪽으로 1비트 시프트 | ×2 (곱하기 2) |
| SHR | 오른쪽으로 1비트 시프트 | ÷2 (나누기 2) |

---

## 4. 상태 플래그 (Condition Flags)

ALU는 연산 결과뿐 아니라 결과의 **특성**을 나타내는 플래그도 출력한다:

| 플래그 | 이름 | 의미 | 감지 방법 |
|--------|------|------|-----------|
| **Z** | Zero | 결과가 0인가? | 모든 출력 비트를 NOR |
| **C** | Carry | 올림이 발생했는가? | 가산기의 최종 Carry-out |
| **V** | Overflow | 오버플로우인가? | C₇ XOR C₆ (Ch 16) |
| **N** | Negative | 결과가 음수인가? | 결과의 최상위 비트(MSB) |

이 플래그들은 **플래그 레지스터(flag register)** 또는 **상태 레지스터(status register)**에 저장된다.

### 플래그의 중요성

플래그는 Chapter 24의 **조건부 점프(conditional jump)**에서 핵심 역할을 한다:

```
CMP A, B         ← A-B를 수행하고 결과는 버리지만 플래그는 설정
JZ  label        ← Zero 플래그=1이면 (즉, A==B이면) label로 점프
```

> **핵심 통찰**: ALU의 플래그는 "결과가 어떤 성질을 가지는가"를 1비트로 요약한다. 이 요약 정보가 프로그램의 흐름 제어(if-else, 반복)를 가능하게 한다. 플래그 없이는 조건 분기가 불가능하고, 조건 분기 없이는 의미 있는 프로그램을 만들 수 없다.

## 5. ALU 블록 다이어그램

```
        A (8비트)    B (8비트)
        │           │
        ↓           ↓
   ┌────────────────────────┐
   │         ALU            │
   │                        │
   │  ┌─────┐  ┌─────┐     │
   │  │가산기│  │논리  │     │
   │  │/뺄셈│  │연산  │     │
   │  └──┬──┘  └──┬──┘     │
   │     │        │        │
   │     ↓        ↓        │
   │  ┌────────────────┐   │
   │  │  멀티플렉서     │   │
   │  │  (연산 선택)    │   │
   │  └───────┬────────┘   │
   │          │             │
   └──────────┼─────────────┘
              │
        ┌─────┴─────┐
        ↓           ↓
    결과 (8비트)  플래그 (Z,C,V,N)
```

---

## 6. 실제 ALU — 74181

1970년에 Texas Instruments가 출시한 **74181**은 최초의 단일칩 4비트 ALU IC다:

- 4비트 입력 2개, 4비트 출력 1개
- 16가지 산술 연산 + 16가지 논리 연산
- 캐리 입출력 지원 (여러 칩 연결로 8, 16, 32비트 확장)
- 약 75개의 논리 게이트로 구성

이 하나의 칩이 Chapter 8-16에서 개별적으로 만든 모든 회로를 통합한 것이다.

---

## 요약

- **ALU**는 산술(ADD, SUB)과 논리(AND, OR, XOR, NOT) 연산을 하나의 회로로 통합한 CPU의 연산 엔진
- **멀티플렉서(MUX)**로 여러 연산 결과 중 하나를 선택하여 출력
- **상태 플래그**: Zero, Carry, Overflow, Negative — 결과의 특성을 1비트로 요약
- 플래그는 조건부 점프(Ch 24)의 기초 — 프로그램의 흐름 제어를 가능하게 함
- 모든 연산이 동시에 실행되고 MUX가 선택하는 구조 — 하드웨어 병렬성
