# Chapter 12: Bytes and Hexadecimal (바이트와 16진법)

## 핵심 질문

> 왜 비트를 8개씩 묶어 "바이트"라 부르며, 16진법이 프로그래머의 일상 표기법이 된 이유는 무엇인가?

---

## 1. 바이트(Byte) — 왜 8비트인가?

### 1.1 바이트의 정의

**바이트(byte)** = **8비트**의 묶음. 컴퓨터에서 데이터를 처리하는 기본 단위다.

```
1바이트 = 8비트 = □□□□□□□□
                   7654 3210  ← 비트 번호 (오른쪽부터)
```

8비트로 표현 가능한 값의 범위:
- 최솟값: 00000000₂ = 0₁₀
- 최댓값: 11111111₂ = 255₁₀
- 총 조합: 2⁸ = **256가지**

### 1.2 왜 하필 8인가?

8비트가 표준이 된 데에는 역사적, 실용적 이유가 있다:

| 이유 | 설명 |
|------|------|
| **문자 표현** | 영어 대소문자(52) + 숫자(10) + 기호/제어 문자 → 128(7비트) 이상 필요. 8비트면 256개로 충분 |
| **2의 거듭제곱** | 8 = 2³. 2의 거듭제곱은 이진 체계에서 자연스러움 |
| **IBM System/360** | 1964년 IBM이 System/360에서 8비트를 표준 단위로 채택. 업계 표준이 됨 |
| **16진 친화적** | 8비트 = 16진수 정확히 2자리. 깔끔한 대응 |

이전에는 6비트(64가지), 7비트(128가지) 등 다양한 크기가 사용되었지만, IBM System/360 이후 8비트 = 1바이트가 사실상의 세계 표준이 되었다.

---

## 2. 니블(Nibble) — 바이트의 절반

**니블(nibble)** = **4비트** = 바이트의 절반

```
1바이트 = 상위 니블 + 하위 니블
         ████ ████
         ↑     ↑
      상위4비트 하위4비트
```

4비트 = 2⁴ = 16가지 값 = **16진수 한 자리**에 정확히 대응

| 니블 값 (2진) | 니블 값 (16진) | 니블 값 (10진) |
|---------------|----------------|----------------|
| 0000 | 0 | 0 |
| 0001 | 1 | 1 |
| ... | ... | ... |
| 1001 | 9 | 9 |
| 1010 | A | 10 |
| 1011 | B | 11 |
| 1100 | C | 12 |
| 1101 | D | 13 |
| 1110 | E | 14 |
| 1111 | F | 15 |

> **Petzold의 비유**: "nibble"이라는 이름은 "byte"에 대한 말장난이다. Byte(바이트)가 bite(한 입)과 발음이 같으므로, 그 절반인 nibble(한 입 베어 물기)이 된 것이다. 컴퓨터 과학자들도 유머 감각이 있다.

## 3. 16진법이 프로그래머의 도구인 이유

### 3.1 1바이트 = 16진 2자리

| 바이트 (2진) | 상위 니블 | 하위 니블 | 16진 표현 | 10진 값 |
|-------------|-----------|-----------|-----------|---------|
| 00000000 | 0000 | 0000 | 00 | 0 |
| 00101010 | 0010 | 1010 | 2A | 42 |
| 01100001 | 0110 | 0001 | 61 | 97 |
| 10110111 | 1011 | 0111 | B7 | 183 |
| 11111111 | 1111 | 1111 | FF | 255 |

### 3.2 16진법의 실용적 이점

```
10진:  2진수 10110111을 10진수로 바꾸려면...
       128+32+16+4+2+1 = 183  (계산 필요)

16진:  10110111 → 1011 0111 → B7  (즉시 변환, 계산 불필요!)
```

프로그래머가 16진법을 일상적으로 사용하는 이유:

| 상황 | 16진 사용 이유 |
|------|----------------|
| 메모리 주소 | `0x7FFF` — 이진 구조가 바로 보임 |
| 색상 코드 | `#FF8000` — R=FF, G=80, B=00 |
| 바이트 값 | `0xA5` — 10100101₂임을 즉시 알 수 있음 |
| 디버깅 | 메모리 덤프가 16진으로 표시됨 |
| 비트 마스크 | `0x0F` = 하위 4비트 선택 |

## 4. 메모리 크기 단위

### 4.1 이진 접두어 (Binary Prefixes)

컴퓨터 메모리는 2의 거듭제곱 단위로 구성된다:

| 단위 | 바이트 수 | 2의 거듭제곱 | 대략적 크기 |
|------|-----------|-------------|-------------|
| 1 KB (킬로바이트) | 1,024 | 2¹⁰ | ~천 |
| 1 MB (메가바이트) | 1,048,576 | 2²⁰ | ~백만 |
| 1 GB (기가바이트) | 1,073,741,824 | 2³⁰ | ~십억 |
| 1 TB (테라바이트) | 1,099,511,627,776 | 2⁴⁰ | ~일조 |

### 4.2 SI 접두어 vs 이진 접두어

혼란의 원인: "킬로"가 1,000인지 1,024인지?

| SI 접두어 | 값 | 이진 접두어 | 값 |
|-----------|-----|------------|-----|
| kilo (k) | 10³ = 1,000 | kibi (Ki) | 2¹⁰ = 1,024 |
| mega (M) | 10⁶ = 1,000,000 | mebi (Mi) | 2²⁰ = 1,048,576 |
| giga (G) | 10⁹ = 1,000,000,000 | gibi (Gi) | 2³⁰ = 1,073,741,824 |
| tera (T) | 10¹² | tebi (Ti) | 2⁴⁰ |

IEC에서 1998년에 이진 접두어(KiB, MiB, GiB)를 공식 도입했지만, 현실에서는 여전히 KB, MB, GB가 맥락에 따라 1,000 기반 또는 1,024 기반으로 혼용된다.

> **핵심 통찰**: "500GB SSD"라고 광고하지만, 운영체제에서는 465GB로 표시되는 이유가 여기 있다. 제조사는 1GB = 10⁹ = 1,000,000,000 바이트로 계산하고, OS는 1GB = 2³⁰ = 1,073,741,824 바이트로 계산하기 때문이다.

## 5. 바이트의 다양한 해석

같은 바이트 값 `01100001` (= 0x61 = 97₁₀)이 맥락에 따라 완전히 다른 의미를 가진다:

| 해석 | 값 | 의미 |
|------|-----|------|
| 부호 없는 정수 | 97 | 숫자 97 |
| ASCII 문자 | 'a' | 소문자 a |
| 기계어 명령어 | (아키텍처에 따라 다름) | CPU 명령어 |
| 색상 채널 | 97/255 ≈ 38% | 밝기 38% |

> **핵심 통찰**: 비트와 바이트 자체에는 의미가 없다. **해석(interpretation)**이 의미를 부여한다. 같은 비트 패턴이 숫자, 문자, 명령어, 색상이 될 수 있다. 컴퓨터는 비트 패턴이 "무엇을 의미하는지" 모른다 — 프로그램이 정해준 규칙에 따라 처리할 뿐이다.

---

## 6. 워드(Word) — 바이트의 확장

CPU가 한 번에 처리하는 비트의 수를 **워드(word)**라 한다:

| 시대 | 워드 크기 | 바이트 수 | 대표 CPU |
|------|-----------|-----------|----------|
| 1970s | 8비트 | 1 | Intel 8080, Z80 |
| 1980s | 16비트 | 2 | Intel 8086, 68000 |
| 1990s | 32비트 | 4 | Intel 386, Pentium |
| 2000s~ | 64비트 | 8 | x86-64, ARM64 |

워드 크기가 클수록:
- 더 큰 수를 한 번에 처리 가능 (32비트: ~43억, 64비트: ~1.8×10¹⁹)
- 더 큰 메모리 주소 공간 (32비트: 4GB, 64비트: 16EB)
- 더 정밀한 부동소수점 연산

---

## 요약

- **바이트(byte)** = 8비트 = 256가지 값(0-255). 데이터 처리의 기본 단위
- **니블(nibble)** = 4비트 = 16가지 값. 16진수 한 자리에 대응
- **16진법**은 2진법의 축약 표기: 1바이트 = 16진 2자리. 4비트씩 끊어 변환
- **메모리 단위**: KB(2¹⁰), MB(2²⁰), GB(2³⁰), TB(2⁴⁰). SI 접두어와 이진 접두어의 차이에 주의
- 같은 바이트 값도 맥락에 따라 숫자, 문자, 명령어 등 다르게 **해석**된다
- **워드(word)**: CPU가 한 번에 처리하는 비트 수 (현대 CPU: 64비트)
