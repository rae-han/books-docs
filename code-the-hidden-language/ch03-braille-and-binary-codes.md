# Chapter 3: Braille and Binary Codes (점자와 이진 코드)

## 핵심 질문

> 고정 길이 이진 코드는 가변 길이 코드와 어떻게 다르며, 제한된 조합 수를 넘어서는 정보를 표현하려면 어떤 트릭이 필요한가?

---

## 1. Louis Braille — 15세 소년의 발명

Louis Braille(1809-1852)는 세 살 때 사고로 시력을 잃었다. 파리의 왕립 맹인학교에 다니던 그는 당시 사용되던 양각 문자(embossed letters)가 읽기 어렵다는 문제를 절감했다. 양각 문자는 눈으로 읽는 글자를 그대로 볼록하게 만든 것이어서, 손끝으로 구분하기가 극히 어려웠다.

군대의 야간 통신 체계인 "야간 문자(night writing)"에서 영감을 받은 Braille는 1824년, 불과 15세에 자신만의 코드 체계를 고안했다. 이것이 오늘날까지 전 세계에서 사용되는 **점자(Braille)**다.

> **Petzold의 비유**: 양각 문자가 실패한 이유는 "눈을 위해 설계된 코드를 손끝에 강제한 것"이기 때문이다. Braille의 혁신은 손끝이라는 **수신 장치**에 최적화된 완전히 새로운 코드를 만든 것이다. 이것은 컴퓨터 설계에서도 반복되는 교훈이다 — 매체(medium)에 맞는 코드를 설계해야 한다.

---

## 2. 점자 셀의 구조

점자의 기본 단위는 **셀(cell)**이다. 하나의 셀은 2열 × 3행 = **6개의 점 위치**로 구성된다:

```
┌───┬───┐
│ 1 │ 4 │
├───┼───┤
│ 2 │ 5 │
├───┼───┤
│ 3 │ 6 │
└───┴───┘
```

각 위치에 점이 **올라와 있거나(raised)** **평평하거나(flat)** 둘 중 하나다. 즉, 각 점은 두 가지 상태를 가진다:

- **올라옴** = 1
- **평평함** = 0

6개의 점, 각각 2가지 상태 → 총 조합 수:

```
2⁶ = 64가지
```

Chapter 2에서 배운 2ⁿ 공식이 그대로 적용된다.

## 3. 모스 부호와 점자의 근본적 차이

| 특성 | 모스 부호 | 점자 |
|------|-----------|------|
| 코드 길이 | **가변 길이** (1~4개 신호) | **고정 길이** (항상 6비트) |
| 기본 신호 | 점(·)과 선(—) | 올라옴(1)과 평평함(0) |
| 구분자 필요? | **필요** (침묵으로 글자 구분) | **불필요** (항상 6비트씩 끊으면 됨) |
| 인간 효율 | 높음 (자주 쓰는 글자가 짧음) | 보통 (모든 글자가 같은 크기) |
| 기계 처리 | 어려움 (타이밍 해석 필요) | **쉬움** (고정 크기 단위 처리) |

> **핵심 통찰**: 점자는 **고정 길이 이진 코드(fixed-length binary code)**의 가장 오래된 실용적 예시다. 각 셀이 정확히 6비트이므로 구분자 없이도 글자 경계를 알 수 있다. 이것은 현대 컴퓨터가 데이터를 8비트(1바이트) 단위로 처리하는 것과 같은 원리다.

---

## 4. 점자 코드표

### 4.1 알파벳

점자 알파벳은 놀라울 정도로 체계적으로 설계되어 있다. 처음 10개 글자(a-j)는 상위 4개 점(1, 2, 4, 5)만 사용한다:

| 글자 | 점 번호 | 패턴 | 글자 | 점 번호 | 패턴 |
|------|---------|------|------|---------|------|
| a | 1 | `⠁` | f | 1,2,4 | `⠋` |
| b | 1,2 | `⠃` | g | 1,2,4,5 | `⠛` |
| c | 1,4 | `⠉` | h | 1,2,5 | `⠓` |
| d | 1,4,5 | `⠙` | i | 2,4 | `⠊` |
| e | 1,5 | `⠑` | j | 2,4,5 | `⠚` |

다음 10개(k-t)는 a-j의 패턴에 점 3을 추가한 것이다:

| 글자 | 점 번호 | 기반 |
|------|---------|------|
| k | 1,3 | a + 점3 |
| l | 1,2,3 | b + 점3 |
| m | 1,3,4 | c + 점3 |
| n | 1,3,4,5 | d + 점3 |
| o | 1,3,5 | e + 점3 |

그 다음(u-z)은 a-j에 점 3과 점 6을 추가한 패턴이다(w 제외 — Braille가 프랑스인이라 당시 프랑스어에서 w가 쓰이지 않았기 때문).

이처럼 점자 알파벳은 무작위가 아니라 **규칙적인 패턴**으로 구성되어 있어 학습이 용이하다.

### 4.2 숫자

여기서 중요한 문제가 발생한다. 6비트 = 64가지 조합으로 알파벳 26개 + 구두점 + 숫자까지 표현해야 한다. 조합이 부족하다!

## 5. 이스케이프 코드와 시프트 코드 — 64개의 한계를 넘어서

Braille의 가장 영리한 해결책이 여기 있다.

### 5.1 숫자 지시자 (Number Indicator)

점자에서 숫자 1-9, 0은 글자 a-j와 **동일한 패턴**을 사용한다:

| 숫자 | 동일한 글자 | 점 번호 |
|------|-------------|---------|
| 1 | a | 1 |
| 2 | b | 1,2 |
| 3 | c | 1,4 |
| 4 | d | 1,4,5 |
| 5 | e | 1,5 |
| 6 | f | 1,2,4 |
| 7 | g | 1,2,4,5 |
| 8 | h | 1,2,5 |
| 9 | i | 2,4 |
| 0 | j | 2,4,5 |

그렇다면 수신자가 "a"와 "1"을 어떻게 구분하는가?

**숫자 지시자(number indicator)**: 점 3,4,5,6(`⠼`)이라는 특별한 셀이 숫자들 앞에 온다. 이 셀 자체는 어떤 문자도 나타내지 않으며, "다음에 오는 셀들은 숫자로 읽어라"라는 **모드 전환 신호**다.

```
⠼ ⠁ ⠃ ⠉  →  "123"  (숫자 지시자 + a패턴 + b패턴 + c패턴)
   ⠁ ⠃ ⠉  →  "abc"  (숫자 지시자 없이 → 글자로 읽음)
```

### 5.2 대문자 지시자 (Capital Indicator)

마찬가지로, 점 6(`⠠`)은 "다음 글자를 대문자로 읽어라"는 신호다:

```
⠠ ⠓ ⠑ ⠇ ⠇ ⠕  →  "Hello"  (대문자 지시자 + h + e + l + l + o)
```

### 5.3 이것이 의미하는 것

이 특별한 셀들의 공식 이름이 바로 **이스케이프 코드(escape code)** 또는 **시프트 코드(shift code)**다.

| 용어 | 의미 | 점자 예시 |
|------|------|-----------|
| 이스케이프 코드 | "다음 코드의 해석 방식을 바꿔라" | 숫자 지시자: 글자→숫자 |
| 시프트 코드 | "다음 코드를 다른 모드로 읽어라" | 대문자 지시자: 소문자→대문자 |

> **핵심 통찰**: 이스케이프 코드는 **유한한 코드 공간을 확장하는 보편적 기법**이다. 64개의 조합으로 표현할 수 있는 것은 64가지뿐이지만, 이스케이프 코드를 사용하면 같은 패턴을 여러 의미로 재사용할 수 있다. 대가는 **추가 셀(오버헤드)**이 필요하다는 것이다.

이 원리는 컴퓨터에서 끊임없이 반복된다:

```
점자 숫자 지시자    →  ASCII 이스케이프 문자 (ESC, 0x1B)
점자 대문자 지시자  →  키보드 Shift 키
점자 모드 전환      →  UTF-8의 멀티바이트 인코딩
```

## 6. 점자에서 컴퓨터 코드로

점자와 컴퓨터 문자 코드의 구조적 유사성을 정리하면:

| 점자 | 컴퓨터 | 공통 원리 |
|------|--------|-----------|
| 6비트 셀 | 8비트 바이트 | 고정 길이 이진 코드 |
| 64가지 조합 | 256가지 조합 | 2ⁿ 조합 |
| 숫자 지시자 | Shift 키, ESC 시퀀스 | 이스케이프/시프트 코드 |
| 점의 올라옴/평평함 | 전압의 높음/낮음 | 이진 상태 |
| 규칙적 패턴 (a-j → k-t) | ASCII의 대소문자 차이=32 | 체계적 코드 설계 |

> **Petzold의 비유**: 점자는 1800년대에 만들어진 "6비트 컴퓨터 코드"라고 할 수 있다. 고정 길이, 이진 상태, 이스케이프 코드까지 — 현대 컴퓨터의 문자 인코딩과 놀라울 정도로 동일한 구조를 가지고 있다.

## 요약

- **점자(Braille)**는 2×3 격자의 6개 점을 사용하는 **고정 길이 6비트 이진 코드**다
- 6비트 = 2⁶ = 64가지 조합으로 알파벳, 숫자, 구두점을 모두 표현한다
- 64개로 부족한 경우 **이스케이프 코드/시프트 코드**로 같은 패턴을 다른 의미로 재사용한다
- 고정 길이 코드는 구분자가 필요 없어 기계적 처리에 유리하다
- 점자의 설계 원리(고정 길이, 이진 상태, 이스케이프 코드)는 현대 컴퓨터의 문자 인코딩과 구조적으로 동일하다
