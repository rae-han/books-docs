# Chapter 26: The Operating System (운영체제)

## 핵심 질문

> 하드웨어 위에 소프트웨어 계층을 쌓아 사용자와 애플리케이션에 편리한 환경을 제공하는 운영체제란 무엇이며, 어떤 핵심 기능을 수행하는가?

---

## 1. 운영체제란?

**운영체제(Operating System, OS)**는 하드웨어를 관리하고 애플리케이션에 서비스를 제공하는 **시스템 소프트웨어**다.

```
┌────────────────────────────────┐
│   애플리케이션 (브라우저, 에디터)  │
├────────────────────────────────┤
│     운영체제 (Windows, Linux)    │ ← 하드웨어를 추상화
├────────────────────────────────┤
│     하드웨어 (CPU, RAM, 디스크)   │
└────────────────────────────────┘
```

> **Petzold의 비유**: 운영체제는 호텔의 프론트 데스크와 같다. 투숙객(애플리케이션)은 방(메모리), 시설(CPU), 서비스(I/O)를 직접 관리할 필요 없이 프론트(OS)에 요청하면 된다. 프론트가 방 배정, 시설 예약, 서비스 조율을 담당한다.

---

## 2. 역사적 발전

### 2.1 OS 없는 시대 (1940-50s)

- 프로그래머가 기계어로 직접 하드웨어 제어
- 한 번에 하나의 프로그램만 실행
- 프로그램 교체 = 물리적 카드/테이프 교환 + 대기

### 2.2 배치 처리 (Batch Processing, 1950-60s)

- 작업(job)을 미리 준비하여 순차적으로 실행
- 운영자가 작업을 관리
- CPU가 유휴 상태인 시간(I/O 대기)이 많다는 문제

### 2.3 시분할 (Time-Sharing, 1960-70s)

- 여러 사용자/프로그램이 CPU를 **번갈아** 사용
- 각 프로그램에 짧은 시간(time slice)을 할당
- 사용자에게는 마치 컴퓨터를 독점하는 것처럼 느껴짐
- Unix (1969): 시분할 OS의 대표

---

## 3. 사용자 인터페이스

### 3.1 명령줄 인터페이스 (CLI)

텍스트 명령어로 컴퓨터를 제어:

```
$ ls            ← 파일 목록
$ cd documents  ← 디렉토리 이동
$ cp a.txt b.txt ← 파일 복사
```

- **CP/M** (1974): 최초의 마이크로컴퓨터 OS
- **MS-DOS** (1981): IBM PC와 함께 대중화
- **Unix shell**: 강력한 파이프라인과 스크립팅

### 3.2 그래픽 사용자 인터페이스 (GUI)

- **Xerox PARC** (1973): 최초의 GUI 연구
- **Apple Macintosh** (1984): GUI를 대중에 소개
- **Windows** (1985~): GUI의 대중적 표준

**WIMP**: Window, Icon, Menu, Pointer — GUI의 4대 요소

---

## 4. 핵심 OS 기능

### 4.1 프로세스 관리

**프로세스(process)** = 실행 중인 프로그램의 인스턴스

| 기능 | 설명 |
|------|------|
| **멀티태스킹** | 여러 프로세스를 번갈아 실행 (시분할) |
| **컨텍스트 스위칭** | CPU 상태(레지스터, PC, 플래그)를 저장/복원하여 프로세스 전환 |
| **스케줄링** | 어떤 프로세스에 CPU를 할당할지 결정 |

컨텍스트 스위칭:
```
프로세스 A 실행 중
  → 타이머 인터럽트 발생
  → A의 상태 저장 (레지스터, PC 등)
  → 스케줄러가 프로세스 B 선택
  → B의 상태 복원
  → B 실행 계속
```

### 4.2 메모리 관리

| 기능 | 설명 |
|------|------|
| **메모리 할당** | 각 프로세스에 메모리 영역 배정 |
| **보호** | 한 프로세스가 다른 프로세스의 메모리에 접근 못하게 |
| **가상 메모리** | 물리 RAM보다 큰 메모리 공간을 제공 |

**가상 메모리(Virtual Memory)**:
- 각 프로세스에게 "자기만의 메모리 공간"이 있는 것처럼 보이게 함
- 실제로는 물리 RAM + 디스크 스왑 공간을 조합
- **페이징(paging)**: 메모리를 고정 크기 페이지(4KB 등)로 나누어 관리

### 4.3 파일 시스템

| 개념 | 설명 |
|------|------|
| **파일** | 이름이 붙은 데이터의 묶음 |
| **디렉토리(폴더)** | 파일을 조직하는 계층 구조 |
| **경로** | 파일의 위치: /home/user/document.txt |
| **파일 시스템** | 디스크에 파일을 저장하고 관리하는 체계 (FAT, NTFS, ext4 등) |

### 4.4 장치 드라이버

- 각 하드웨어 장치의 세부사항을 추상화
- OS에 표준 인터페이스를 제공
- 애플리케이션은 "파일에 쓰기"만 하면 되고, 드라이버가 HDD/SSD의 차이를 처리

## 5. API와 시스템 콜

### 5.1 시스템 콜 (System Call)

애플리케이션이 OS의 서비스를 요청하는 메커니즘:

```
애플리케이션: "파일을 열어줘" → 시스템 콜 → OS 커널이 처리 → 결과 반환
```

주요 시스템 콜:

| 카테고리 | 시스템 콜 | 기능 |
|----------|-----------|------|
| 파일 | open, read, write, close | 파일 조작 |
| 프로세스 | fork, exec, exit, wait | 프로세스 관리 |
| 메모리 | malloc, free, mmap | 메모리 할당 |
| I/O | ioctl, select | 장치 제어 |

### 5.2 API (Application Programming Interface)

OS가 애플리케이션에 제공하는 **함수와 규칙의 집합**. 시스템 콜을 더 편리하게 사용할 수 있도록 래핑한 것.

> **핵심 통찰**: 운영체제의 본질은 **추상화(abstraction)**다. 복잡한 하드웨어 세부사항을 숨기고, 애플리케이션에 단순한 인터페이스(API)를 제공한다. "파일에 쓰기"라는 하나의 API 호출 뒤에는 디스크 컨트롤러, 파일 시스템, 인터럽트 핸들러 등 수많은 하드웨어 상호작용이 숨어 있다.

---

## 6. 현대 운영체제

| OS | 특징 | 사용처 |
|----|------|--------|
| **Windows** | GUI 중심, 호환성, 시장 점유율 1위 | 데스크톱, 노트북 |
| **macOS** | Unix 기반, Apple 하드웨어 전용 | Mac |
| **Linux** | 오픈소스, Unix 계열 | 서버, 클라우드, 임베디드 |
| **iOS** | macOS 파생, 터치 최적화 | iPhone, iPad |
| **Android** | Linux 커널 기반 | 스마트폰, 태블릿 |

---

## 요약

- **운영체제**: 하드웨어를 관리하고 애플리케이션에 서비스를 제공하는 시스템 소프트웨어
- **역사**: OS 없음 → 배치 처리 → 시분할 → 현대 멀티태스킹 OS
- **핵심 기능**: 프로세스 관리(멀티태스킹), 메모리 관리(가상 메모리), 파일 시스템, 장치 드라이버
- **시스템 콜/API**: 애플리케이션이 OS 서비스를 요청하는 인터페이스
- OS의 본질은 **추상화** — 복잡한 하드웨어를 단순한 인터페이스로 감싸기
