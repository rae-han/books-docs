# Chapter 17: Feedback and Flip-Flops (피드백과 플립플롭)

## 핵심 질문

> 논리 게이트의 출력을 다시 입력에 연결하면 어떤 일이 일어나며, 이 "피드백"으로 어떻게 1비트를 기억하는 회로를 만들 수 있는가?

---

## 1. 조합 논리에서 순차 논리로

지금까지 만든 모든 회로(가산기, 뺄셈기 등)는 **조합 논리(combinational logic)**였다:
- 출력은 **현재 입력에만** 의존
- 입력이 바뀌면 출력이 즉시 바뀜
- **기억(memory)** 능력이 없음

계산기를 만들려면 **이전 결과를 기억**해야 한다. 첫 번째 덧셈의 결과를 저장해두고 그 위에 두 번째 덧셈을 해야 한다.

이를 위해 **순차 논리(sequential logic)** 회로가 필요하다:
- 출력이 현재 입력 **+ 이전 상태**에 의존
- **상태(state)**를 저장할 수 있음 = **기억**

핵심 기법: **피드백(feedback)** — 출력을 다시 입력에 연결

---

## 2. SR 래치 (SR Latch)

### 2.1 NOR 게이트로 만드는 SR 래치

두 NOR 게이트의 출력을 서로의 입력에 **교차 연결(cross-coupled)**한다:

```
S ──┐                  ┌── Q
    │  ┌─────────┐     │
    ├──┤ NOR  1  ├─────┤
    │  └─────────┘     │
    │         ↑         │
    │         │ (교차)   │
    │         ↓         │
    │  ┌─────────┐     │
    ├──┤ NOR  2  ├─────┤
R ──┘  └─────────┘     └── Q̄ (Q의 반전)
```

- **S (Set)**: Q를 1로 만든다
- **R (Reset)**: Q를 0으로 만든다
- **Q**: 출력 (저장된 값)
- **Q̄**: Q의 반전

### 2.2 동작 분석

**S=1, R=0 (Set):**
- NOR 1의 입력에 S=1 → 출력 Q=0... 아니, 더 정확하게:
- S=1이 NOR 1에 들어가면 → Q̄=0 → Q̄=0이 NOR 2에 들어가고 R=0이므로 → Q=1
- 결과: **Q=1** (Set 상태)

**S=0, R=1 (Reset):**
- R=1이 NOR 2에 들어가면 → Q=0 → Q=0이 NOR 1에 들어가고 S=0이므로 → Q̄=1
- 결과: **Q=0** (Reset 상태)

**S=0, R=0 (유지):**
- 둘 다 0이면 이전 상태가 그대로 유지된다!
- 만약 Q=1이었으면 → Q̄=0 → NOR 1에 (S=0, Q=0) → Q̄=0 유지 → Q=1 유지
- 만약 Q=0이었으면 → Q̄=1 → NOR 1에 (S=0, Q̄=1) → 결과 유지

**S=1, R=1 (금지):**
- 두 출력이 모두 0이 되어 모순 상태. **사용 금지.**

### 2.3 진리표

| S | R | Q (다음) | 동작 |
|---|---|----------|------|
| 0 | 0 | Q (이전) | **유지** (기억!) |
| 1 | 0 | 1 | **Set** |
| 0 | 1 | 0 | **Reset** |
| 1 | 1 | — | **금지** |

> **핵심 통찰**: SR 래치는 **1비트 메모리**다. S=1로 1을 저장하고, R=1로 0을 저장하고, S=R=0으로 저장된 값을 유지한다. 이것이 컴퓨터 메모리의 가장 원시적인 형태다.

## 3. 레벨 트리거 D 래치 (Level-Triggered D Latch)

### 3.1 SR 래치의 문제

SR 래치는 S와 R 두 개의 입력을 관리해야 하고, S=R=1이라는 금지 상태가 존재한다. 더 편리한 구조가 필요하다.

### 3.2 D 래치의 구조

**D(Data)** 입력과 **E(Enable)** 또는 **CLK(Clock)** 입력을 가진다:

```
D ──┬──────── [AND] ──── S ──┐
    │            │             │ SR
    │           E/CLK         │ 래치 ── Q
    │            │             │
    └── [NOT] ── [AND] ──── R ──┘
```

- E=1일 때: D의 값이 Q에 전달됨 (D=1 → S=1,R=0 → Q=1 / D=0 → S=0,R=1 → Q=0)
- E=0일 때: S=0, R=0 → 이전 상태 유지 (기억)

### 3.3 진리표

| E (Enable) | D (Data) | Q (다음) |
|------------|----------|----------|
| 0 | × | Q (이전) — **유지** |
| 1 | 0 | 0 — D를 저장 |
| 1 | 1 | 1 — D를 저장 |

> **Petzold의 비유**: D 래치는 "문이 있는 방"과 같다. Enable이 1이면 문이 열려서 데이터가 들어온다. Enable이 0이면 문이 닫혀서 안에 있는 데이터가 그대로 보존된다.

---

## 4. 에지 트리거 D 플립플롭 (Edge-Triggered D Flip-Flop)

### 4.1 레벨 트리거의 문제

D 래치는 E=1인 **동안** 계속 D의 값을 통과시킨다(투명). 복잡한 회로에서 이것은 문제를 일으킬 수 있다 — 클럭이 1인 동안 입력이 여러 번 바뀌면 출력도 여러 번 바뀐다.

### 4.2 에지 트리거

**에지 트리거 D 플립플롭**은 클럭의 **상승 에지(rising edge)**에서만 — 즉, 클럭이 0→1로 전환되는 **그 순간**에만 — 데이터를 캡처한다.

```
CLK:  ___╱‾‾‾╲___╱‾‾‾╲___
          ↑        ↑
       이 순간만  이 순간만
       D를 캡처   D를 캡처
```

### 4.3 마스터-슬레이브 구조

에지 트리거 플립플롭은 D 래치 2개를 직렬로 연결하여 구현한다:

```
D ──→ [D래치 (마스터)] ──→ [D래치 (슬레이브)] ──→ Q
       E = NOT CLK          E = CLK
```

- CLK=0: 마스터 열림(D 받음), 슬레이브 닫힘(유지)
- CLK=1: 마스터 닫힘(유지), 슬레이브 열림(마스터의 값을 출력)
- 효과: CLK가 0→1로 바뀌는 순간의 D 값만 캡처

## 5. 래치 vs 플립플롭

| 특성 | D 래치 | D 플립플롭 |
|------|--------|-----------|
| 동작 시점 | Enable=1인 **동안** | 클럭 에지 **순간** |
| 투명도 | 투명 (Enable=1이면 입력이 바로 출력) | 불투명 (에지에서만 캡처) |
| 동기화 | 비동기적 | **동기적** (클럭에 맞춤) |
| 복잡도 | 낮음 | 높음 (래치 2개) |
| 용도 | 간단한 저장 | **동기식 회로의 표준** |

> **핵심 통찰**: 현대 디지털 회로의 대부분은 **동기식(synchronous)** 설계다. 모든 플립플롭이 같은 클럭 신호에 맞춰 동시에 데이터를 캡처한다. 이것이 복잡한 회로에서 타이밍 문제를 방지하고 예측 가능한 동작을 보장하는 방법이다. 클럭은 오케스트라의 지휘자와 같다.

---

## 6. 1비트 메모리에서 컴퓨터 메모리로

플립플롭 1개 = 1비트 저장

```
플립플롭 1개    = 1비트
플립플롭 8개    = 1바이트 (8비트 레지스터)
플립플롭 N×8개  = N바이트 메모리
```

이것이 Chapter 19(RAM)와 Chapter 22(레지스터)의 기초다.

|------|------|------|
| 1 | NOR 교차 연결 | SR 래치 — 기본 1비트 기억 |

## 요약

- **피드백(feedback)**: 출력을 입력에 연결하면 회로가 상태를 기억할 수 있다
- **SR 래치**: NOR 게이트 2개로 만든 가장 단순한 1비트 메모리. Set/Reset으로 제어
- **D 래치**: 단일 Data 입력 + Enable. E=1이면 투명, E=0이면 기억
- **D 플립플롭**: 클럭의 상승 에지에서만 데이터 캡처. 동기식 회로의 핵심
- 조합 논리(가산기 등)에 순차 논리(플립플롭)를 더하면 **기억하고 계산하는 기계** = 컴퓨터의 기본 구조가 완성된다
