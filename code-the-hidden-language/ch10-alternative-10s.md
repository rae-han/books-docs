# Chapter 10: Alternative 10s (대안적 10들)

## 핵심 질문

> 10 이외의 기수를 사용하는 수 체계는 어떻게 작동하며, 왜 컴퓨터에는 2진법이, 프로그래머에게는 16진법이 필수적인가?

---

## 1. "10"은 항상 기수를 의미한다

Petzold는 제목 "Alternative 10s"에서 영리한 관찰을 한다: **어떤 진법에서든 기수 자체는 "10"으로 쓰인다.**

- 10진법에서 10₁₀ = 십 (10개의 손가락)
- 2진법에서 10₂ = 이 (2개의 값)
- 8진법에서 10₈ = 여덟 (8개의 값)
- 16진법에서 10₁₆ = 열여섯 (16개의 값)

왜냐하면 "10"이란 "한 자리가 꽉 차서 다음 자리로 넘어간 최소 수"를 의미하기 때문이다.

> **Petzold의 비유**: 자동차 주행 거리계(odometer)를 생각해보라. 한 자리가 9에서 한 칸 더 돌면 0으로 돌아가고 왼쪽 자리가 1 증가한다. 이것이 모든 진법에서 동일하게 작동한다 — 다만 "꽉 차는" 숫자가 다를 뿐이다.

## 2. 8진법 (Octal, 기수 8)

### 2.1 기본 구조

| 10진법 | 8진법 | 설명 |
|--------|-------|------|
| 0 | 0 | |
| 1 | 1 | |
| 2 | 2 | |
| ... | ... | |
| 7 | 7 | 8진법의 마지막 한 자리 숫자 |
| 8 | **10** | 한 자리 꽉 참 → 올림 |
| 9 | 11 | |
| 10 | 12 | |
| 15 | 17 | |
| 16 | 20 | |
| 63 | 77 | |
| 64 | **100** | 8² = 64 |

자릿값: 8⁰=1, 8¹=8, 8²=64, 8³=512, ...

### 2.2 8진법 계산 예시

8진법 수 372₈를 10진법으로:

```
372₈ = 3×8² + 7×8¹ + 2×8⁰
     = 3×64 + 7×8 + 2×1
     = 192 + 56 + 2
     = 250₁₀
```

## 3. 2진법 (Binary, 기수 2)

### 3.1 왜 컴퓨터에 2진법인가?

컴퓨터가 2진법을 사용하는 이유는 물리적 장치가 **두 가지 상태**를 구분하기 가장 쉽기 때문이다:

| 장치 | 상태 1 (0) | 상태 2 (1) |
|------|-----------|-----------|
| 스위치 | 열림 | 닫힘 |
| 전압 | 낮음 (0V) | 높음 (5V) |
| 전구 | 꺼짐 | 켜짐 |
| 자기 | N극 | S극 |
| 릴레이 | 비활성 | 활성 |
| 트랜지스터 | 차단 | 도통 |

세 가지 이상의 상태를 구분하려면 회로가 복잡해지고 오류에 취약해진다. 두 가지만 구분하면 **잡음(noise)**에 강하고 **안정적**이다.

> **핵심 통찰**: 2진법은 수학적으로 가장 비효율적인 진법이다 — 같은 수를 표현하는 데 가장 많은 자릿수가 필요하다. 하지만 **물리적으로 가장 안정적**이다. 컴퓨터는 이 트레이드오프에서 안정성을 선택했다. 자릿수가 많아지는 대신, 초당 수십억 번 연산할 수 있는 속도로 보상한다.

### 3.2 2진법 세기

| 10진법 | 2진법 | 비트 수 |
|--------|-------|---------|
| 0 | 0 | 1 |
| 1 | 1 | 1 |
| 2 | 10 | 2 |
| 3 | 11 | 2 |
| 4 | 100 | 3 |
| 5 | 101 | 3 |
| 6 | 110 | 3 |
| 7 | 111 | 3 |
| 8 | 1000 | 4 |
| 9 | 1001 | 4 |
| 10 | 1010 | 4 |
| 15 | 1111 | 4 |
| 16 | 10000 | 5 |
| 255 | 11111111 | 8 |
| 256 | 100000000 | 9 |

자릿값: 2⁰=1, 2¹=2, 2²=4, 2³=8, 2⁴=16, 2⁵=32, 2⁶=64, 2⁷=128, ...

### 3.3 10진 → 2진 변환

**방법: 반복 나누기**

10진수 42를 2진법으로:

```
42 ÷ 2 = 21  나머지 0  ← 최하위 비트(LSB)
21 ÷ 2 = 10  나머지 1
10 ÷ 2 = 5   나머지 0
 5 ÷ 2 = 2   나머지 1
 2 ÷ 2 = 1   나머지 0
 1 ÷ 2 = 0   나머지 1  ← 최상위 비트(MSB)
```

나머지를 아래에서 위로 읽으면: **101010₂**

검산: 1×32 + 0×16 + 1×8 + 0×4 + 1×2 + 0×1 = 32 + 8 + 2 = 42 ✓

### 3.4 2진 덧셈

10진법의 덧셈과 같은 원리. 단, 1+1=10 (올림 발생):

```
  1 0 1 1 0 1   (45)
+ 0 1 1 0 1 1   (27)
────────────────
```

오른쪽부터:
1. 1+1 = 10 → 0 쓰고 1 올림
2. 0+1+1 = 10 → 0 쓰고 1 올림
3. 1+0+1 = 10 → 0 쓰고 1 올림
4. 1+1+1 = 11 → 1 쓰고 1 올림
5. 0+1+1 = 10 → 0 쓰고 1 올림
6. 1+0+1 = 10 → 0 쓰고 1 올림

```
  1 0 1 1 0 1   (45)
+ 0 1 1 0 1 1   (27)
────────────────
1 0 0 1 0 0 0   (72)
```

검산: 64 + 8 = 72 ✓

## 4. 16진법 (Hexadecimal, 기수 16)

### 4.1 왜 16진법인가?

2진법은 컴퓨터에 적합하지만 사람에게는 읽기 어렵다: 11010110은 한눈에 얼마인지 알기 어렵다.

16진법은 2진법의 **축약 표기법**이다:

- 16 = 2⁴ → 16진수 한 자리 = 정확히 4비트
- 8비트(1바이트) = 16진수 **정확히 2자리**
- 변환이 기계적이고 단순

### 4.2 16진법의 숫자

0-9까지는 10진법과 같고, 10-15에는 알파벳 A-F를 사용한다:

| 10진 | 16진 | 2진 |
|------|------|------|
| 0 | 0 | 0000 |
| 1 | 1 | 0001 |
| 2 | 2 | 0010 |
| 3 | 3 | 0011 |
| 4 | 4 | 0100 |
| 5 | 5 | 0101 |
| 6 | 6 | 0110 |
| 7 | 7 | 0111 |
| 8 | 8 | 1000 |
| 9 | 9 | 1001 |
| 10 | **A** | 1010 |
| 11 | **B** | 1011 |
| 12 | **C** | 1100 |
| 13 | **D** | 1101 |
| 14 | **E** | 1110 |
| 15 | **F** | 1111 |

### 4.3 2진 ↔ 16진 변환

**이 변환은 기계적으로 할 수 있다 — 4비트씩 끊어서 변환:**

```
2진:   1101 0110
       ↓     ↓
16진:   D    6       → D6₁₆
```

```
2진:   1010 1111 0000 0011
       ↓     ↓     ↓     ↓
16진:   A    F     0     3  → AF03₁₆
```

역방향도 동일하게 간단:

```
16진:   4    B    2    E
       ↓     ↓     ↓     ↓
2진:   0100 1011 0010 1110  → 0100101100101110₂
```

### 4.4 16진법 표기 관례

16진수임을 나타내는 방법들:

| 표기법 | 예시 | 사용처 |
|--------|------|--------|
| 0x 접두어 | 0xFF | C, C++, Java, Python |
| h 접미어 | FFh | 어셈블리 언어 |
| # 접두어 | #FF00FF | HTML/CSS 색상 코드 |
| $ 접두어 | $FF | 일부 어셈블러 |

> **핵심 통찰**: 16진법은 인간을 위한 것이지, 컴퓨터를 위한 것이 아니다. 컴퓨터는 항상 2진법으로 동작한다. 16진법은 사람이 2진 데이터를 읽고 쓰기 편하도록 만든 **인간 인터페이스**다. 4비트=16진 한 자리라는 깔끔한 대응이 이를 가능하게 한다.

## 5. 진법 변환 총정리

### 5.1 N진법 → 10진법

각 자리에 자릿값을 곱하여 합산:

```
1A3₁₆ = 1×16² + 10×16¹ + 3×16⁰
       = 256 + 160 + 3
       = 419₁₀
```

### 5.2 10진법 → N진법

반복 나누기 (나머지를 역순으로 읽기):

```
419 ÷ 16 = 26  나머지 3
 26 ÷ 16 = 1   나머지 10 (A)
  1 ÷ 16 = 0   나머지 1
```
→ 1A3₁₆ ✓

### 5.3 2진 ↔ 8진

3비트씩 끊어서 변환 (8 = 2³):

```
2진:  101 110 011
       ↓   ↓   ↓
8진:   5   6   3  → 563₈
```

### 5.4 변환 관계도

```
     10진법 (인간의 일상)
      ↕ (나누기/곱하기)
     2진법 (컴퓨터의 언어)
    ↙        ↘
  8진법      16진법
 (3비트)    (4비트)
 (역사적)   (현대 표준)
```

## 요약

- 어떤 진법에서든 **"10"은 기수 자체**를 나타낸다 — 한 자리가 꽉 차서 올림이 발생하는 수
- **2진법**: 0과 1만 사용, 자릿값은 2의 거듭제곱. 물리적 장치의 두 상태에 대응하여 컴퓨터에 최적
- **8진법**: 0-7을 사용, 2진법의 3비트에 대응. 역사적으로 사용됨
- **16진법**: 0-F를 사용, 2진법의 4비트에 대응. 현대 프로그래밍의 표준 표기법
- 진법 간 변환은 **위치 기수법의 일반 원리**에 의해 체계적으로 수행된다
- 2진 ↔ 8진은 3비트씩, 2진 ↔ 16진은 4비트씩 끊어서 기계적으로 변환 가능
