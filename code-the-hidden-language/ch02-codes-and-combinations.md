## 핵심 질문

> 제한된 수의 신호로 얼마나 많은 정보를 표현할 수 있는가? 코드의 "용량"은 어떻게 결정되는가?

## 1. 모스 부호의 수학적 분석

Chapter 1에서 모스 부호가 점(dot)과 선(dash)으로 알파벳을 표현하는 것을 보았다. 이제 한 걸음 뒤로 물러서서 수학적으로 생각해보자.

### 1.1 길이별 가능한 조합

점과 선, 두 가지 신호를 사용하여 특정 길이의 코드를 만들면 몇 가지 조합이 가능한가?

**길이 1 (신호 1개)**:

| 코드 | 문자 |
|------|------|
| `·` | E |
| `—` | T |

→ **2가지** 조합

**길이 2 (신호 2개)**:

| 코드 | 문자 |
|------|------|
| `· ·` | I |
| `· —` | A |
| `— ·` | N |
| `— —` | M |

→ **4가지** 조합

**길이 3 (신호 3개)**:

| 코드 | 문자 | 코드 | 문자 |
|------|------|------|------|
| `· · ·` | S | `— · ·` | D |
| `· · —` | U | `— · —` | K |
| `· — ·` | R | `— — ·` | G |
| `· — —` | W | `— — —` | O |

→ **8가지** 조합

**길이 4 (신호 4개)**:

| 코드 | 문자 | 코드 | 문자 |
|------|------|------|------|
| `· · · ·` | H | `— · · ·` | B |
| `· · · —` | V | `— · · —` | X |
| `· · — ·` | F | `— · — ·` | C |
| `· · — —` | (없음) | `— · — —` | Y |
| `· — · ·` | L | `— — · ·` | Z |
| `· — · —` | (없음) | `— — · —` | Q |
| `· — — ·` | P | `— — — ·` | (없음) |
| `· — — —` | J | `— — — —` | (없음) |

→ **16가지** 조합 (이 중 12개만 알파벳에 할당)

### 1.2 패턴의 발견

여기서 명확한 패턴이 보인다:

| 신호 수 | 조합 수 | 패턴 |
|---------|---------|------|
| 1 | 2 | 2¹ |
| 2 | 4 | 2² |
| 3 | 8 | 2³ |
| 4 | 16 | 2⁴ |
| n | ? | 2ⁿ |

> **핵심 통찰**: **n개의 이진 신호로 만들 수 있는 조합의 수는 2ⁿ이다.** 이것은 컴퓨터 과학 전체를 관통하는 가장 기본적인 수학 공식이다. 8비트 = 2⁸ = 256가지, 16비트 = 2¹⁶ = 65,536가지, 32비트 = 2³² = 약 43억 가지.

모스 부호에서 1~4개 신호로 만들 수 있는 조합의 총 수는:

```
2 + 4 + 8 + 16 = 30가지
```

영어 알파벳 26개를 표현하기에 충분하다. 실제로 Morse는 26개 글자를 이 30개 조합 중에서 골라 배정했고, 4개가 남았다.

## 2. 트리 다이어그램으로 보는 코드

모스 부호의 모든 조합을 **이진 트리(binary tree)**로 시각화할 수 있다. 왼쪽은 점(·), 오른쪽은 선(—)이다:

```
                    (시작)
                   /      \
                  ·        —
                 / \      / \
                E   T
               /\  /\   /\  /\
              ·  — ·  — ·  — ·  —
              I  A N  M S  U D  K  ...
             ...        ...
```

더 상세하게 전개하면:

```
시작
├── · (E)
│   ├── · · (I)
│   │   ├── · · · (S)
│   │   │   ├── · · · · (H)
│   │   │   └── · · · — (V)
│   │   └── · · — (U)
│   │       ├── · · — · (F)
│   │       └── · · — —
│   └── · — (A)
│       ├── · — · (R)
│       │   ├── · — · · (L)
│       │   └── · — · —
│       └── · — — (W)
│           ├── · — — · (P)
│           └── · — — — (J)
└── — (T)
    ├── — · (N)
    │   ├── — · · (D)
    │   │   ├── — · · · (B)
    │   │   └── — · · — (X)
    │   └── — · — (K)
    │       ├── — · — · (C)
    │       └── — · — — (Y)
    └── — — (M)
        ├── — — · (G)
        │   ├── — — · · (Z)
        │   └── — — · — (Q)
        └── — — — (O)
            ├── — — — ·
            └── — — — —
```

이 트리는 단순한 시각화가 아니다. **디코딩 알고리즘**이기도 하다. 수신자는 신호를 하나씩 받으면서 트리를 따라 내려가고, 글자 간격을 만나면 현재 위치의 문자를 읽으면 된다.

> **Petzold의 비유**: 이 트리는 마치 "네/아니오"로만 답하는 스무고개와 같다. 각 점/선은 "왼쪽/오른쪽" 중 하나를 선택하는 것이고, 충분한 횟수의 선택을 거치면 정확히 하나의 문자에 도달한다.

## 3. 고정 길이 코드 vs 가변 길이 코드

모스 부호는 **가변 길이 코드(variable-length code)**다. E는 신호 1개, H는 4개. 이것의 장단점을 정리하면:

### 가변 길이 코드

| 장점 | 단점 |
|------|------|
| 자주 쓰는 기호에 짧은 코드 → 평균 전송 효율 높음 | 글자 간 구분자(간격)가 필수 |
| 전체 메시지 길이가 짧아짐 | 디코딩이 복잡 — 타이밍에 의존 |
| 대역폭 절약 | 한 비트 오류가 이후 전체를 틀리게 할 수 있음 |

만약 모든 글자를 **고정 길이**로 표현한다면 어떨까? 26개 알파벳을 표현하려면:

- 4개 신호: 2⁴ = 16가지 → **부족** (26 > 16)
- 5개 신호: 2⁵ = 32가지 → **충분** (26 ≤ 32)

고정 길이 5비트 코드를 쓰면:

### 고정 길이 코드

| 장점 | 단점 |
|------|------|
| 구분자가 필요 없음 — 5개씩 끊으면 됨 | 자주 쓰는 글자도 5개 신호 필요 |
| 디코딩이 단순 | 평균 메시지 길이가 더 길어짐 |
| 오류 전파가 제한적 | 대역폭 낭비 |

> **핵심 통찰**: 가변 길이 코드는 효율적이지만 복잡하고, 고정 길이 코드는 비효율적이지만 단순하다. 컴퓨터 내부에서는 대부분 **고정 길이 코드(8비트, 16비트, 32비트, 64비트)**를 사용한다. 기계에게는 단순함이 속도이기 때문이다. 가변 길이 코드는 데이터 압축(ZIP, MP3, JPEG)이나 문자 인코딩(UTF-8) 같은 특수 목적에 사용된다.

## 4. 2의 거듭제곱 — 컴퓨터 과학의 마법 숫자

2ⁿ 공식은 이 책의 나머지 전체에서 반복적으로 등장한다. 주요 값을 기억해두자:

| n (비트 수) | 2ⁿ (조합 수) | 용도 |
|------------|-------------|------|
| 1 | 2 | 참/거짓, 예/아니오 |
| 2 | 4 | |
| 3 | 8 | |
| 4 | 16 | 16진수 한 자리(니블) |
| 5 | 32 | 알파벳 표현 가능 |
| 7 | 128 | ASCII 문자 집합 |
| 8 | 256 | 1바이트, 확장 ASCII |
| 10 | 1,024 | ≈ 1K (킬로) |
| 16 | 65,536 | 초기 Unicode |
| 20 | 1,048,576 | ≈ 1M (메가) |
| 24 | 16,777,216 | RGB 색상 (트루컬러) |
| 30 | 1,073,741,824 | ≈ 1G (기가) |
| 32 | 4,294,967,296 | 32비트 주소, IPv4 |
| 64 | 18,446,744,073,709,551,616 | 64비트 주소 |

왜 컴퓨터의 메모리가 항상 1024, 2048, 4096처럼 "이상한" 숫자인지 궁금했다면, 이제 그 답을 알 수 있다 — 모두 2의 거듭제곱이기 때문이다.

## 5. 조합과 순열

Petzold는 코드의 수학적 기초를 설명하면서 **조합론(combinatorics)**의 기본 개념을 소개한다.

### 5.1 순서가 중요한 경우 (순열)

신호의 **순서가 다르면 다른 코드**다:

```
· — (A) ≠ — · (N)
```

이것은 **순열(permutation)**의 특성이다. 모스 부호에서 `· —`와 `— ·`는 완전히 다른 글자를 나타낸다.

### 5.2 독립적인 선택

각 위치에서의 선택(점 또는 선)은 다른 위치와 **독립적**이다. 첫 번째 신호가 점이든 선이든, 두 번째 신호는 여전히 점/선 중 하나를 자유롭게 선택할 수 있다.

독립적인 선택의 경우, 전체 가능한 조합 수는 **각 선택지의 수를 곱한 것**이다:

```
2 × 2 = 4        (2자리)
2 × 2 × 2 = 8    (3자리)
2 × 2 × 2 × 2 = 16  (4자리)
```

이것은 이진 신호에만 국한되지 않는다. 만약 손전등으로 빨강/초록/파랑 세 가지 색을 낼 수 있다면:

```
3 × 3 = 9        (2자리)
3 × 3 × 3 = 27   (3자리)
```

일반화하면: **k가지 선택지, n자리 → kⁿ 가지 조합**

## 6. 정보와 선택의 관계

이 챕터의 가장 깊은 통찰은 **정보(information)**와 **선택(choice)**의 관계다.

하나의 이진 선택(점 또는 선, ON 또는 OFF)은 정보의 가장 작은 단위다. 이 단위에는 아직 이름이 없지만, Chapter 11에서 **비트(bit)**라는 이름을 얻게 된다.

핵심 아이디어를 정리하면:

```
1번의 이진 선택 → 2가지 중 하나를 구분
2번의 이진 선택 → 4가지 중 하나를 구분
3번의 이진 선택 → 8가지 중 하나를 구분
n번의 이진 선택 → 2ⁿ가지 중 하나를 구분
```

거꾸로 말하면:

```
2가지를 구분하려면 → 1번의 선택 필요
4가지를 구분하려면 → 2번의 선택 필요
8가지를 구분하려면 → 3번의 선택 필요
N가지를 구분하려면 → log₂(N)번의 선택 필요
```

> **핵심 통찰**: 정보를 표현하는 데 필요한 이진 신호의 수는 표현해야 할 가능한 값의 수에 대한 **로그(밑 2)**이다. 이것이 Claude Shannon이 1948년에 정보 이론(Information Theory)에서 수학적으로 정립한 개념이다. "정보"란 결국 "불확실성을 줄이는 것"이며, 하나의 이진 선택은 불확실성을 정확히 절반으로 줄인다.

## 요약

- **2ⁿ 공식**: n개의 이진 신호는 2ⁿ가지 서로 다른 조합을 만든다 — 컴퓨터 과학의 가장 기본적인 공식
- **가변 길이 vs 고정 길이**: 가변 길이는 효율적이나 복잡하고, 고정 길이는 단순하여 기계에 적합하다
- **이진 트리**: 모스 부호의 모든 조합을 트리로 표현할 수 있으며, 이 구조가 곧 디코딩 알고리즘이 된다
- **조합론의 곱셈 원리**: 독립적인 k가지 선택을 n번 하면 kⁿ가지 조합이 가능하다
- **정보와 로그**: N가지를 구분하는 데 필요한 이진 선택 횟수는 log₂(N)이다
