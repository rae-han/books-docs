# Chapter 7: Telegraphs and Relays (전신과 릴레이)

## 핵심 질문

> 전기 신호가 먼 거리를 이동하면서 약해지는 문제를 어떻게 해결할 수 있으며, 릴레이라는 장치가 왜 컴퓨터의 역사에서 그토록 중요한가?

---

## 1. 전신의 탄생

1830~40년대, **Samuel Morse**와 그의 조수 **Alfred Vail**은 전기를 이용한 장거리 통신 시스템을 개발했다. 이것이 **전신(telegraph)**이다.

전신의 기본 아이디어는 간단하다:
1. 긴 전선으로 두 지점을 연결
2. 한쪽에서 키(key)를 눌러 전류를 보냄
3. 다른 쪽에서 전자석이 금속 막대를 끌어당겨 딸깍 소리를 냄
4. 짧은 누름 = 점(dot), 긴 누름 = 선(dash)
5. 모스 부호로 메시지 전달

### 1.1 전신의 구조

```
[송신국]                                        [수신국]

                    전선 (수 km ~ 수백 km)
  키(Key) ───────────────────────────────── 사운더(Sounder)
    │                                           │
   ─┤├─ 배터리                                    │
    │                                           │
    ┤                                           ┤
   ═╧═ 접지                                접지 ═╧═
```

- **키(key/telegraph key)**: 스프링으로 열려 있는 스위치. 누르면 닫힘.
- **사운더(sounder)**: 전자석 + 금속 막대. 전류가 흐르면 딸깍 소리

### 1.2 전자석(Electromagnet)

전신의 핵심 부품은 **전자석**이다:

```
전선을 코일로 감기      전류 흐름      자기장 발생

    ┌─┐                ┌─┐          N
    │ │  ← 철심        │●│ ← 전류    │   금속 조각을
    │ │                │●│           │   끌어당김
    └─┘                └─┘          S
```

- 전선을 철심(iron core)에 감으면 코일이 됨
- 코일에 전류가 흐르면 자기장이 발생 → 전자석
- 전류를 끊으면 자기장도 사라짐 → 제어 가능한 자석

> **Petzold의 비유**: 전자석은 "전기로 제어할 수 있는 자석"이다. 일반 자석은 항상 켜져 있지만, 전자석은 전류를 켜고 끌 수 있다. 이것이 바로 전신을 가능하게 한 핵심 기술이다.

## 2. 장거리 통신의 문제

1844년, Morse는 워싱턴 D.C.에서 볼티모어까지 약 60km의 전신선을 설치하여 유명한 첫 메시지 "What hath God wrought"를 보냈다.

하지만 전선이 더 길어지면 심각한 문제가 발생한다:

| 거리 | 전선 저항 | 전류 | 사운더 동작 |
|------|-----------|------|-------------|
| 단거리 (~10km) | 낮음 | 충분 | 정상 |
| 중거리 (~50km) | 중간 | 약해짐 | 약하게 동작 |
| 장거리 (~200km) | 높음 | 매우 약함 | 동작 불가 |

Chapter 5에서 예고한 문제 그대로다. 전선의 저항이 거리에 비례하여 증가하므로, 충분한 전류가 도달하지 못한다.

해결 시도:
- 더 굵은 전선? → 비용 폭증
- 더 높은 전압? → 위험하고 절연 문제
- 중간에 사람을 배치? → 사람이 수신한 내용을 다시 송신 (실제로 사용됨!)

세 번째 방법이 실제로 쓰였지만, 사람이 항상 대기해야 하고 오류가 누적되는 문제가 있었다. 이 문제를 완벽하게 해결한 것이 **릴레이(relay)**다.

---

## 3. 릴레이 — 핵심 발명

### 3.1 릴레이의 구조

**릴레이(relay)**는 전자석을 사용하여 **전기 신호로 스위치를 제어하는 장치**다:

```
[입력 회로]                    [출력 회로]

  ┌─── 입력 전류 ───┐          ┌── 출력 배터리 ──┐
  │                 │          │                 │
  │          ┌──────┤          │    스위치(접점)   │
  │          │ 코일 │←─ 전자석   │   ↑ 전자석이     │
  │          │      │          │   │ 끌어당김      │
  │          └──────┤          │                 │
  │                 │          │    출력 장치     │
  └─────────────────┘          └─────────────────┘
```

동작 과정:
1. 입력 회로에 전류가 흐르면
2. 코일이 전자석이 되어
3. 금속 접점(armature)을 끌어당겨
4. 출력 회로의 스위치가 닫힘
5. 출력 회로에 (새로운 배터리의) 전류가 흐름

### 3.2 릴레이의 핵심 특성

| 특성 | 설명 | 의미 |
|------|------|------|
| **전기적 격리** | 입력 회로와 출력 회로가 물리적으로 분리 | 약한 신호로 강한 신호 제어 가능 |
| **신호 재생** | 출력은 새 배터리에서 나옴 | 신호 강도가 완전히 복원됨 |
| **자동 동작** | 사람 없이 전기 신호만으로 동작 | 무인 중계 가능 |

> **핵심 통찰**: 릴레이의 본질은 **"약한 신호로 강한 신호를 제어하는 것"**이다. 입력에 사용되는 전류는 전자석을 동작시킬 정도만 되면 충분하고, 출력은 완전히 새로운 배터리에서 나오므로 신호가 100% 복원된다. 이것이 **증폭(amplification)**의 원리이며, 진공관과 트랜지스터도 같은 원리로 작동한다.

---

## 4. 릴레이로 장거리 전신 구현

릴레이를 중간에 배치하면 신호를 무한정 중계할 수 있다:

```
[송신국]          [중계국 1]         [중계국 2]         [수신국]

키 ──── 약한신호 ──→ 릴레이 ──── 강한신호 ──→ 릴레이 ──── 강한신호 ──→ 사운더
배터리1            배터리2            배터리3
```

각 구간에서:
1. 약해진 신호가 릴레이의 전자석을 동작시키고
2. 릴레이가 새 배터리의 강한 전류를 다음 구간에 보냄
3. 이 과정을 반복하면 어떤 거리든 통신 가능

이것은 현대 네트워크의 **리피터(repeater)**와 본질적으로 같은 개념이다.

---

## 5. 릴레이가 컴퓨터의 역사에서 중요한 이유

릴레이는 단순한 중계 장치를 넘어서 **혁명적인 가능성**을 열었다. 핵심은 이것이다:

> **릴레이 = 전기 신호로 제어되는 스위치**

Chapter 6에서 사람이 손으로 누르는 스위치로 AND, OR, NOT을 구현했다. 이제 릴레이를 사용하면 **전기 신호가 스위치를 누른다**. 즉:

```
사람이 누르는 스위치  →  릴레이 (전기 신호가 누르는 스위치)
                        ↓
수동 논리             →  자동 논리
                        ↓
계산하는 사람         →  계산하는 기계
```

> **핵심 통찰**: 릴레이의 진정한 중요성은 장거리 통신이 아니라, **전기 신호로 전기 신호를 제어**할 수 있게 된 것이다. 한 릴레이의 출력이 다른 릴레이의 입력이 될 수 있으므로, 릴레이를 연결하여 복잡한 논리 회로를 만들 수 있다. 이것이 Chapter 8의 "논리 게이트"로 직결된다.

---

## 6. 릴레이의 종류

### 6.1 상시 열림 (Normally Open, NO)

평소에는 접점이 열려 있고, 전류가 흐르면 닫힘:
- 전자석 비활성 → 스위치 열림 → 출력 0
- 전자석 활성 → 스위치 닫힘 → 출력 1

### 6.2 상시 닫힘 (Normally Closed, NC)

평소에는 접점이 닫혀 있고, 전류가 흐르면 열림:
- 전자석 비활성 → 스위치 닫힘 → 출력 1
- 전자석 활성 → 스위치 열림 → 출력 0

이것은 NOT 연산을 구현하는 데 사용된다!

| 릴레이 종류 | 전류 없을 때 | 전류 있을 때 | 논리 기능 |
|-------------|-------------|-------------|-----------|
| 상시 열림 (NO) | 열림 (0) | 닫힘 (1) | 버퍼 (입력=출력) |
| 상시 닫힘 (NC) | 닫힘 (1) | 열림 (0) | NOT (입력≠출력) |

## 7. 릴레이에서 컴퓨터로 — 역사적 발전

| 시기 | 장치 | 특징 |
|------|------|------|
| 1830s | 전자기 릴레이 | 전신 중계용, 느림 |
| 1930s | 릴레이 계산기 | 콘라트 추제(Z3), 하워드 에이큰(Harvard Mark I) |
| 1940s | 진공관 | 릴레이보다 1000배 빠르지만 크고 뜨거움 |
| 1950s | 트랜지스터 | 진공관보다 작고 안정적 |
| 1960s~ | 집적회로(IC) | 트랜지스터를 칩에 집적 |

릴레이 기반 컴퓨터는 실제로 존재했다. 독일의 Konrad Zuse가 1941년에 만든 Z3는 약 2,000개의 릴레이로 구성된 세계 최초의 프로그래머블 컴퓨터 중 하나였다.

> **Petzold의 비유**: 릴레이 → 진공관 → 트랜지스터 → IC의 발전은 마치 마차 → 자동차 → 비행기 → 로켓의 발전과 같다. 이동 수단의 **원리**(힘으로 물체를 이동)는 같지만, **구현**(말, 엔진, 제트, 추진제)이 혁명적으로 변했다. 마찬가지로, 논리 게이트의 **원리**는 릴레이 시대나 트랜지스터 시대나 동일하다.

## 회로 구성

전신 시스템의 전체 구성:

```
[송신측]
  키(스위치) ─────────→ 전선 (수 km) ─────────→ [수신측]
     │                                           전자석
    ─┤├─ 배터리                                    │
     │                                          접점(armature)
     ┤                                            │
    ═╧═                                         사운더/전구
                                                   │
                                                  ─┤├─ (수신측 배터리 — 릴레이 사용 시)
                                                   │
                                                   ┤
                                                  ═╧═
```

## 요약

- **전신(telegraph)**은 전선, 전자석, 모스 부호를 사용한 최초의 전기 통신 시스템이다
- **전자석(electromagnet)**은 전류로 제어할 수 있는 자석으로, 전신과 릴레이의 핵심 부품이다
- 장거리 통신에서 전선의 저항으로 인해 신호가 약해지는 문제가 발생한다
- **릴레이(relay)**는 약한 입력 신호로 강한 출력 신호를 만들어 이 문제를 해결한다
- 릴레이의 진정한 중요성은 **전기 신호로 전기 신호를 제어**할 수 있다는 것이며, 이것이 자동 논리 연산과 컴퓨터로 이어진다
