# Chapter 16: But What About Subtraction? (그런데 뺄셈은?)

## 핵심 질문

> 뺄셈을 위한 별도의 회로가 필요한가? 보수(complement)라는 아이디어로 어떻게 덧셈 회로만으로 뺄셈까지 처리할 수 있는가?

---

## 1. 뺄셈의 문제

Chapter 14에서 가산기를 만들었다. 뺄셈을 위해 별도의 감산기(subtractor)를 만들어야 할까? 그러면 회로가 두 배로 복잡해진다. 더 영리한 방법이 있다.

핵심 아이디어: **뺄셈을 덧셈으로 변환하자.**

---

## 2. 10진법의 보수 — 원리 이해

### 2.1 9의 보수 (Nines' Complement)

3자리 10진수에서 각 자릿수를 9에서 빼면 **9의 보수**가 된다:

```
원래 수:    4  2  3
9에서 빼기: 9-4=5  9-2=7  9-3=6
9의 보수:   5  7  6
```

특성: 원래 수 + 9의 보수 = 999 (모든 자리가 9)

### 2.2 10의 보수 (Tens' Complement)

10의 보수 = 9의 보수 + 1

```
423의 9의 보수  = 576
423의 10의 보수 = 576 + 1 = 577
```

특성: 원래 수 + 10의 보수 = 1000 (자릿수보다 1개 큰 1)

### 2.3 보수로 뺄셈하기

**A - B = A + (B의 10의 보수) — 최상위 올림을 버린다**

예: 725 - 423 = ?

```
방법 1 (일반 뺄셈):  725 - 423 = 302

방법 2 (보수 덧셈):
  423의 10의 보수 = 577
  725 + 577 = 1302
  최상위 1을 버리면: 302  ✓
```

> **Petzold의 비유**: 보수 뺄셈은 마치 시계에서 "3시간 전"을 계산하는 것과 같다. 12시 시계에서 현재 5시라면, 3시간 전 = 2시. 이것은 5 - 3 = 2이지만, 5 + 9 = 14에서 12를 빼면 2가 되는 것과 같다 (9 = 12 - 3 = 3의 보수).

## 3. 이진법의 보수

### 3.1 1의 보수 (Ones' Complement)

이진법에서 1의 보수는 **모든 비트를 반전**하는 것이다:

```
원래 수:   1 0 1 1 0 1
1의 보수:  0 1 0 0 1 0   (각 비트: 0→1, 1→0)
```

이것은 NOT 연산이다! **인버터(NOT 게이트)만 있으면 1의 보수를 구할 수 있다** — 복잡한 뺄셈 회로가 필요 없다!

### 3.2 2의 보수 (Two's Complement)

2의 보수 = 1의 보수 + 1

```
원래 수:   10110100  (180)
1의 보수:  01001011  (NOT)
2의 보수:  01001100  (+1)  = 76
```

검증: 180 + 76 = 256 = 2⁸ (8비트의 "자릿수 초과") ✓

### 3.3 2의 보수로 뺄셈하기

**A - B = A + (B의 2의 보수) = A + NOT(B) + 1**

예: 211 - 136 = ?

```
A = 211 = 11010011
B = 136 = 10001000

B의 1의 보수 = 01110111  (NOT)
B의 2의 보수 = 01111000  (+1)

  11010011  (211)
+ 01111000  (B의 2의 보수)
──────────
1 01001011  (75 + 최상위 올림)

최상위 올림을 버리면: 01001011 = 75  ✓
```

## 4. 부호 있는 정수의 표현

### 4.1 2의 보수 표현법

**최상위 비트(MSB)가 부호를 나타낸다:**
- MSB = 0 → 양수 (또는 0)
- MSB = 1 → 음수

8비트 2의 보수 표현:

| 이진수 | 부호 없는 값 | 2의 보수 값 |
|--------|-------------|-------------|
| 00000000 | 0 | **0** |
| 00000001 | 1 | **+1** |
| 01111111 | 127 | **+127** |
| 10000000 | 128 | **-128** |
| 10000001 | 129 | **-127** |
| 11111110 | 254 | **-2** |
| 11111111 | 255 | **-1** |

범위: **-128 ~ +127** (총 256가지 = 2⁸)

일반화: n비트 2의 보수 범위 = -2ⁿ⁻¹ ~ +2ⁿ⁻¹-1

| 비트 수 | 최솟값 | 최댓값 |
|---------|--------|--------|
| 8 | -128 | +127 |
| 16 | -32,768 | +32,767 |
| 32 | -2,147,483,648 | +2,147,483,647 |

### 4.2 양수 ↔ 음수 변환

어떤 수의 부호를 바꾸려면: **비트 반전 + 1** (2의 보수 취하기)

```
+5 = 00000101
비트 반전: 11111010
+1:        11111011 = -5

검증: -5의 부호를 다시 바꾸면?
-5 = 11111011
비트 반전: 00000100
+1:        00000101 = +5  ✓
```

> **핵심 통찰**: 2의 보수 표현의 가장 큰 장점은 **덧셈 회로 하나로 양수의 덧셈, 음수의 덧셈, 뺄셈을 모두 처리할 수 있다**는 것이다. 컴퓨터가 부호를 신경 쓸 필요 없이 그냥 비트를 더하면 올바른 결과가 나온다. 이것이 2의 보수가 사실상 모든 컴퓨터에서 사용되는 이유다.

## 5. 하드웨어로 구현하기

### 5.1 덧셈/뺄셈 통합 회로

가산기에 약간의 추가 논리만 있으면 뺄셈도 가능하다:

```
SUB 신호 (0=덧셈, 1=뺄셈)
    │
    ↓
B₇──[XOR]──┐
B₆──[XOR]──┤
B₅──[XOR]──┤
B₄──[XOR]──┤    8비트           결과
B₃──[XOR]──┼──→ 리플 캐리 ──→ S₇~S₀
B₂──[XOR]──┤    가산기
B₁──[XOR]──┤
B₀──[XOR]──┘
    ↑               ↑
    SUB             SUB → Carry-in
```

**SUB = 0 (덧셈)**: XOR이 B를 그대로 통과. Carry-in = 0. 결과: A + B
**SUB = 1 (뺄셈)**: XOR이 B를 반전(1의 보수). Carry-in = 1(+1). 결과: A + NOT(B) + 1 = A - B

게이트 추가량: **XOR 8개 + 선 1개**만으로 뺄셈 기능 완성!

## 6. 오버플로우 (Overflow)

### 6.1 부호 있는 덧셈의 오버플로우

8비트 2의 보수에서 범위는 -128 ~ +127:

```
  01111111  (+127)
+ 00000001  (+1)
──────────
  10000000  → 2의 보수로 해석하면 -128!
```

양수 + 양수 = 음수? 이것은 **오버플로우**다.

### 6.2 오버플로우 감지

| C₇ (마지막 올림 출력) | C₆ (직전 올림) | 오버플로우? |
|----------------------|---------------|------------|
| 0 | 0 | 아니오 |
| 0 | 1 | **예** |
| 1 | 0 | **예** |
| 1 | 1 | 아니오 |

규칙: **C₇ XOR C₆ = 1이면 오버플로우**

> **핵심 통찰**: 오버플로우는 "하드웨어의 한계로 인한 수학적 오류"다. 유한한 비트로 무한한 수를 표현할 수 없으므로, 프로그래머는 항상 데이터의 범위를 의식해야 한다. 이것이 int, long, short 같은 정수 타입이 존재하는 이유이며, 버그의 원인이 되기도 한다.

---

## 요약

- **보수(complement)** 기법으로 뺄셈을 덧셈으로 변환할 수 있다
- **1의 보수** = 모든 비트 반전 (NOT 연산). **2의 보수** = 1의 보수 + 1
- **A - B = A + NOT(B) + 1**: 가산기와 인버터만으로 뺄셈 구현
- **2의 보수 표현**: MSB가 부호 비트. n비트 범위: -2ⁿ⁻¹ ~ +2ⁿ⁻¹-1
- 가산기에 XOR 게이트와 SUB 제어 신호만 추가하면 **덧셈/뺄셈 통합 회로** 완성
- **오버플로우**: 결과가 표현 범위를 넘을 때 발생. C₇ XOR C₆로 감지
