# 1. ES6에서의 리스트와 이터러블/이터레이터 프로토콜

실무에서 리스트 순회는 매우 중요하다. 그리고 함수형 프로그래밍에서는 더욱 중요하다.

- ES6 이전의 순회문(*`for i++`*)은 어떻게 순회를 하는지 명령한다.
- ES6의 순회문(*`for of`*)은 어떻게 순회를 하는지 선언을 한다.
- 즉, ES5의 for 문은 인덱스(i)와 length에 의존해 배열, 문자열 등 인덱스가 있는 자료형에만 사용 가능하지만, ES6의 for..of문은 내부적으로 [Symbol.iterator]()를 호출해 이터러블 객체에 사용 가능하며 인덱스가 업

자바스크립트 개발자라면 ES6이라는 언어가 어떤 규약을 열어주고 어떻게 순회에 대해 추상화를 했고, 어떻게 사용도록 했는지에 대해 생각해 봐야 한다.
```typescript
const list = [1, 2, 3];
const str = 'abc';

for (var i = 0; i < list.length; i++) {
	console.log(list[i]); // 1 2 3
}
for (var i = 0; i < str.length; i++) {
	console.log(str[i]); // a b c
}

for (const a of list) {
	console.log(a); // 1 2 3
}
for (const a of str) {
	console.log(a); // a b c
}
```

# 1. 이터러블/이터레이터 프로토콜 with Array, Set, Map
for … of 는 특정 키(i)로 접근해서 하나씩 값을 순회하는 하는 것이 아니다. 그런 동작 원리가 숨겨져 있는 것 또한 아니다. set, map 같은 경우에 인덱스(i) 값으로 접근할 수 없다(undefined). 즉 키와 값으로 순회 하면서 동작하는 것이 아닌 다른 방법으로 동작한다.

## 1.1. Symbol.iterator
순회 가능한 객체(Array, Set, Map 등.)의 키로 사용될 수 있다. 그리고 그 안에는 어떤 함수가 들어 있다. 만약 이 함수를 지운다면 순회 가능한 객체는 순회할 수 없어진다(`TypeError:[순회 가능한 객체] is not iterable`).

## 1.2. 순회의 진화 for vs for…of
- **ES5의 for문**
	- 인덱스(**`i`**)와 **`length`**에 의존
	- 배열, 문자열 등 인덱스가 있는 자료형에만 사용 가능
- **ES6의 for...of문**
	- 내부적으로 **`[Symbol.iterator]()`**를 호출
	- 배열, Set, Map, 문자열 등 **이터러블 객체**에 사용 가능
	- 인덱스가 없는 Set, Map도 순회 가능

## 1.3. 이터러블, 이터레이터 프로토콜
- 이터러블: **`[Symbol.iterator]()`** 메서드를 가지고, 이 메서드가 **이터레이터**를 반환하는 객체
- 이터레이터: **`next()`** 메서드를 가지고, **`{ value, done }`** 객체를 반환
- 이터러블/이터레이터 프로토콜: 이터러블 객체는 for...of, 전개 연산자 등에서 순회할 수 있도록 규약을 제공
	```typescript
const arr = [1, 2, 3]
const iter = arr[Symbol.iterator]();

iter.next(); // { value:1, done: false }
	```
	for … of 는 이터레이터의 next() 함수에서 나온 value의 값을 담아 출력하다가, done이 true가 되면 빠져나오도록 돼 있다.
	```typescript
const arr = [1, 2, 3];
const iter = arr[Symbol.iterator]();

console.log(iter.next()); // { value: 1, done: false }
console.log(iter.next()); // { value: 2, done: false }
console.log(iter.next()); // { value: 3, done: false }
console.log(iter.next()); // { value: undefined, done: true }

for (const item of arr) {
  console.log(item); // 1, 2, 3
}

for (const item of iter) { // 이미 next() 함수로 순회를 끝내기 때문에 순회하지 않는다.
	console.log(item);
}
	```

## 1.4. 이터레이터도 이터러블
이터레이터 객체도 **`[Symbol.iterator]()`**를 구현하며, 이 메서드는 **자기 자신(this)**을 반환한다.
그래서 이터레이터 자체도 for...of에서 순회가 가능하다.
```typescript
const iter = arr[Symbol.iterator]();
console.log(iter[Symbol.iterator]() === iter); // true
```
이런 구조 덕분에 이터레이터 객체는 for…of 문, 전개 연산자와 같은 이터러블을 요구하는 곳에서 그대로 사용 가능하다.
for…of 문은 내부적으로 Symbol.iterator를 호출해서 이터레이터를 얻고, 그 이터레이터의 next() 메서드를 사용해 값을 순회한다.

## 1.5. 이터러블/이터레이터 직접 구현하기
```typescript
const iterable = {
	[Symbol.iterator]() {
    let i = 3;
    return {
      next() {
        return i === 0 ? { value: undefined, done: true } : { value: i--, done: false };
      },
      [Symbol.iterator]() { return this; }
    }
  }
}
```
이터레이터가 자기 자신을 반환하는 [Symbol.iterator] 메서드를 가지고 있을 때, well-formed iterator 또는 well-formed iterable이라고 부른다. 이터레이터 프로토콜을 준수하는 이터러블은 자신을 반복(iterate)하는 것이 가능합니다.
즉, 잘 만들어진 이터레이터 객체는 Symbol.iterator를 호출하면 자기 자신을 반환한다. 그래서 이터레이터를 for...of문에 넣거나, 다시 Symbol.iterator를 호출해도 새로운 이터레이터가 아니라 기존의 이터레이터(즉, 현재 상태를 가진 이터레이터)가 사용된다.
이 덕분에, 이터레이터의 순회가 중간에 멈췄더라도, 다시 순회할 때 이전까지 진행된 상태에서 계속 이어서(next) 순회할 수 있다.

## 1.6. 전개 연산자
전개 연산자 역시 이터레이터 프로토콜을 준수하는 이터러블을 받아서 이터레이터를 반환하는 것을 기대한다.
