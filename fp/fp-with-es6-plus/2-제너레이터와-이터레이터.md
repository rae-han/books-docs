# 2. 제너레이터와 이터레이터

## 2.1. 제너레이터/이터레이터란?
**이터레이터**
- **`next()`** 메서드로 순차적으로 값을 반환하는 객체이다
- **`{ value, done }`** 형태의 결과를 반환한다.

**제너레이터**
- **`function*`** 문법으로 선언, **`yield`**를 통해 값을 순차적으로 생성한다.
- 실행하면 이터레이터를 반환하는 **함수이다**.
- 제너레이터는 순회할 값을 문장으로 표현하는 것이라고도 할 수 있다.

## 2.2. 기본 제너레이터 예제
```typescript
let isSecond = false;
function* generator() {
  yield 1;
  if (isSecond) yield 2;
  yield 3;
  return 100; // 리턴 값이 있지만 순회 시에는 포함되지 않음
}

let iter = generator(); // 제너레이터를 실행한 결과는 이터레이터
console.log(iter[Symbol.iterator]() === iter); // true - 이터레이터는 자기 자신을 반환하는 Symbol.iterator 메서드를 가짐

console.log(iter.next()); // { value: 1, done: false }
console.log(iter.next()); // { value: 3, done: false } (isSecond가 false이므로 2는 생략됨)
console.log(iter.next()); // { value: 100, done: true } (return 값)
console.log(iter.next()); // { done: true } (value가 없음)

// for...of로 순회 시 return 값은 포함되지 않음
for (const a of generator()) console.log(a); // 1, 3
```
- 리턴 값은 순회할 때 포함되지 않는 다는 것에 주의 하자.
- 제너레이터를 실행한 결과는 이터레이터이고 이터레이터는 Symbole.iterator를 가지고 있는 이터러블이기 때문에 순회 가능하게 해준다.

## 2.3. 제너레이터로 조합성과 재사용성 올리는 과정

### 2.3.1. 기본 홀수 생성기
```typescript
function* odds(l) {
  for (let i = 0; i < l; i++) {
    if(i % 2) yield i;
  }
}
```
- 특징
	- 0부터 l-1까지 직접 반복한다.
	- 홀수 생성과 범위 제한이 결합돼 있다.
- 단점
	- 무한 수열 생성이 불가능하며, 재사용성이 낮고 확장성이 떨어진다.

### 2.3.2. 무한 수열 도입
```typescript
function* infinity(i = 0) {
  while (true) yield i++;
}

function* odds(l) {
  for (const i of infinity(1)) {
    if (i % 2) yield i;
    if (i >= l) return;
  }
}
```
- 개선점
	- 무한 수열 생성하는 부분을 분리 했다.
	- 조건문을 통해 부분 값을 매칭이 가능하다.
	- 확장성이 높다.
- 한계
	- 하지만 여전히 홀수 생성과 범위 제어가 결합돼 있다.
	- 다른 필터 조건 적용이 어렵다.

### 2.3.3. 모듈화
```typescript
function* infinity(i = 0) {
  while (true) yield i++;
}

function* limit(l, iter) {
  for (const a of iter) {
    yield a;
    if (a >= l) return;
  }
}

function* odds(l) {
  for (const a of limit(l, infinity(1))) {
    if (a % 2) yield a;
  }
}
```
- 계층적 분리
	- **`infinity`**: 순차적 숫자 생성 (기본 데이터 소스)
	- **`limit`**: 범위 제한 필터 (중간 처리기)
	- **`odds`**: 홀수 필터 (최종 처리기)
- 개선점
	- odds 함수는 순수하게 홀수를 필터링하는 역할만 하고 있고, 숫자 범위를 제한하는 역할은 없기 때문에 다른 필터 함수를 만드려면 해당 부분만 수정하면 된다.
	- limit 제너레이터는 다른 제너레이터와 조합 가능하다. 예를 들어 짝수 생성이나 피보나치수열을 생성하는 범위를 제한하는 함수에도 조합 가능하다.

## 2.4. 문장을 값으로 만드는 제너레이터
1. 프로그래밍에서 문장
	- 프로그래밍에서 문장(statement)은 어떤 동작을 수행하는 코드의 한 줄 또는 한 구문이다.
	```typescript
let x = 10;        // 변수 선언 및 할당 (문장)
console.log(x);    // 출력 (문장)
yield 1;           // 값을 내보내고 실행을 멈춤 (문장)
	```
2. 프로그래밍에서 값
	- 값(value)은 변수에 저장되거나 함수에서 반환될 수 있는 데이터 그 자체이다.
	```typescript
10                // 숫자 값
'hello'           // 문자열 값
true              // 불리언 값
[1, 2, 3]         // 배열 값
	```
3. 문장을 값으로 만든다 의 의미
	- 일반적으로 문장은 실행만 되고, 그 결과가 값으로 직접 다뤄지진 않습니다.
	- 제너레이터에서 `yield`를 사용하면, *실행되는 문장*이 *값*으로 외부에 전달됩니다.
	- 즉, 코드의 실행 흐름(문장)을 끊어서, 그 중간 결과(값)를 바깥에서 하나씩 받을 수 있게 만든다.
	```typescript
function* userFlow() {
  const name = yield '이름 입력:';  // 첫 번째 값
  const age = yield '나이 입력:';   // 두 번째 값
  yield `${name}님, ${age}세 환영!`; // 최종 값
}

const flow = userFlow();

// 값 추출 과정
console.log(flow.next().value); // "이름 입력:"
console.log(flow.next('김코딩').value); // "나이 입력:"
console.log(flow.next(25).value); // "김코딩님, 25세 환영!"
	```
