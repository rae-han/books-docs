# 4. 코드를 값으로 다루어 표현력 높이기

함수형 프로그래밍에서는 코드(함수)를 값처럼 다루기 때문에, 함수를 인자로 전달하거나 반환할 수 있고, 그 함수가 실제로 실행되는 시점도 원하는 대로 제어할 수 있다. 이로 인해 코드의 표현력이 높아지고, 복잡한 로직을 더 유연하고 명확하게 작성할 수 있다.

## 1. go
go 함수는 여러 개의 인자를 받아서, 첫 번째 인자를 시작값으로 두고 나머지 인자(함수)들을 차례로 적용해가며 값을 전달하는 함수이다.
즉 reduce를 사용해 구현하면 된다.
```typescript
const go = (...args) => {
  return reduce((a, f) => f(a), args);
};

go(
  0,
  (a) => a + 1,
  (a) => a + 10,
  (a) => a + 100,
  console.log
);
```

## 2. pipe
go 함수와 달리 함수를 리턴한다. go 함수는 인자와 함수를 받아 즉시 값을 평가하는데 사용한다.
pipe 함수는 인자와 함수를 받아 우선 함수를 합성하고, 합성된 함수를 리턴한다. 합성된 함수는 인자를 받아 즉시 값을 평가하는데 사용한다.
go 함수를 사용해 인자만 나중에 받게하면 된다.
```typescript
const pipe =
  (...fns) =>
  (arg) =>
    go(arg, ...fns);
```
근데 여기서 조금 아쉬운 건 go 함수는 필요하다면 처음에 인자를 2개 이상 받을수 있지만 위의 pipe 함수는 받을수 없다.
```typescript
const add(a, b) => a + b;

go(
  add(0, 1), // 인자를 두개 받음.
  (a) => a + 1,
  (a) => a + 10,
  (a) => a + 100,
  console.log
);
```
pipe 함수도 첫 함수의 경우 실행해 줄 때 인자를 2개 이상 전달할 수 있도록 구성돼 있으면 좋을 것 같다.
```typescript
const pipe =
  (fn, ...fns) =>
  (...args) =>
    go(fn(...args), ...fns);
```
참고로 함수가 하나만 들어와도 나머지 매개변수에는 undefined가 아닌 빈 배열이 들어 오므로 정상 동작한다.
```typescript
const f = pipe(
  (a, b) => a + b,
  (a) => a + 10,
  (a) => a + 100,
  console.log
);

f(0, 1); // 111
```

## 3. go, pipe 함수 활용
go, pipe 함수는 함수를 위에서 부터 아래로 도는 왼쪽에서 부터 오른쪽으로 평가하며 함수를 실행 가능하기 때문에 코드 표현을 바꿀 수 있다.
아래와 같은 코드가 있다고 하자.
```typescript
console.log(
	reduce(
		add,
		map(
			(p) => p.price,
			filter((p) => p.price < 20000, products),
		),
	),
);
```
go 함수를 사용해 아래와 같이 표현 가능하다.
```typescript
go(
	products,
	(products) => filter((p) => p.price < 20000, products),
	(products) => map((p) => p.price, products),
	(price) => reduce(add, price),
	(total_price) => console.log(total_price),
);
```
코드는 좀 더 많아지고 어떤 측면에선 간결성도 줄긴 했지만 읽긴 보단 편해졌다.

## 4. curry
curry 함수는 함수를 값으로 다루면서, 인자를 원하는 시점에 나눠 받아서, 인자가 모두 모였을 때 원래 함수를 실행하는 함수이다.
구현하면 아래와 같다.
```typescript
const curry =
	(fn) =>
	(arg, ...args) =>
		args.length ? fn(arg, ...args) : (...args) => fn(arg, ...args);
```
```typescript
const add = curry((a, b) => a + b);

console.log(add(1, 2)); // 3
console.log(add(1)(2)); // 3
```
위 curry 함수를 사용해 map, filter, reduce 함수를 감싼다면 아래와 같이 표현 가능하다.
```typescript
go(
	products,
	(products) => filter((p) => p.price < 20000)(products),
	(products) => map((p) => p.price)(products),
	(price) => reduce(add)(price),
	(total_price) => console.log(total_price),
);
```
그리고 go함수는 이전 단계의 결과값을 다음 함수에 전달한다. 또한 map, filter, reduce 같은 함수는 curry 함수를 이용해 부분 적용 할수 있으므로 다음과 같이 표현 가능하다.
```typescript
go(
	products,
	filter((p) => p.price < 20000),
	map((p) => p.price),
	reduce(add),
	console.log,
);
```

## 5. 함수 조합으로 함수 만들기
```typescript
const add = curry((a, b) => a + b);

const total_price = pipe(
	map((p) => p.price),
	reduce(add),
);

const base_total_price = (predicate) => pipe(filter(predicate), total_price);

go(
	products,
	base_total_price((p) => p.price < 20000),
	console.log,
);

go(
	products,
	base_total_price((p) => p.price >= 20000),
	console.log,
);
```
위 코드를 보면 조건에 해당하는 predicate 함수를 제외한 모든 로직은 재사용 되고 있다.
이런 식으로 함수형 프로그래밍에서는 고차 함수들을 함수의 조합으로 만들어가며, 잘게 나눠진 함수들을 계속 작게 만들어가며 중복을 제거하고 더 많은 곳에서 재사용되게 할수 있다.
