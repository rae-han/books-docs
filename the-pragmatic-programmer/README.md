# The Pragmatic Programmer (실용주의 프로그래머, 20주년 기념판)

> *The Pragmatic Programmer: Your Journey to Mastery* (20th Anniversary Edition, 2nd Edition) (David Thomas & Andrew Hunt, Addison-Wesley, 2019)

데이비드 토머스와 앤드류 헌트가 1999년 초판을 낸 지 20년 만에 전면 개정한 소프트웨어 개발의 고전. 특정 언어·프레임워크가 아니라 **문제와 해법에 접근하는 태도·방식·철학**을 다룬다 — 9개 장, 53개 Topic, 그리고 책 전체를 관통하는 **100개의 실용주의 Tip**으로 이루어져 있다.

## 실용주의 프로그래머란

> 실용주의 프로그래머는 직면한 문제 너머를 고민한다. 문제를 더 큰 맥락에 놓고 더 큰 그림을 보려고 노력한다.

실용주의 프로그래머는 **얼리 어답터**(새 기술에 호기심을 갖고 빠르게 익힘), **탐구적**(끊임없이 질문), **비판적**("늘 그렇게 해 왔으니까"에 도전), **현실주의자**(문제의 본질과 난도를 파악), **다방면에 능숙한 사람**의 특징을 지닌다. 그리고 모든 실용주의 프로그래머에게 공통되는, 팁의 형식으로 소개할 만큼 기본적인 두 가지 특징이 있다.

- **Tip 1**: 자신의 기예(craft)에 관심을 가져라.
- **Tip 2**: 자기 일에 대해 생각하라. (IBM의 표어 'THINK!'가 실용주의 프로그래머의 계명)

이 책의 진짜 주제는 **당신 자신**이다 — 책은 Ch1 "당신의 인생이다"(**Tip 3**)로 시작해 "결국 당신의 삶이다"(**Tip 100**)로 닫힌다.

## 핵심 개념 지도

- **좋은 설계 = ETC(바꾸기 더 쉽게)** — 모든 설계 원칙(DRY·직교성·결합도 줄이기·가역성)이 ETC의 특수한 경우
- **완벽한 소프트웨어는 없다** — 방어적으로(계약·단정문·일찍 멈추기) 코딩하고 **작은 단계**로 나아가라
- **변환 프로그래밍·액터·칠판** — 상태를 쌓지 말고 전달하고, 공유 상태를 피하라
- **테스트는 피드백** — 버그 찾기가 아니라 코드를 이끄는 첫 번째 사용자
- **요구 사항은 발견하는 과정** — 아무도 자신이 뭘 원하는지 정확히 모른다
- **애자일은 방식**(명사 아님) — 어디 있는지 알고, 작게 나아가고, 평가·수정하라
- 우리의 진짜 직함은 **문제 해결사** — 사용자를 기쁘게 하고, 작품에 서명하라

## 시그니처 요소와 표기 규칙 (노트 작성 스타일)

- **Topic 기반 구성**: `## Topic N: 한글 (English)` — 9개 장 안의 53개 Topic 단위
- **Tip(시그니처)**: 본문 등장 지점에 `> **Tip N**: 내용` 콜아웃 + 각 장 끝 `## 이 장의 Tip 목록`(불릿 리스트), 전체는 [tips.md](tips.md)
- **연습 문제**: 각 장 끝 `## 연습 문제 (해답 예시)`에 **전부 포함** + 부록 해답 요지를 **인라인** 연결(옵션 A). 코드 구현형은 접근법 요지로
- **도전 과제**: 해답 없는 열린 질문은 각 Topic 안에 `**도전 과제**` 불릿으로
- **우화·일화**(돌멩이 수프·삶은 개구리·깨진 창문·헬리콥터·고르디우스의 매듭 등): 레이블 없는 `>` 인용
- **핵심 통찰**: `> **핵심 통찰**:` 콜아웃(Notion 업로드 시 💡 콜아웃 블록)
- **Topic 여는 인용문**: `> 인용<br>— 이름`(브루스 리·비트겐슈타인·공자 등)
