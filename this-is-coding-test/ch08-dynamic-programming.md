# Chapter 08: 다이나믹 프로그래밍 (Dynamic Programming)

> 한 번 계산한 결과를 저장해 중복 계산을 없애는 기법. 큰 문제를 작은 문제로 나누고, 같은 부분 문제는 한 번만 푼다. "동적 계획법"이라고도 한다.

## 핵심 질문

- 다이나믹 프로그래밍을 적용할 수 있는 두 가지 조건은 무엇인가?
- 탑다운(메모이제이션)과 바텀업(DP 테이블)의 차이와 선택 기준은?
- 문제에서 **점화식**을 어떻게 세우는가?

## 1. DP란 — 두 가지 조건

DP는 아무 문제에나 쓸 수 없다. 다음 두 조건을 만족해야 한다.

| 조건 | 의미 |
|------|------|
| 최적 부분 구조(*Optimal Substructure*) | 큰 문제를 작은 문제로 나눌 수 있고, 작은 문제의 답이 큰 문제에서도 그대로 쓰인다 |
| 중복되는 부분 문제(*Overlapping Subproblems*) | 같은 작은 문제가 여러 번 반복해서 등장한다 |

> **핵심 통찰**: DP의 출발점은 "완전 탐색(재귀)이 **같은 계산을 반복**하고 있다"는 발견이다. 피보나치를 단순 재귀로 짜면 `f(30)`에 약 10억 번 연산이 필요하지만(O(2ⁿ)), 한 번 구한 값을 저장하면 O(N)으로 줄어든다.

## 2. 피보나치로 보는 DP

피보나치 점화식: `f(n) = f(n-1) + f(n-2)`, 단 `f(1) = f(2) = 1`. 점화식(*인접 항들 사이의 관계식*)이 곧 DP 코드의 뼈대가 된다.

### 탑다운 (메모이제이션) — 재귀 + 캐싱

```python
# Python (책 원본)
d = [0] * 100

def fibo(x):
    if x == 1 or x == 2:
        return 1
    if d[x] != 0:        # 이미 계산했으면 그대로 반환
        return d[x]
    d[x] = fibo(x - 1) + fibo(x - 2)
    return d[x]
```

```ts
// TypeScript
const d = new Array(100).fill(0);

function fibo(x: number): number {
  if (x === 1 || x === 2) {
    return 1;
  }
  if (d[x] !== 0) {
    // 이미 계산한 적 있으면 그대로 반환 (메모이제이션)
    return d[x];
  }
  d[x] = fibo(x - 1) + fibo(x - 2);
  return d[x];
}
```

### 바텀업 (DP 테이블) — 반복문

```ts
const n = 99;
const dp = new Array(n + 1).fill(0);
dp[1] = 1;
dp[2] = 1;
for (let i = 3; i <= n; i++) {
  dp[i] = dp[i - 1] + dp[i - 2];
}
// dp[n]이 답
```

| | 탑다운 (메모이제이션) | 바텀업 (DP 테이블) |
|------|---------------------|--------------------|
| 방식 | 재귀 (큰 문제 → 작은 문제) | 반복문 (작은 문제 → 큰 문제) |
| 저장소 | 캐시 | DP 테이블 |
| 권장도 | 직관적이나 **재귀 깊이 제한** 위험 | **전형적·안전** (권장) |

> **함정**: 탑다운(재귀)은 입력이 크면 호출 스택 한도를 넘는다(파이썬은 `sys.setrecursionlimit`로 완화, Node.js도 한계 존재). **가능하면 바텀업**으로 짜는 게 안전하다. 또 TS의 DP 테이블은 `new Array(n).fill(0)`처럼 **반드시 초기화**해야 한다(안 하면 빈 슬롯/`undefined`로 연산이 `NaN`이 된다).

## 3. DP 문제 접근법

1. 완전 탐색(재귀)으로 떠올려 본다 → 중복 부분 문제가 보이는가?
2. **점화식**을 세운다 (현재 항을 이전 항들로 표현).
3. 작은 값부터 채우는 **바텀업**으로 구현한다.

## 4. 실전 문제 1 — 1로 만들기

| 항목 | 값 |
|------|-----|
| 난이도 | ★★ 중하 |
| 풀이 시간 | 20분 |
| 시간 제한 | 1초 |

### 문제 요약

정수 X에 ①5로 나누기 ②3으로 나누기 ③2로 나누기 ④1 빼기 연산을 써서 1을 만드는 **최소 연산 횟수**를 구하라. (X ≤ 30,000)

예: X=26 → `3` (26→25→5→1)

### 점화식

> `d[i] = min(d[i-1], d[i/2], d[i/3], d[i/5]) + 1` (나누어떨어지는 경우에만 해당 항 적용)

### 코드

```ts
const x = 26;
const d = new Array(30001).fill(0);

for (let i = 2; i <= x; i++) {
  // 1을 빼는 경우
  d[i] = d[i - 1] + 1;
  // 2로 나누어떨어지는 경우
  if (i % 2 === 0) {
    d[i] = Math.min(d[i], d[i / 2] + 1);
  }
  // 3으로 나누어떨어지는 경우
  if (i % 3 === 0) {
    d[i] = Math.min(d[i], d[i / 3] + 1);
  }
  // 5로 나누어떨어지는 경우
  if (i % 5 === 0) {
    d[i] = Math.min(d[i], d[i / 5] + 1);
  }
}
console.log(d[x]); // 3
```

**시간복잡도**: O(X).

## 5. 실전 문제 2 — 개미 전사

| 항목 | 값 |
|------|-----|
| 난이도 | ★★ 중하 |
| 풀이 시간 | 30분 |
| 시간 제한 | 1초 |

### 문제 요약

일직선 식량창고들에서 **인접하지 않게** 골라 식량 합을 최대로 만들어라. (N ≤ 100, 각 ≤ 1,000)

예: `[1, 3, 1, 5]` → `8` (2번째 3 + 4번째 5)

### 점화식

i번째 창고를 털지(이전 d[i-2] + 현재) 안 털지(d[i-1]) 중 큰 쪽을 택한다.

> `d[i] = max(d[i-1], d[i-2] + array[i])`

### 코드

```ts
const array = [1, 3, 1, 5];
const n = array.length;
const d = new Array(n).fill(0);

d[0] = array[0];
d[1] = Math.max(array[0], array[1]);
for (let i = 2; i < n; i++) {
  // i번째를 안 털기 vs 털기(i-2까지 + 현재)
  d[i] = Math.max(d[i - 1], d[i - 2] + array[i]);
}
console.log(d[n - 1]); // 8
```

**시간복잡도**: O(N).

> **핵심 통찰**: "인접한 것을 동시에 못 고른다"류의 선택 최적화는 **"현재를 포함 vs 미포함"** 두 경우의 max로 점화식이 선다. 이 패턴은 매우 자주 등장한다.

## 6. 실전 문제 3 — 바닥 공사

| 항목 | 값 |
|------|-----|
| 난이도 | ★★ 중하 |
| 풀이 시간 | 20분 |
| 시간 제한 | 1초 |

### 문제 요약

2×N 바닥을 1×2, 2×1, 2×2 덮개로 채우는 **경우의 수**를 796,796으로 나눈 나머지로 구하라. (N ≤ 1,000)

예: N=3 → `5`

### 점화식

왼쪽부터 채울 때, i-1까지 채워졌으면 2×1 한 개(1가지), i-2까지 채워졌으면 1×2 두 개 또는 2×2 한 개(2가지).

> `d[i] = d[i-1] + 2 × d[i-2]`

### 코드

```ts
const n = 3;
const MOD = 796796;
const d = new Array(1001).fill(0);

d[1] = 1;
d[2] = 3;
for (let i = 3; i <= n; i++) {
  d[i] = (d[i - 1] + 2 * d[i - 2]) % MOD; // 큰 결과를 나머지로 관리
}
console.log(d[n]); // 5
```

> **함정**: DP 문제는 "결과를 어떤 수로 나눈 나머지를 출력"하라는 경우가 많다. 값이 폭발하기 때문이다. **매 계산마다 `% MOD`** 를 취해야 오버플로(JS는 2⁵³ 초과 시 정밀도 손실)를 막는다.

## 7. 실전 문제 4 — 효율적인 화폐 구성

| 항목 | 값 |
|------|-----|
| 난이도 | ★★★ 중 |
| 풀이 시간 | 30분 |
| 시간 제한 | 1초 |

### 문제 요약

N가지 화폐(각 무한 사용)로 합 M원을 만드는 **최소 화폐 개수**를 구하라. 불가능하면 -1. (N ≤ 100, M ≤ 10,000)

예: 화폐 `[2,3]`, M=15 → `5` (3원 5개) / 화폐 `[3]`, M=4 → `-1`

### 점화식

금액 j를 만드는 최소 개수는, (j - 화폐단위)를 만들 수 있다면 거기에 1을 더한 값.

> `d[j] = min(d[j], d[j - coin] + 1)`

거스름돈(그리디)과 닮았지만 **화폐 단위가 배수 관계가 아니라** 그리디로는 안 되고 DP가 필요하다.

### 코드

```ts
const m = 15;
const coins = [2, 3];
const INF = 10001; // 만들 수 없음을 의미하는 큰 값
const d = new Array(m + 1).fill(INF);

d[0] = 0; // 0원은 0개로 만든다
for (const coin of coins) {
  for (let j = coin; j <= m; j++) {
    // (j - coin)을 만들 수 있는 경우에만 갱신
    if (d[j - coin] !== INF) {
      d[j] = Math.min(d[j], d[j - coin] + 1);
    }
  }
}
console.log(d[m] === INF ? -1 : d[m]); // 5
```

**시간복잡도**: O(N × M).

> **핵심 통찰**: "최소 개수로 목표를 만드는" 동전/배낭류는 **금액(또는 무게)을 인덱스로 둔 1차원 DP 테이블**에 `d[j-coin]+1`을 누적하는 패턴이다. 화폐 루프가 바깥, 금액 루프가 안쪽이라는 순서가 "각 화폐를 무한히 쓸 수 있음"을 보장한다.

## 자주 하는 실수

| 실수 | 원인 | 해결 |
|------|------|------|
| DP 테이블 미초기화 | 빈 슬롯/`undefined`로 `NaN` | `new Array(n).fill(0)` |
| 큰 입력에 탑다운 재귀 | 호출 스택 초과 | 바텀업 반복문으로 |
| 나머지(`% MOD`) 누락 | 값 폭발, 정밀도 손실 | 매 계산마다 `% MOD` |
| `//` 를 `/` 로만 옮김 | 나누어떨어지지 않으면 실수 | 나누어떨어질 때만 적용 또는 `Math.floor` |
| 점화식 없이 코딩 시작 | 구현 방향 상실 | **점화식 먼저** 종이에 적기 |

## 패턴 체크리스트

- [ ] 완전 탐색이 **같은 부분 문제를 중복** 계산하는가? → DP 후보
- [ ] **점화식**을 `d[i] = ... (이전 항들)` 형태로 세웠는가?
- [ ] 작은 값부터 채우는 **바텀업**으로 구현했는가?
- [ ] DP 테이블을 `fill()`로 초기화했는가?
- [ ] "나머지 출력" 조건이면 매 단계 `% MOD`를 적용했는가?

## 요약

- DP = **최적 부분 구조 + 중복 부분 문제**일 때, 한 번 푼 답을 저장해 O(2ⁿ)을 O(N)으로.
- **탑다운**(재귀+메모이제이션)은 직관적, **바텀업**(반복문+DP 테이블)은 안전 — 큰 입력은 바텀업.
- 모든 문제는 **점화식**에서 출발: 1로 만들기 `min(...)+1`, 개미 전사 `max(d[i-1], d[i-2]+a)`, 바닥 공사 `d[i-1]+2·d[i-2]`, 화폐 구성 `min(d[j], d[j-coin]+1)`.
- TS 주의: 테이블 `fill()` 초기화, `% MOD`로 오버플로 방지, 재귀 깊이 한계.
- 재귀-점화식 사고(Ch 05)가 여기서 꽃핀다. 다음 Ch 09 플로이드 워셜은 **2차원 DP**로 이어진다.
