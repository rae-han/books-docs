# Chapter 11: 그리디 기출문제

> Ch 03에서 배운 그리디 유형의 기출문제 모음(Q01~Q06). "매 단계 가장 좋아 보이는 선택"으로 최적해에 도달하는지, **정당성**을 함께 고민한다.

## 핵심 질문

- 정렬한 뒤 한 방향으로 훑으면 그리디가 보이는가?
- "최대 그룹 수", "못 만드는 최소 금액"처럼 **경계를 키워 가며** 판단하는 패턴인가?
- 단순해 보이는 선택이 정말 전역 최적인지 반례로 검증했는가?

---

## Q01 모험가 길드

| 항목 | 값 |
|------|-----|
| 난이도 | ★★ 중하 |
| 시간 제한 | 1초 |
| 기출 | 핵심 유형 |

### 문제 요약

공포도가 X인 모험가는 **X명 이상**인 그룹에 속해야 한다. 만들 수 있는 **최대 그룹 수**를 구하라. (모든 모험가를 포함할 필요는 없음)

예: 공포도 `[2,3,1,2,2]` → `2`

### 핵심 아이디어

공포도를 **오름차순 정렬**한 뒤 작은 것부터 그룹에 넣는다. 현재 그룹 인원이 그 그룹 최대 공포도(=현재 보고 있는 값) 이상이 되면 그룹을 완성한다. 공포도가 낮은 사람부터 묶으면 더 많은 그룹이 나온다.

```ts
const data = [2, 3, 1, 2, 2];
data.sort((a, b) => a - b);

let result = 0; // 그룹 수
let count = 0; // 현재 그룹에 포함된 인원
for (const x of data) {
  count++;
  // 현재 인원이 현재 공포도 이상이면 그룹 결성
  if (count >= x) {
    result++;
    count = 0;
  }
}
console.log(result); // 2
```

**시간복잡도**: O(N log N).

## Q02 곱하기 혹은 더하기

| 항목 | 값 |
|------|-----|
| 난이도 | ★★ 중하 |
| 시간 제한 | 1초 |
| 기출 | Facebook 인터뷰 |

### 문제 요약

숫자 문자열의 각 자리 사이에 `×` 또는 `+`를 넣어(왼쪽부터 순서대로 계산) 만들 수 있는 **최댓값**을 구하라.

예: `"02984"` → `576`, `"567"` → `210`

### 핵심 아이디어

두 수 중 **하나라도 0 또는 1이면 더하기**, 그 외에는 곱하기가 항상 크거나 같다. 누적값과 다음 수를 비교하며 진행한다.

```ts
const s = "02984";
let result = Number(s[0]);
for (let i = 1; i < s.length; i++) {
  const num = Number(s[i]);
  // 둘 중 하나라도 0 또는 1이면 더하기, 아니면 곱하기
  if (num <= 1 || result <= 1) {
    result += num;
  } else {
    result *= num;
  }
}
console.log(result); // 576
```

## Q03 문자열 뒤집기

| 항목 | 값 |
|------|-----|
| 난이도 | ★ 하 |
| 시간 제한 | 2초 |
| 기출 | 백준 1439 |

### 문제 요약

0과 1로 된 문자열을, 연속 구간을 통째로 뒤집는 연산으로 **모두 같게** 만드는 최소 횟수.

예: `"0001100"` → `1`

### 핵심 아이디어

연속된 **0 묶음의 개수**와 **1 묶음의 개수** 중 작은 값이 답이다(둘 중 적은 쪽을 모두 뒤집으면 되므로).

```ts
const s = "0001100";
let count0 = 0; // 0 묶음 수
let count1 = 0; // 1 묶음 수
// 첫 문자
if (s[0] === "1") {
  count1++;
} else {
  count0++;
}
// 값이 바뀌는 지점마다 새 묶음 시작
for (let i = 1; i < s.length; i++) {
  if (s[i] !== s[i - 1]) {
    if (s[i] === "1") {
      count1++;
    } else {
      count0++;
    }
  }
}
console.log(Math.min(count0, count1)); // 1
```

## Q04 만들 수 없는 금액

| 항목 | 값 |
|------|-----|
| 난이도 | ★★ 중하 |
| 시간 제한 | 1초 |
| 기출 | K 대회 |

### 문제 요약

N개의 동전으로 **만들 수 없는 양의 정수 금액 중 최솟값**을 구하라.

예: `[3,2,1,1,9]` → `8`, `[3,5,7]` → `1`

### 핵심 아이디어

동전을 **오름차순 정렬**하고 `target = 1`부터 시작한다. 현재 동전이 `target` 이하면 `target += coin`(1 ~ target까지 모두 만들 수 있음이 유지됨). 동전이 `target`보다 크면, 그 `target`이 만들 수 없는 최솟값이다.

```ts
const coins = [3, 2, 1, 1, 9];
coins.sort((a, b) => a - b);

let target = 1;
for (const coin of coins) {
  // coin이 target보다 크면 target을 만들 수 없음
  if (coin > target) {
    break;
  }
  target += coin;
}
console.log(target); // 8
```

> **핵심 통찰**: "1 ~ target까지 모두 만들 수 있다"는 불변식을 유지하는 게 정당성의 핵심이다. 정렬된 상태에서 `coin <= target`이면 범위가 `target + coin`까지 연속 확장된다.

## Q05 볼링공 고르기

| 항목 | 값 |
|------|-----|
| 난이도 | ★★ 중하 |
| 시간 제한 | 1초 |
| 기출 | 2019 SW마에스트로 입학테스트 |

### 문제 요약

무게가 다른 두 볼링공을 고르는 경우의 수. 무게는 1~M. (N개 공)

예: N=5, M=3, 무게 `[1,3,2,3,2]` → `8`

### 핵심 아이디어

무게별 개수를 센 뒤, 무게 `i`인 공을 A가 고르면 B는 **무게가 다른(아직 안 센) 공** 중 하나. `count[i] × (남은 공 수)`를 누적한다(중복 카운트 방지).

```ts
const input = `5 3
1 3 2 3 2`;
const lines = input.split("\n");
const [n, m] = lines[0].split(" ").map(Number);
const weights = lines[1].split(" ").map(Number);

const count = new Array(m + 1).fill(0);
for (const w of weights) {
  count[w]++;
}

let result = 0;
let remaining = n;
for (let i = 1; i <= m; i++) {
  remaining -= count[i]; // 무게 i인 공을 후보에서 제외
  result += count[i] * remaining; // A=무게i × B=다른무게
}
console.log(result); // 8
```

**시간복잡도**: O(N + M).

## Q06 무지의 먹방 라이브

| 항목 | 값 |
|------|-----|
| 난이도 | ★★★ 중 |
| 시간 제한 | 1초 |
| 기출 | 2019 카카오 신입공채 (프로그래머스) |

### 문제 요약

회전판의 음식들을 1초씩 돌아가며 먹는다. K초 후 네트워크 장애가 났을 때 **다음에 먹을 음식 번호**를 구하라. 다 먹었으면 -1. (효율성 테스트: 길이 20만, K ≤ 2×10¹³)

예: `food_times=[3,1,2]`, `k=5` → `1`

### 핵심 아이디어

1초씩 시뮬레이션하면 K가 커서 시간 초과다. **시간이 적은 음식부터 정렬**해, "가장 적은 음식 시간 × 남은 음식 수"만큼 한 번에 건너뛴다. K를 다 소진하지 못하면, 남은 음식을 **번호순**으로 정렬해 `(K - 소진시간) % 남은개수` 위치의 음식이 답이다.

```ts
function solution(foodTimes: number[], k: number): number {
  const total = foodTimes.reduce((a, b) => a + b, 0);
  // 전체 시간 이내에 다 먹으면 더 먹을 음식 없음
  if (total <= k) {
    return -1;
  }

  // [시간, 번호(1-indexed)] 를 시간 오름차순 정렬 (힙 대신 정렬 + 포인터)
  const q = foodTimes.map((t, i): [number, number] => [t, i + 1]);
  q.sort((a, b) => a[0] - b[0]);

  let head = 0;
  let sumValue = 0; // 지금까지 소진한 시간
  let previous = 0; // 직전에 다 먹은 음식의 시간
  let length = foodTimes.length; // 남은 음식 수

  // 가장 적은 음식을 통째로 끝낼 수 있는 동안 반복
  while (sumValue + (q[head][0] - previous) * length <= k) {
    const now = q[head][0];
    head++;
    sumValue += (now - previous) * length;
    length--;
    previous = now;
  }

  // 남은 음식들을 번호순으로 정렬해, 남은 K만큼 이동한 위치
  const rest = q.slice(head).sort((a, b) => a[1] - b[1]);
  return rest[(k - sumValue) % length][1];
}

console.log(solution([3, 1, 2], 5)); // 1
```

**시간복잡도**: O(N log N).

> **핵심 통찰**: "1초씩"을 보면 **수학적으로 한 번에 건너뛸 수 있는지**(Ch 03 "1이 될 때까지"와 같은 발상) 의심하라. 여기서는 "최소 시간 음식 기준으로 전체를 일괄 차감"이 그 핵심이다.

## 요약

- **모험가 길드·만들 수 없는 금액**: 정렬 후 작은 것부터 훑으며 **경계(인원/target)를 키운다**.
- **곱하기 혹은 더하기**: 0·1이면 더하기, 아니면 곱하기 — 단순 규칙의 정당성.
- **문자열 뒤집기**: 0 묶음/1 묶음 중 작은 쪽.
- **볼링공 고르기**: 무게별 카운트 + `count[i] × 남은 수`로 중복 없이 조합.
- **무지의 먹방**: "1초씩"을 **정렬 + 일괄 차감**으로 (효율성 테스트 대응).
- 공통 교훈: 그리디는 **정렬이 출발점**인 경우가 많고, 선택의 **정당성**을 늘 확인한다(Ch 03).
