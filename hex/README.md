## 개요

헥사고날 아키텍처(포트와 어댑터 아키텍처)를 **실제 애플리케이션을 만들면서** 배우는 책. 비즈니스 코드와 기술 코드를 분리해 변경에 강한 시스템을 만드는 것이 핵심 주제다.

이 책은 헥사고날 아키텍처를 세 개의 동심 헥사곤으로 모델링한다.

- **도메인 헥사곤**: 가장 안쪽. 엔티티·값 객체·명세 등 비즈니스 규칙과 데이터. 어떤 헥사곤에도 의존하지 않는다
- **애플리케이션 헥사곤**: 유스케이스와 포트(입력/출력)로 시스템의 동작을 표현·조율
- **프레임워크 헥사곤**: 가장 바깥. 입력/출력 어댑터로 외부 기술(REST, DB, 메시징 등)과 연결

외부와의 상호작용은 **드라이빙 오퍼레이션**(외부가 시스템을 구동 — 프런트엔드, 다른 애플리케이션)과 **드리븐 오퍼레이션**(시스템이 외부를 구동 — 데이터베이스, 메시지 브로커)으로 구분한다.

## 예제 도메인

전 장을 관통하는 실습 예제는 **통신 회사의 네트워크 토폴로지 & 인벤토리 관리 시스템**이다. 라우터(코어/에지)·스위치·네트워크를 엔티티와 명세로 모델링하고, 이를 3개 헥사곤으로 구현한 뒤 쿼커스로 클라우드 네이티브 전환까지 진행한다.

## 학습 가이드

1. **1부(Ch1~5)로 개념 어휘부터** — 헥사곤 3층·포트·어댑터·드라이빙/드리븐 구분이 이후 전부의 어휘
2. **2부(Ch6~9)는 손으로** — 같은 도메인을 세 헥사곤으로 나눠 구현하는 과정이라 코드를 따라가야 남는다
3. **3부(Ch10~15)는 자바/쿼커스 비중이 큼** — TS 개발자라면 프레임워크 세부보다 "기술 교체가 어댑터 교체로 끝난다"는 구조 검증 관점으로 읽기
4. 관련 책: 『만들면서 배우는 클린 아키텍처』(위키북스) — 본문에서 여러 차례 참조하는 같은 계열의 실습형 아키텍처 서적

## 핵심 개념 맵

- **구조**: 도메인 헥사곤 → 애플리케이션 헥사곤 → 프레임워크 헥사곤 (의존성은 항상 안쪽으로)
- **도메인 모델링(DDD)**: 엔티티, 값 객체, 애그리게잇, 도메인 서비스, 정책·명세 패턴
- **동작 표현**: 유스케이스(인터페이스) + 입력 포트(구현) / 출력 포트(외부 데이터 요구 정의)
- **외부 연결**: 입력 어댑터(REST 등 드라이빙) / 출력 어댑터(DB 등 드리븐)
- **의존성 역전**: 자바 모듈 시스템(JPMS)과 `ServiceLoader`로 헥사곤 간 결합 제거
- **클라우드 네이티브**: 쿼커스(CDI, RESTEasy 리액티브, 하이버네이트 리액티브) → 도커 이미지 → 쿠버네티스 배포
