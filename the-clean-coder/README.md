# 클린 코더 (The Clean Coder: A Code of Conduct for Professional Programmers)

> *The Clean Coder: A Code of Conduct for Professional Programmers* (Robert C. Martin, Prentice Hall, 2011)
> 한국어판: 『클린 코더 — 단순 기술자에서 진정한 소프트웨어 장인이 되기까지』 (정희종 옮김, 에이콘출판사, 2016)

소프트웨어 개발자가 **프로페셔널**이 된다는 것이 무엇인지를 다루는 행동 강령서. 같은 저자의 『클린 코드』가 **코드의 품질**에 집중한다면, 이 책은 코드를 작성하는 **사람의 태도와 행동**에 집중한다.

## 개요

Uncle Bob이 40년 이상의 개발 경험에서 얻은 실패와 성공의 교훈을 솔직하게 공유하며, 프로 개발자의 규율·기술·태도를 구체적인 행동 강령으로 제시한다. 해고당한 경험, 고객에게 거짓말한 경험, 일정을 지키지 못한 경험을 숨기지 않고, 그 실패담을 각 원칙의 설득력 있는 근거로 쓴다.

"프로가 되어라"라는 모호한 조언 대신 "No라고 말하는 방법", "추정하는 방법", "압박 속에서 행동하는 방법" 같은 실천 지침을 제공하며, 챕터마다 관리자-개발자 대화 시나리오로 이상적인 대응과 잘못된 대응을 대비시킨다. TDD 필수론·야근 반대론·"QA는 아무것도 찾지 못해야 한다" 등 논쟁적 입장도 많다.

『클린 코드』(코드 품질) → 『클린 코더』(개발자의 태도) → 『클린 아키텍처』(시스템 구조)로 이어지는 로버트 마틴 시리즈의 두 번째 책이다.

## 학습 가이드

1. **1단계 — 마인드셋 (Ch0~1)**: 모든 원칙의 기반인 "프로페셔널리즘"의 정의부터
2. **2단계 — 커뮤니케이션 (Ch2~3)**: 개발자가 가장 어려워하는 거절하기와 약속하기. 실전 대화 시나리오 중심
3. **3단계 — 기술 규율 (Ch4~6)**: 코딩할 때의 마음가짐, TDD, 연습(카타·와사)
4. **4단계 — 팀에서 일하기 (Ch7~11)**: 인수 테스트로 소통, 시간 관리, 정직한 추정, 압박 대응
5. **5단계 — 조직과 성장 (Ch12~14)**: 협업, 응집된 팀, 멘토링과 장인정신

## 핵심 개념 맵: 프로의 5가지 핵심 속성

| 속성 | 설명 | 관련 챕터 |
|------|------|-----------|
| **책임감(Responsibility)** | 자신의 코드와 결정에 책임진다. QA가 버그를 찾도록 내버려두지 않는다 | Ch 1, 5, 8 |
| **규율(Discipline)** | 압박 속에서도 TDD·리팩터링 규율을 유지한다. 위기가 규율의 진짜 시험이다 | Ch 4, 5, 11 |
| **정직(Honesty)** | "시도해 보겠다"가 아니라 가능/불가능을 솔직히 말한다. 추정을 약속으로 포장하지 않는다 | Ch 2, 3, 10 |
| **협업(Collaboration)** | 혼자를 선호하더라도 팀과 비즈니스 목표를 위해 적극적으로 소통한다 | Ch 7, 12, 13 |
| **지속 학습(Continuous Learning)** | 주 40시간은 고용주의 시간, 자기 개발 시간은 따로 투자한다 | Ch 1, 6, 14 |
