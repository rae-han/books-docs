# 읽기 쉬운 코드 (Code That Fits in Your Head: Heuristics for Software Engineering)

> *Code That Fits in Your Head: Heuristics for Software Engineering* (Mark Seemann, Addison-Wesley, 2021. Robert C. Martin Series)
> 한국어판: 『읽기 쉬운 코드 — 좋은 코드를 작성하는 좋은 개발자 습관』 (김현규 옮김, 길벗, 2022)

마크 시먼(*Mark Seemann - 『Dependency Injection Principles, Practices, and Patterns』의 저자이자 20년 이상 소프트웨어 컨설턴트로 활동한 개발자*)의 소프트웨어 공학 휴리스틱 안내서. 완벽한 규칙이 아니라 **경험 법칙(*Heuristics - 완벽하지는 않아도 실용적으로 유용한 문제 해결 지침*)** 으로 프로그래밍을 공학에 가깝게 만드는 실용 가이드다.

## 개요

이 책의 중심 기준은 **인간의 뇌**다. 단기 기억 7개 한계에 맞춰 순환 복잡도 7 이하, 활성화된 의존성 7 이하처럼 정량적 임계값을 제시하고, "코드가 머리에 들어오는가"를 설계 판단의 축으로 삼는다. 소프트웨어는 규칙이 아니라 휴리스틱으로 이루어진다 — 상황에 따라 다르지만, 그렇다고 아무거나 해도 되는 것은 아니다.

Part 1(가속)은 새 코드베이스에서 배포까지 속도를 내는 방법(체크리스트·TDD·캡슐화·API 설계·팀워크), Part 2(지속가능성)는 배포된 코드베이스를 오래 유지하는 방법(기능 플래그·테스트 편집·과학적 디버깅·관심사 분리)을 다룬다. C# 레스토랑 예약 시스템 하나가 전체를 관통하므로 순차 읽기가 자연스럽다. 『클린 코더』·『소프트웨어 장인』과 같은 로버트 C. 마틴 시리즈다.

## 학습 가이드

1. **처음이라면 순서대로** — 레스토랑 예약 시스템이 챕터마다 성장하므로 순차 읽기가 자연스럽다
2. **실무 즉시 활용**: Ch2(체크리스트) → Ch3(복잡도 임계값) → Ch11(테스트 편집) → Ch12(과학적 디버깅) → 부록 A(28개 프랙티스)
3. **TDD·설계 관심**: Ch4(수직 슬라이스) → Ch5(캡슐화) → Ch6(다각화) → Ch7(분해) → Ch8(API 설계)
4. **오래된 코드 유지보수 중**: Ch10(기능 플래그·스트랭글러) → Ch13(함수형 코어·명령 셸) → Ch14(리듬) → Ch15(성능·보안)

## 핵심 개념 맵

- **휴리스틱 > 규칙**: 소프트웨어 공학의 지침은 상황 의존적 경험 법칙 — 그래도 방법론 없이 일하는 것보다 낫다 (Ch1)
- **뇌의 한계가 기준**: 단기 기억 7개 → 순환 복잡도 ≤ 7, 의존성 ≤ 7, 80/24 규칙 — 머리에 들어오는 코드 (Ch3·7)
- **캡슐화 = 계약**: 사전조건·사후조건·불변식으로 "항상 유효한 상태"를 보증 (Ch5)
- **테스트가 공학의 실험**: 외부 접근 TDD·삼각측량·결함 재현 테스트 — 확신은 실험에서 나온다 (Ch4·6·12)
- **작은 반복이 큰 계획을 이긴다**: 작은 커밋·지속적 통합·기능 플래그·스트랭글러 패턴 (Ch9·10)
- **함수형 코어·명령 셸**: 순수 함수를 중심에, 부수 효과를 가장자리에 — 관심사 분리의 실천형 (Ch13)
