## 책 소개

저자 데이비드 토머스는 *『실용주의 프로그래머(The Pragmatic Programmer)』*의 공저자이자, *애자일 소프트웨어 개발 선언*을 만든 17명 중 한 명이다. 이 책은 그가 25년 가까이 코드와 함께 살아온 끝에 도달한 결론을 담고 있다 — **복잡함과 싸우기 위한 가장 강력한 무기는 단순함이다.**

> 단순함이란 중요한 것을 드러내기 위해 불필요한 것을 제거하는 것입니다.

저자는 두 단어를 엄격히 구분한다. *복합적(Complex)*인 것은 본질적 어려움이고, *복잡함(Complicated)*은 우리 스스로 만들어 낸 어려움이다. 눈송이는 복합적이지만 그 눈송이를 만든 날씨는 복잡하다. 본질적 복잡성은 받아들이되, 인위적으로 꼬아 낸 복잡함은 제거해야 한다.

AI가 코드를 대신 작성해 주고 프레임워크가 점점 더 거대해지는 시대에 저자는 정면으로 묻는다. *"정말 이렇게까지 복잡해야 할까?"* 책은 코드뿐 아니라 회의·소프트스킬·자동화·데이터 주도 설계까지 폭넓게 다루며, **"복잡함은 기술의 문제가 아니라 태도의 문제"**임을 차분히 설득한다.

## 핵심 메시지

```
[ 우리가 짊어진 짐 ]            [ 미니멀리즘 프로그래머 ]
- 43개 의존성을 가진 빈 프로젝트    무엇을 비울 것인가?
- 2주마다 13.5시간의 회의            현황파악 → 실행 → 학습
- 만들지 않아도 됐던 기능들          기능이 아닌 가치를 전달
- 매일 마주하는 인지 부담            자동화로 기억을 외주
- 코드와 분리되지 않은 데이터        데이터에 운전대 맡기기
```

저자는 **29개의 실천법(Practice)**을 제시하지만, 정답으로 따르지 말 것을 거듭 강조한다. 단순함은 *방법*이 아니라 *마음가짐*이며, 책에 담긴 모든 프랙티스는 저자가 걸어온 한 여정의 기록이다.

## 핵심 개념

### 두 단어의 구분 — 본질과 인위

| 구분 | 의미 | 예시 |
|------|------|------|
| **Complex (복합적)** | 본질적 어려움. 규칙과 패턴이 존재 | 눈송이, 잘 설계된 소프트웨어 |
| **Complicated (복잡함)** | 우리가 만들어 낸 어려움 | 변덕스러운 날씨, 엉킨 레거시 |

본문에서는 *복합적복잡성*(Complex Complexity)으로 본질적 어려움을 가리키고, 그 반대인 *단순함*(Simplicity)을 추구한다.

### 메타 프레임워크 — 현황파악·실행·학습

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 현황파악  │ ──▶│   실행   │ ──▶│   학습   │
│  Orient  │    │   Step   │    │  Learn   │
└──────────┘    └──────────┘    └──────────┘
      ▲                                │
      │                                │
      └────────────────────────────────┘
              다음 사이클로 반영
```

이 사이클은 모든 챕터의 모든 프랙티스에 적용된다. 1장에서 도입되어 9장의 5가지 평가 기준으로 닫힌다.

### 단순함의 5가지 평가 기준 (9장)

1. *이해*: S는 C보다 이해하기 쉬운가?
2. *파악 속도*: S가 어떻게 동작하는지 더 빠르게 파악되는가?
3. *설명 가능성*: S를 더 간단한 문장으로 다른 사람에게 설명할 수 있는가?
4. *진행 속도*: S 사용 시 멈춰서 고민하는 시간이 더 적은가?
5. *변경 자신감*: S를 변경할 수 있다는 자신감이 더 높은가?
