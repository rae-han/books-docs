# Chapter 30: 소스 제어 (Source Control)

## 핵심 질문

여러 개발자가 같은 코드를 안전하게 함께 작업하게 해주는 소스 제어란 무엇이며, 저장소·브랜치·병합·충돌 같은 핵심 개념은 어떻게 작동하는가?

> HP의 앤트이터 프로젝트 시절, 팀원 브라이언이 임시 파일 변수를 `C:\temp`로 초기화하도록 헤더를 바꿨다. 나는 같은 헤더를 수정했지만 **아직 병합하지 않은 상태**여서, 내 변수는 초기화되지 않아 기본값 `C:\`(루트)를 쓰고 있었다. 그런데 내 앱은 그 위치의 파일을 모조리 지우도록 돼 있었다 — 실행할 때마다 **내 컴퓨터의 파일을 전부 삭제**한 것이다. 소스 제어는 이렇게나 재미있다.

## 1. 소스 제어란

소스 제어(버전 제어)는 **파일의 버전과 소스 코드를 추적**하고, 여러 개발자가 같은 파일로 작업할 때 이를 **조율**하는 방법이다.

## 2. 왜 중요한가

- **코드 분실 방지**: 안전한 저장 장소가 생겨 실수로 코드를 삭제할 일이 없고, 수정 이력을 추적해 되돌릴 수 있다.
- **여러 개발자의 동시 작업**: 서로의 수정을 덮어쓰지 않고 나중에 병합할 수 있다.
- **여러 버전 관리**: 이미 배포한 버전1의 버그를 고치면서 동시에 버전2 신기능을 개발하고, 버그 수정만 신기능 버전에 반영할 수 있다.

## 3. 소스 제어의 기본 개념

- **저장소(repository)**: 코드가 '사는' 곳. 저장소에서 코드를 가져와 작업한 뒤 수정사항을 체크인한다.
- **체크아웃(checkout)**: 저장소에서 로컬 버전 코드를 자신의 컴퓨터로 가져오는 것. 수정 후 **체크인/병합**해야 남에게 보인다.
- **리비전(revision)**: 파일의 이전 버전. 망치면 이전 리비전으로 되돌아가고, 이력을 비교해 누가 무엇을 바꿨는지 볼 수 있다(손메즈는 이를 "비난하기"라 부른다).
- **분기(branch)**: 코드를 **나무**로 생각하라 — 몸통(트렁크)에서 가지(브랜치)가 나온다. 버전1을 브랜치로 떼어 버그를 고치고, 트렁크에서는 신기능을 개발한다.
- **병합(merge)**: 한 코드의 변경사항을 다른 코드에 합치는 것. 버전1 브랜치의 버그 수정을 트렁크로 병합한다(한 방향).
- **충돌(conflict)**: 두 개발자가 **같은 파일의 같은 부분**을 수정하면 컴퓨터가 우선순위를 몰라 **수작업으로 해결**해야 한다.

> **존의 조언**: "간단한 병합이니 금요일 퇴근 직전에 후딱 하자"는 생각은 현명치 못하다. `<<<<`와 `>>>>` 기호를 쳐다보며 밤을 새울 수 있다. **병합은 월요일 아침에 하자.**

## 4. 소스 제어 기술 — 중앙집중형 vs 분산제어형

| 구분 | 중앙집중형 | 분산제어형(DVCS) |
|------|-----------|------------------|
| 저장소 | 중앙 서버에 **한 개** | 각 개발자가 **완전한 사본**을 보유 |
| 작업 | 중앙에서 체크아웃/체크인 | 로컬에서 대부분 작업 후 **pull/push** |
| 특징 | 이해하기 쉬움, 다소 느림 | 오프라인 작업 가능, 더 복잡·인기 |
| 예 | CVS, 서브버전(Subversion) | 깃(Git), 머큐리얼(Mercurial) |

> **핵심 통찰**: 분산제어형에서도 "중앙 저장소는 없고 모두 평등하다"는 말은 틀렸다 — 현실적으로 배포·조율을 위한 **마스터 저장소는 있어야** 한다.

**인기 시스템**: **깃(Git)이 소스 제어의 동의어**가 될 만큼 압도적이다(깃허브를 통해 오픈소스도 지원). 서브버전은 중앙집중형 중 가장 낫고, 머큐리얼은 깃과 유사하나 조금 더 이해하기 쉽다.

> **존의 조언**: 이제 소스 제어의 기본을 배웠다. **미리 커밋하고 자주 커밋하라. 그리고 부탁하건대 커밋 메시지 좀 제대로 작성하자.**

## 실천 항목

- [ ] **깃(Git)** 기본 흐름(clone/branch/commit/merge/push)을 직접 연습한다.
- [ ] 최신 코드를 **병합**한 뒤 작업하는 습관을 들여(브라이언 사례 같은 사고 방지) 충돌을 조기에 해결한다.
- [ ] 기능마다 **브랜치**를 만들고, **작고 자주, 의미 있는 커밋 메시지**로 커밋한다.
- [ ] 병합 충돌이 나면 **수작업으로 신중히** 해결한다(금요일 저녁 X).

## 요약

- 소스 제어는 코드를 **분실 방지·동시 작업·여러 버전 관리**하게 해준다.
- 핵심 개념: **저장소·체크아웃·리비전·브랜치/트렁크·병합·충돌**.
- 기술은 **중앙집중형(CVS·서브버전)**과 **분산제어형(깃·머큐리얼)**으로 나뉘며, **깃이 사실상 표준**이다.
- 실무 수칙: **작게 자주 커밋하고, 좋은 커밋 메시지를 쓰고, 병합은 여유 있을 때** 하라.
