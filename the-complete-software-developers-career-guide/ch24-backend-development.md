# Chapter 24: 백엔드 개발 (Backend Development)

## 핵심 질문

사용자 눈에 보이지 않는 "백엔드"에서는 무슨 일이 일어나며, 백엔드 개발자는 어떤 기술을 갖춰야 하는가? 그리고 요즘 각광받는 "풀스택 개발자"는 어떻게 접근해야 하는가?

> **핵심 통찰**: 소프트웨어 애플리케이션은 **빙산과 같다.** 사용자에게 드러난 부분(프런트엔드)보다 보이지 않는 영역(백엔드)이 더 크다. 프런트엔드가 영광을 가져가지만, 세상 코드 대부분은 **최종 사용자가 볼 수 없는 백엔드 코드**다.

## 1. 백엔드 개발이란

**사용자 인터페이스를 만드는 코드와 관련 없는 모든 개발**을 백엔드 개발이라 한다. API 작성, 라이브러리 제작, UI 없는 시스템 컴포넌트, 과학적 프로그래밍까지 아우른다.

## 2. 백엔드 개발자가 하는 일

프런트엔드 개발자가 버튼을 만들면, 백엔드 개발자는 **그 버튼을 작동시키는 코드**를 작성한다 — 데이터베이스에서 데이터를 가져와 프런트엔드로 전달하고, 비즈니스 로직과 규칙을 따라 결과를 계산한다. 또한 **시스템 아키텍처**에 깊이 개입해 로직을 조직하고 유지보수하기 쉽게 만든다.

> **핵심 통찰**: 백엔드 개발자는 프런트엔드보다 **알고리즘 적용과 문제 해결**에 더 많은 시간을 쓴다. 손메즈가 백엔드를 좋아하는 이유도 더 어렵다고 느끼기 때문이다.

## 3. 백엔드 개발 기본 기술

- **SQL·데이터베이스**: 백엔드는 데이터베이스에 연결돼 데이터를 읽고 쓰고 처리하므로 SQL 같은 기술이 필수다.
- **서버 측 언어·프레임워크**: 프런트엔드가 HTML·CSS·자바스크립트에 집중할 때, 백엔드는 PHP 웹 프레임워크·루비 온 레일스·ASP.NET MVC 등 **서버 측 프레임워크**를 배워야 한다.
- **애플리케이션 아키텍처**: 애플리케이션의 아키텍처와 내부 설계를 대개 백엔드 개발자가 담당한다. 프레임워크·라이브러리 통합, 유지보수하기 쉬운 코드·비즈니스 로직 구조화를 알아야 한다.

## 4. 풀스택 개발자는?

프런트엔드와 백엔드를 **둘 다** 하는 사람을 풀스택 개발자라 한다. 시스템의 모든 컴포넌트·계층(서버 하드웨어·아키텍처·데브옵스까지)을 다룬다. 요즘 이런 인재를 더 많이 찾는다(앵귤러 같은 프레임워크가 프런트/백 경계를 흐리고, 애자일 팀은 개인에게 전문 영역 이상을 요구하기 때문).

> **핵심 통찰**: 하지만 풀스택 **'전문가'**가 되겠다는 건 좋은 생각이 아니다 — **풀스택은 전문 분야로 볼 수 없기** 때문이다. 기술 스택 전반에 **폭넓은 지식**을 갖추되, **깊이 있게 파고들 핵심 분야를 한두 개** 골라라. 그 정도면 스스로를 '풀스택'이라 부르면서도 팀에서 훨씬 높은 가치를 인정받는다.

## 실천 항목

- [ ] **SQL과 데이터베이스** 기본(조회·삽입·수정·삭제·조인)을 실습으로 익힌다.
- [ ] 자신의 스택에 맞는 **서버 측 프레임워크** 하나를 정해 학습한다.
- [ ] 기술 스택 전반을 폭넓게 이해하되, **깊이 팔 전문 분야 1~2개**를 정한다.

## 요약

- 백엔드는 **보이지 않는 곳의 코드** — 로직·데이터·아키텍처를 담당하는 소프트웨어 개발의 진짜 일꾼이다.
- 기본 기술은 **SQL·데이터베이스, 서버 측 언어·프레임워크, 애플리케이션 아키텍처**다.
- 풀스택은 폭넓은 지식으로 접근하되 **전문 분야를 한두 개** 확보하는 것이 현명하다("풀스택 전문가"는 지양).
