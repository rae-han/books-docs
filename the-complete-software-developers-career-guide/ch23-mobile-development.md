# Chapter 23: 모바일 개발 (Mobile Development)

## 핵심 질문

모바일 개발은 왜 1인 개발자에게 특별한 기회이자 개발의 미래로 손꼽히는가? 주요 플랫폼과 개발 방식에는 어떤 선택지가 있는가?

> **핵심 통찰**: 모바일 앱은 보통 **한 가지 목적을 위해 소규모로** 제작하므로 접근성이 좋아, 개발자 혼자서도 단기간에 앱을 만들고 사업을 시작해 볼 수 있다. 게다가 모바일 기기가 삶에서 차지하는 비중이 커지면서 **모바일 개발이 개발의 미래**가 될 것이라는 의견이 우세하다.

## 1. 모바일 개발이란

단순히 "휴대전화용 앱"이 아니라 **휴대전화·태블릿·스마트워치 등 모바일 OS로 구동되는 모든 기기용 앱 개발**을 아우른다. 오늘날엔 웹 개발자도 모바일 기기에서의 접근을 고려해야 하므로 순수 모바일 앱 개발자만 모바일 개발자라 하긴 어렵다.

## 2. 주요 플랫폼

시장은 **두 강자와 나머지**로 나뉜다.

| 항목 | iOS | 안드로이드 |
|------|-----|-----------|
| 제공 | 애플(애플 기기 전용) | 구글(**오픈소스**, 다양한 제조사) |
| 기반 | 다윈(BSD 계열, 유닉스 유사) | 리눅스 커널 |
| 네이티브 언어 | 오브젝티브-C / **스위프트**(요즘 대세) | **자바**(공식적으로 C/C++도) |
| 개발 도구 | 엑스코드(맥 필수) | 안드로이드 스튜디오 |

그 외(윈도우·블랙베리 등)는 시장 지분이 2%도 안 되므로, 손메즈는 **전용 앱 개발을 권하지 않는다** — 모바일 앱이면 iOS나 안드로이드 중 하나를 선택하라.

## 3. 개발 방식

- **네이티브 개발**: 판매사 제공 툴로 개발. **속도가 가장 빠르고** 하드웨어·네이티브 기능을 직접 다룰 수 있다. 하지만 iOS·안드로이드 코드를 완전히 따로 써야 해 **유지보수가 악몽**이 될 수 있다(도구·언어·패러다임이 서로 다르다).
- **크로스플랫폼 프레임워크/툴**: 자마린(Xamarin, C#으로 네이티브 앱), 코르도바(하이브리드 앱) 등. 선택 시 고려할 점:
  - 어떤 **프로그래밍 언어**를 지원하는가?(쓸 줄 아는 언어 지원하는 것)
  - **네이티브 vs 하이브리드** 중 무엇을 원하는가?(네이티브가 대체로 빠르고 플랫폼과 유사)
  - 지원해야 할 **플랫폼**은 몇 가지인가?(게임이라면 유니티3D처럼 많이 지원하는 툴)
  - **코드 재사용**을 얼마나 원하는가?(네이티브에 가까울수록 재사용 코드는 줄어듦)
- **모바일 웹 앱**: 모바일 브라우저에서 작동하는 웹 앱. 브라우저가 위치·카메라 등 네이티브 기능을 호출할 수 있게 되면서 가능해졌다. 손메즈는 "**모바일 웹 앱이 미래**"라고 본다.

## 4. 모바일 개발로 입문하기

> **존의 조언**: 모바일 개발로 소프트웨어 개발에 입문하는 건 아주 좋은 생각이다 — **진입 장벽이 낮고 성장 가능성이 높다.** 앱은 태생적으로 범위가 제한적이라 혼자서도 단기간에 만들 수 있어, 앱스토어에 배포하면 **코딩·앱 개발 능력이 입증**되고 좋은 포트폴리오가 된다.

## 실천 항목

- [ ] iOS/안드로이드 중 **하나의 플랫폼**을 정한다(군소 플랫폼 전용 개발은 피한다).
- [ ] 네이티브 vs 크로스플랫폼(자마린 등) 중 **자신의 언어·목표에 맞는** 방식을 고른다.
- [ ] 범위가 좁은 **작은 앱 하나를 앱스토어에 배포**해 포트폴리오를 만든다.

## 요약

- 모바일 개발은 **1인 개발자에게 기회**를 열어주고 개발의 미래로 여겨진다.
- 두 강자는 **iOS(스위프트)·안드로이드(자바)**이며, 군소 플랫폼 전용 개발은 피하라.
- 방식은 **네이티브(빠름·유지보수 부담)·크로스플랫폼(자마린 등)·모바일 웹 앱**으로 나뉜다.
- 진입 장벽이 낮아 **입문·포트폴리오용으로 최적**이다.
