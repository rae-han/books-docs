# Chapter 21: 프로그래밍 언어 개요 (A Brief Overview of Programming Languages)

## 핵심 질문

수많은 프로그래밍 언어를 전부 익힐 필요는 없다면, 개발자로서 최소한 알아두어야 할 주요 언어들은 무엇이고 각각 어떤 특징과 쓰임새를 가지는가?

> **핵심 통찰**: 현존하는 모든 언어를 전문가 수준으로 익힐 필요는 없다. 하지만 **주요 언어가 무엇이고 각각 어떤 차이가 있는지** 알아두면, 해야 할 작업에 적합한 도구가 무엇인지 알 수 있다. 이 장은 손메즈의 **주관적 관점**에서 고른 개요다.

## 1. 주요 프로그래밍 언어

| 언어 | 등장·개발 | 특징과 쓰임새 |
|------|----------|--------------|
| **C** | 1969~73, 데니스 리치(벨 연구소) | 메모리를 직접 조작하는 **저수준(시스템) 언어**. 아마 세계에서 가장 많이 쓰였고, 다수 현대 언어의 뿌리. OS·임베디드·게임. |
| **C++** | 비야네 스트롭스트룹 | C에 객체지향(시뮬라의 클래스·가상 함수)을 더한 **C의 상위 집합**. 게임 개발에 많이 쓰임. 매우 강력하지만 복잡해 초보엔 비추천. |
| **C#** | 마이크로소프트(아네르스 하일스베르) | .NET을 대표하는 언어. 초기엔 자바의 복제판 소리를 들었고 지금도 **한 언어를 알면 다른 언어의 90%를 아는** 셈. 우아하고 사용하기 쉬운 객체지향 언어. |
| **자바** | 1995, 제임스 고슬링(선) | **가상 머신** 위에서 실행돼 한 번 작성하면 여러 플랫폼에서 구동. C/C++ 기반 객체지향이지만 저수준 메모리 조작은 금지. 현재 오라클 소유. |
| **파이썬** | 1989, 귀도 반 로섬 | **가독성**을 중시한 우아하고 간결한 인터프리터 언어. 객체지향·절차지향·함수형 모두 가능. 초보 학습·구글 주요 언어. |
| **루비** | 1993, 마츠모토 유키히로 | "쉽고 재미있게"가 설계 목표인 객체지향 스크립트 언어. **루비 온 레일스(2003)**로 인기 폭발. 부트캠프 입문 언어로 인기. |
| **자바스크립트** | 1995, 브렌던 아이크(10일 만에!) | 급조된 탓에 문제가 많지만, **편의성·보편성**으로 웹의 기본 언어가 됨. 최신판(ECMAScript)은 대규모 개발에도 적합해짐. |
| **펄(Perl)** | 1987, 래리 월 | 유닉스 스크립트·문자열 처리에 강한 "스크립트 언어계의 스위스 아미 전기톱". 유연·강력하지만 **가독성이 떨어짐**. |
| **PHP** | 1994, 라스무스 러도프 | 우아하지 않고 '지저분'하지만 자바스크립트와 함께 웹의 큰 부분을 구성. 워드프레스·초기 페이스북. 배우기 쉬움. |
| **오브젝티브-C** | 1980년대 초, 브래드 콕스·톰 러브 | C에 스몰토크의 객체지향을 더함. 애플이 macOS·iOS에 채택하며 부활. 배우기 어렵고 장황함. |
| **스위프트** | 애플 | 오브젝티브-C를 대체하는 iOS/OS X용 새 언어. 유연함은 유지하되 **더 단순·간결**. iOS 입문이면 바로 스위프트로 시작해도 좋음. |
| **Go** | 2007, 구글(그리스머·파이크·톰프슨) | C와 비슷하지만 **가비지 컬렉션·동시성**을 갖춰 더 단순·강력. 시스템 언어로 출발해 웹까지 확장. |
| **얼랭(Erlang)** | 1986, 에릭슨 | 동시·분산 처리용 함수형 언어. **핫스와핑**(멈추지 않고 코드 변경)을 지원. 가장 안정적인 환경으로 평가. |
| **하스켈(Haskell)** | 1990(위원회) | **순수 함수형** 언어. 강력한 정적 타입 체계로 배우기 어렵지만, 부작용 없이 예측 가능한 코드를 만듦. |

### 대화: 〈잠깐만요, 존!〉

> **독자**: "자바스크립트가 웹 개발 언어로 그렇게 인기인 이유가 뭐죠? 최고의 언어도 아닌데요."
>
> **존**: 훌륭해서가 아니라 **편의성과 보편성** 때문이다. 웹 초창기부터 거의 모든 브라우저에 내장돼 있었고, 인터프리터 언어라 컴파일 없이 웹페이지로 전달돼 브라우저에서 바로 실행된다. 플래시 등 경쟁자도 있었지만 **"어디서나 찾을 수 있는 보편성"**으로 자바스크립트가 이겼다.

## 2. 빠뜨린 것에 대하여

정적/동적, 객체지향/함수형 같은 **분류법**은 이 장에서 자세히 다루지 않았다. 요즘 언어는 여러 패러다임을 통합해 하나로 명확히 분류하기 어렵기 때문이다. (깊이 알고 싶다면 『Types and Programming Languages』(Benjamin C. Pierce) 추천.)

### 대화: 〈잠깐만요, 존!〉

> **독자**: "제가 가장 좋아하는 러스트(Rust)·스칼라·리스프가 빠졌어요! 인터프리터·컴파일러가 뭔지도 알아야죠!"
>
> **존**: 심호흡하라. 서두에 **주관적으로 고른 '일부'**라고 했다. 배스킨라빈스에서 31가지를 다 맛볼 필요는 없다. 이 장의 핵심은 앞으로 만날 세상을 짧게 엿보고 강타자들의 개요를 아는 것이다. 인터프리터·컴파일러·정적/동적 타입은 분명 중요하지만, 그걸 다 배우려면 한 장으로는 부족하다. 3부의 초점은 **"무엇을 알아야 하는지"**를 알려주는 데 있으니, 더 깊은 내용은 스스로 배워라.

## 실천 항목

- [ ] 위 표에서 **자신이 쓰는(쓸) 언어**와 그 **뿌리 언어**(예: 자바스크립트 ← C 계열)를 확인한다.
- [ ] 관심 가는 언어 1~2개를 골라 **정적/동적, 객체지향/함수형** 성격을 스스로 조사한다.
- [ ] "작업에 맞는 도구"를 고르는 관점에서 각 언어의 **주 용도**를 정리한다.

## 요약

- 모든 언어를 익힐 필요는 없고, **주요 언어의 특징과 차이**를 아는 것으로 충분하다.
- C 계열(C·C++·C#·자바)이 큰 뿌리를 이루며, 파이썬·루비는 가독성·재미, 자바스크립트는 웹의 보편 언어다.
- 자바스크립트의 인기는 **편의성·보편성**의 산물이지 우수성의 결과가 아니다.
- 언어 분류법·이론은 중요하지만, 3부의 목표는 **큰 그림 파악** — 깊이는 스스로 채운다.
