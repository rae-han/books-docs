# 부록 B: 악취 제거 기법 (Smells to Refactorings)

3장에서 다룬 **24가지 코드 악취(bad smells)**를, 그 악취를 치유하는 리팩터링 기법에 연결한 표다. 코드에서 어떤 악취가 느껴질 때 "그럼 무슨 기법을 꺼내야 하나"를 찾는 색인으로 쓴다. 각 악취의 상세는 [ch03](ch03-bad-smells-in-code.md), 각 기법의 상세는 [부록 A](appendix-a-list-of-refactorings.md)에서 가리키는 장을 참고한다. (절 번호는 한국어판 기준, 악취는 3.1~3.24 순서.)

---

- **기이한 이름 (Mysterious Name, 3.1)**: 함수 선언 바꾸기(6.5), 변수 이름 바꾸기(6.7), 필드 이름 바꾸기(9.2).
- **중복 코드 (Duplicated Code, 3.2)**: 함수 추출하기(6.1), 문장 슬라이드하기(8.6), 메서드 올리기(12.1).
- **긴 함수 (Long Function, 3.3)**: 함수 추출하기(6.1), 임시 변수를 질의 함수로 바꾸기(7.4), 매개변수 객체 만들기(6.8), 객체 통째로 넘기기(11.4), 함수를 명령으로 바꾸기(11.9), 조건문 분해하기(10.1), 조건부 로직을 다형성으로 바꾸기(10.4), 반복문 쪼개기(8.7).
- **긴 매개변수 목록 (Long Parameter List, 3.4)**: 매개변수를 질의 함수로 바꾸기(11.5), 객체 통째로 넘기기(11.4), 매개변수 객체 만들기(6.8), 플래그 인수 제거하기(11.3), 여러 함수를 클래스로 묶기(6.9).
- **전역 데이터 (Global Data, 3.5)**: 변수 캡슐화하기(6.6).
- **가변 데이터 (Mutable Data, 3.6)**: 변수 캡슐화하기(6.6), 변수 쪼개기(9.1), 문장 슬라이드하기(8.6), 함수 추출하기(6.1), 질의 함수와 변경 함수 분리하기(11.1), 세터 제거하기(11.7), 파생 변수를 질의 함수로 바꾸기(9.3), 여러 함수를 클래스로 묶기(6.9), 여러 함수를 변환 함수로 묶기(6.10), 참조를 값으로 바꾸기(9.4).
- **뒤엉킨 변경 (Divergent Change, 3.7)**: 단계 쪼개기(6.11), 함수 옮기기(8.1), 함수 추출하기(6.1), 클래스 추출하기(7.5).
- **산탄총 수술 (Shotgun Surgery, 3.8)**: 함수 옮기기(8.1), 필드 옮기기(8.2), 여러 함수를 클래스로 묶기(6.9), 여러 함수를 변환 함수로 묶기(6.10), 단계 쪼개기(6.11), 함수 인라인하기(6.2), 클래스 인라인하기(7.6).
- **기능 편애 (Feature Envy, 3.9)**: 함수 옮기기(8.1), 함수 추출하기(6.1).
- **데이터 뭉치 (Data Clumps, 3.10)**: 클래스 추출하기(7.5), 매개변수 객체 만들기(6.8), 객체 통째로 넘기기(11.4).
- **기본형 집착 (Primitive Obsession, 3.11)**: 기본형을 객체로 바꾸기(7.3), 타입 코드를 서브클래스로 바꾸기(12.6), 조건부 로직을 다형성으로 바꾸기(10.4), 클래스 추출하기(7.5), 매개변수 객체 만들기(6.8).
- **반복되는 switch문 (Repeated Switches, 3.12)**: 조건부 로직을 다형성으로 바꾸기(10.4).
- **반복문 (Loops, 3.13)**: 반복문을 파이프라인으로 바꾸기(8.8).
- **성의 없는 요소 (Lazy Element, 3.14)**: 함수 인라인하기(6.2), 클래스 인라인하기(7.6), 계층 합치기(12.9).
- **추측성 일반화 (Speculative Generality, 3.15)**: 계층 합치기(12.9), 함수 인라인하기(6.2), 클래스 인라인하기(7.6), 함수 선언 바꾸기(6.5), 죽은 코드 제거하기(8.9).
- **임시 필드 (Temporary Field, 3.16)**: 클래스 추출하기(7.5), 함수 옮기기(8.1), 특이 케이스 추가하기(10.5).
- **메시지 체인 (Message Chains, 3.17)**: 위임 숨기기(7.7), 함수 추출하기(6.1), 함수 옮기기(8.1).
- **중개자 (Middle Man, 3.18)**: 중개자 제거하기(7.8), 함수 인라인하기(6.2), 서브클래스를 위임으로 바꾸기(12.10), 슈퍼클래스를 위임으로 바꾸기(12.11).
- **내부자 거래 (Insider Trading, 3.19)**: 함수 옮기기(8.1), 필드 옮기기(8.2), 위임 숨기기(7.7), 서브클래스를 위임으로 바꾸기(12.10), 슈퍼클래스를 위임으로 바꾸기(12.11).
- **거대한 클래스 (Large Class, 3.20)**: 클래스 추출하기(7.5), 슈퍼클래스 추출하기(12.8), 타입 코드를 서브클래스로 바꾸기(12.6).
- **서로 다른 인터페이스의 대안 클래스들 (Alternative Classes with Different Interfaces, 3.21)**: 함수 선언 바꾸기(6.5), 함수 옮기기(8.1), 슈퍼클래스 추출하기(12.8).
- **데이터 클래스 (Data Class, 3.22)**: 레코드 캡슐화하기(7.1), 세터 제거하기(11.7), 함수 옮기기(8.1), 함수 추출하기(6.1), 단계 쪼개기(6.11).
- **상속 포기 (Refused Bequest, 3.23)**: 메서드 내리기(12.4), 필드 내리기(12.5), 서브클래스를 위임으로 바꾸기(12.10), 슈퍼클래스를 위임으로 바꾸기(12.11).
- **주석 (Comments, 3.24)**: 함수 추출하기(6.1), 함수 선언 바꾸기(6.5), 어서션 추가하기(10.6).

## 가장 자주 등장하는 처방

여러 악취에 반복해서 처방되는 기법일수록 **기초 체력**에 가깝다. 다음 기법들은 여러 악취를 가로질러 등장하므로 먼저 몸에 익혀 두면 좋다.

- **함수 추출하기(6.1)**: 중복 코드·긴 함수·기능 편애·뒤엉킨 변경·메시지 체인·주석 등 — 가장 널리 쓰이는 기본기.
- **함수 옮기기(8.1)**: 기능 편애·산탄총 수술·내부자 거래·임시 필드·데이터 클래스 등 — 요소를 올바른 자리로.
- **클래스 추출하기(7.5)**: 데이터 뭉치·기본형 집착·임시 필드·거대한 클래스 등 — 책임을 갈라낸다.
- **매개변수 객체 만들기(6.8)**: 긴 매개변수 목록·데이터 뭉치·기본형 집착 — 함께 다니는 데이터를 묶는다.
- **서브/슈퍼클래스를 위임으로 바꾸기(12.10·12.11)**: 중개자·내부자 거래·상속 포기 — 잘못 끼워진 상속을 푼다.
