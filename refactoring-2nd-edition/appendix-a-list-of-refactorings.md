# 부록 A: 리팩터링 목록 (List of Refactorings)

이 책에 실린 **66가지 리팩터링 기법**의 전체 목록이다. 각 기법의 상세(스케치·배경·절차)는 해당 장에 있으니, 이 목록은 "무슨 기법이 있고 어디서 찾는지"를 한눈에 보기 위한 색인으로 쓴다. 절 번호는 한국어판 기준이다.

---

## 6장 — 기본적인 리팩터링 ([ch06](ch06-a-first-set-of-refactorings.md))

- **함수 추출하기 (Extract Function, 6.1)**: 코드 조각을 목적이 드러나는 이름의 함수로 뽑아낸다.
- **함수 인라인하기 (Inline Function, 6.2)**: 본문이 이름만큼 명확하면 호출부에 직접 넣고 함수를 없앤다.
- **변수 추출하기 (Extract Variable, 6.3)**: 복잡한 표현식에 이름을 붙여 지역 변수로 뽑는다.
- **변수 인라인하기 (Inline Variable, 6.4)**: 변수가 표현식보다 나을 게 없으면 표현식으로 되돌린다.
- **함수 선언 바꾸기 (Change Function Declaration, 6.5)**: 함수 이름·매개변수를 바꿔 인터페이스를 개선한다.
- **변수 캡슐화하기 (Encapsulate Variable, 6.6)**: 데이터 접근을 함수로 감싸 통제한다.
- **변수 이름 바꾸기 (Rename Variable, 6.7)**: 변수 이름을 더 잘 드러나게 바꾼다.
- **매개변수 객체 만들기 (Introduce Parameter Object, 6.8)**: 함께 다니는 데이터 뭉치를 객체로 묶는다.
- **여러 함수를 클래스로 묶기 (Combine Functions into Class, 6.9)**: 같은 데이터를 다루는 함수들을 클래스로 묶는다.
- **여러 함수를 변환 함수로 묶기 (Combine Functions into Transform, 6.10)**: 파생 데이터 계산을 변환 함수 한 곳에 모은다.
- **단계 쪼개기 (Split Phase, 6.11)**: 서로 다른 두 대상을 다루는 코드를 별개의 단계로 나눈다.

## 7장 — 캡슐화 ([ch07](ch07-encapsulation.md))

- **레코드 캡슐화하기 (Encapsulate Record, 7.1)**: 가변 레코드를 클래스로 바꿔 저장 방식을 숨긴다.
- **컬렉션 캡슐화하기 (Encapsulate Collection, 7.2)**: 게터는 복제본을 반환하고 변경은 전용 메서드로만 하게 한다.
- **기본형을 객체로 바꾸기 (Replace Primitive with Object, 7.3)**: 동작이 필요해진 기본형을 전용 값 클래스로 승격한다.
- **임시 변수를 질의 함수로 바꾸기 (Replace Temp with Query, 7.4)**: 계산 결과 임시 변수를 게터(질의 함수)로 바꾼다.
- **클래스 추출하기 (Extract Class, 7.5)**: 일이 많은 클래스에서 일부 데이터·메서드를 새 클래스로 분리한다.
- **클래스 인라인하기 (Inline Class, 7.6)**: 역할을 잃은 클래스를 가장 많이 쓰는 클래스로 흡수한다.
- **위임 숨기기 (Hide Delegate, 7.7)**: `서버.위임.메서드()`를 `서버.메서드()`로 감싸 위임 객체를 숨긴다.
- **중개자 제거하기 (Remove Middle Man, 7.8)**: 단순 위임이 많아지면 위임 객체를 직접 호출하게 한다.
- **알고리즘 교체하기 (Substitute Algorithm, 7.9)**: 복잡한 알고리즘을 더 간명한 것으로 통째 교체한다.

## 8장 — 기능 이동 ([ch08](ch08-moving-features.md))

- **함수 옮기기 (Move Function, 8.1)**: 다른 모듈 요소를 더 많이 참조하는 함수를 그 모듈로 옮긴다.
- **필드 옮기기 (Move Field, 8.2)**: 함께 갱신·전달되는 필드를 어울리는 레코드로 옮긴다.
- **문장을 함수로 옮기기 (Move Statements into Function, 8.3)**: 늘 함께 실행되는 문장을 피호출 함수 안으로 합친다.
- **문장을 호출한 곳으로 옮기기 (Move Statements to Callers, 8.4)**: 일부 호출자만 달라지는 부분을 호출한 곳으로 꺼낸다.
- **인라인 코드를 함수 호출로 바꾸기 (Replace Inline Code with Function Call, 8.5)**: 이미 있는 함수와 같은 일을 하는 인라인 코드를 호출로 대체한다.
- **문장 슬라이드하기 (Slide Statements, 8.6)**: 관련된 문장을 한데 모아 이해와 추출을 쉽게 한다.
- **반복문 쪼개기 (Split Loop, 8.7)**: 한 반복문이 하는 두 가지 일을 각각의 반복문으로 나눈다.
- **반복문을 파이프라인으로 바꾸기 (Replace Loop with Pipeline, 8.8)**: 반복문을 map·filter 등 컬렉션 파이프라인으로 바꾼다.
- **죽은 코드 제거하기 (Remove Dead Code, 8.9)**: 더 이상 쓰이지 않는 코드를 지운다.

## 9장 — 데이터 조직화 ([ch09](ch09-organizing-data.md))

- **변수 쪼개기 (Split Variable, 9.1)**: 역할이 둘 이상인 변수를 역할 하나당 하나로 나눈다.
- **필드 이름 바꾸기 (Rename Field, 9.2)**: 레코드·클래스의 필드(및 접근자) 이름을 더 잘 드러나게 바꾼다.
- **파생 변수를 질의 함수로 바꾸기 (Replace Derived Variable with Query, 9.3)**: 계산으로 구할 수 있는 값의 저장을 없앤다.
- **참조를 값으로 바꾸기 (Change Reference to Value, 9.4)**: 중첩 객체를 불변 값 객체로 만들어 통째로 교체하게 한다.
- **값을 참조로 바꾸기 (Change Value to Reference, 9.5)**: 같은 엔티티의 여러 복제본을 저장소를 통한 단일 참조로 바꾼다.
- **매직 리터럴 바꾸기 (Replace Magic Literal, 9.6)**: 의미를 알기 어려운 리터럴을 이름 있는 상수로 바꾼다.

## 10장 — 조건부 로직 간소화 ([ch10](ch10-simplifying-conditional-logic.md))

- **조건문 분해하기 (Decompose Conditional, 10.1)**: 조건식·조건절을 의도가 드러나는 함수로 추출한다.
- **조건식 통합하기 (Consolidate Conditional Expression, 10.2)**: 동작이 같은 여러 검사를 하나의 조건식으로 합친다.
- **중첩 조건문을 보호 구문으로 바꾸기 (Replace Nested Conditional with Guard Clauses, 10.3)**: 비정상 케이스를 먼저 빠져나가는 보호 구문으로 평탄화한다.
- **조건부 로직을 다형성으로 바꾸기 (Replace Conditional with Polymorphism, 10.4)**: 타입별 분기를 클래스와 다형성으로 대체한다.
- **특이 케이스 추가하기 (Introduce Special Case, 10.5)**: 특정 값(널)의 똑같은 처리를 특이 케이스 객체로 모은다.
- **어서션 추가하기 (Introduce Assertion, 10.6)**: 코드가 항상 참이라 가정하는 조건을 어서션으로 명시한다.
- **제어 플래그를 탈출문으로 바꾸기 (Replace Control Flag with Break, 10.7)**: 흐름 제어 플래그를 `break`·`continue`·`return`으로 없앤다.

## 11장 — API 리팩터링 ([ch11](ch11-refactoring-apis.md))

- **질의 함수와 변경 함수 분리하기 (Separate Query from Modifier, 11.1)**: 값 반환과 부수효과가 섞인 함수를 나눈다.
- **함수 매개변수화하기 (Parameterize Function, 11.2)**: 리터럴만 다른 함수들을 매개변수 하나로 합친다.
- **플래그 인수 제거하기 (Remove Flag Argument, 11.3)**: 로직을 고르는 리터럴 불리언 인수를 명시적 함수로 대체한다.
- **객체 통째로 넘기기 (Preserve Whole Object, 11.4)**: 값 몇 개 대신 레코드를 통째로 넘긴다.
- **매개변수를 질의 함수로 바꾸기 (Replace Parameter with Query, 11.5)**: 피호출 함수가 스스로 알아낼 수 있는 매개변수를 없앤다.
- **질의 함수를 매개변수로 바꾸기 (Replace Query with Parameter, 11.6)**: 함수 안의 거북한 참조를 매개변수로 바꿔 참조 투명성을 얻는다.
- **세터 제거하기 (Remove Setting Method, 11.7)**: 생성 후 변경되면 안 되는 필드의 세터를 없앤다.
- **생성자를 팩터리 함수로 바꾸기 (Replace Constructor with Factory Function, 11.8)**: 생성자의 제약을 팩터리 함수로 우회한다.
- **함수를 명령으로 바꾸기 (Replace Function with Command, 11.9)**: 함수를 그 함수만을 위한 명령 객체로 캡슐화한다.
- **명령을 함수로 바꾸기 (Replace Command with Function, 11.10)**: 로직이 단순한 명령 객체를 평범한 함수로 되돌린다.
- **수정된 값 반환하기 (Return Modified Value, 11.11)**: 갱신하는 함수가 수정된 값을 반환해 데이터 흐름을 드러낸다.
- **오류 코드를 예외로 바꾸기 (Replace Error Code with Exception, 11.12)**: 예상 밖 오류를 나타내는 반환 코드를 예외로 바꾼다.
- **예외를 사전확인으로 바꾸기 (Replace Exception with Precheck, 11.13)**: 호출 전 검사할 수 있는 조건이면 예외 대신 사전 확인한다.

## 12장 — 상속 다루기 ([ch12](ch12-dealing-with-inheritance.md))

- **메서드 올리기 (Pull Up Method, 12.1)**: 서브클래스의 똑같은 메서드를 슈퍼클래스로 올린다.
- **필드 올리기 (Pull Up Field, 12.2)**: 서브클래스의 중복 필드를 슈퍼클래스로 올린다.
- **생성자 본문 올리기 (Pull Up Constructor Body, 12.3)**: 서브클래스 생성자들의 공통 초기화 코드를 슈퍼클래스로 올린다.
- **메서드 내리기 (Push Down Method, 12.4)**: 특정 서브클래스만 쓰는 메서드를 그 서브클래스로 내린다.
- **필드 내리기 (Push Down Field, 12.5)**: 특정 서브클래스만 쓰는 필드를 그 서브클래스로 내린다.
- **타입 코드를 서브클래스로 바꾸기 (Replace Type Code with Subclasses, 12.6)**: 동작을 가르는 타입 코드를 타입별 서브클래스로 바꾼다.
- **서브클래스 제거하기 (Remove Subclass, 12.7)**: 가치를 잃은 서브클래스를 슈퍼클래스의 필드로 대체한다.
- **슈퍼클래스 추출하기 (Extract Superclass, 12.8)**: 비슷한 두 클래스의 공통 요소를 새 슈퍼클래스로 뽑는다.
- **계층 합치기 (Collapse Hierarchy, 12.9)**: 너무 비슷해진 클래스와 부모를 하나로 합친다.
- **서브클래스를 위임으로 바꾸기 (Replace Subclass with Delegate, 12.10)**: 상속으로 표현하던 변종을 위임 객체로 바꾼다.
- **슈퍼클래스를 위임으로 바꾸기 (Replace Superclass with Delegate, 12.11)**: 서브클래스에 맞지 않는 슈퍼클래스를 상속 대신 위임으로 이용한다.

## 참고 — 이름·반대 관계 한눈에 보기

여러 기법은 **반대 방향의 짝**을 이룬다. 한쪽을 알면 다른 쪽도 자연스럽게 이해된다.

- **추출 ↔ 인라인**: 함수 추출(6.1) ↔ 함수 인라인(6.2), 변수 추출(6.3) ↔ 변수 인라인(6.4), 클래스 추출(7.5) ↔ 클래스 인라인(7.6).
- **위임**: 위임 숨기기(7.7) ↔ 중개자 제거(7.8).
- **함수/문장 이동**: 문장을 함수로(8.3) ↔ 호출한 곳으로(8.4).
- **값/참조**: 참조를 값으로(9.4) ↔ 값을 참조로(9.5).
- **매개변수/질의**: 매개변수를 질의로(11.5) ↔ 질의를 매개변수로(11.6).
- **함수/명령**: 함수를 명령으로(11.9) ↔ 명령을 함수로(11.10).
- **오류 처리**: 오류 코드를 예외로(11.12) ↔ 예외를 사전확인으로(11.13).
- **상속 위아래**: 메서드/필드 올리기(12.1·12.2) ↔ 메서드/필드 내리기(12.4·12.5).
- **타입/서브클래스**: 타입 코드를 서브클래스로(12.6) ↔ 서브클래스 제거(12.7).
