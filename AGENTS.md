# Project: 기술 서적 학습 노트

## Codex Init

이 파일은 Codex용 프로젝트 규칙이다. 기존 `CLAUDE.md`와 같은 의도를 공유하며, 프로젝트 전역 규칙을 바꿀 때는 두 파일을 함께 갱신한다.

Claude 전용 설정은 `.claude/` 아래에 있다. Codex는 `.claude/agents/*.md`를 자동 sub-agent로 사용한다고 가정하지 않는다. 필요한 규칙은 이 파일을 기준으로 직접 따른다.

## 프로젝트 개요

소프트웨어 엔지니어링 관련 기술 서적을 책 없이도 학습 가능한 수준의 한국어 마크다운 노트로 정리하는 프로젝트다.

이 저장소는 주로 문서 저장소다. 별도 요청이 없으면 빌드, 테스트, 패키지 설치 같은 코드 프로젝트 명령을 실행하지 않는다.

## 공통 작성 스타일

- 언어는 한국어를 사용한다.
- 문체는 서술형 `-다` 체를 기본으로 한다.
- 각 책은 챕터별 독립 마크다운 파일과 `README.md` 목차/개요를 둔다.
- 챕터 파일명은 기본적으로 `ch{번호:두자리}-{영문-kebab-case}.md` 형식을 따른다.
- 각 챕터는 책을 소유하지 않은 사람도 핵심 개념을 학습할 수 있을 정도로 상세하게 작성한다.
- 원서의 내용을 기반으로 하되, 이해를 돕기 위한 추가 코드 예시와 설명을 포함한다.
- 새 책 작업 시에는 먼저 1~2개 챕터 샘플을 만들어 사용자 확인 후 본격 진행한다.

## 챕터 파일 기본 구성

일반 챕터는 아래 구조를 기본으로 삼되, 책의 성격에 따라 필요한 섹션만 조정한다.

- `# Chapter N: 영문 제목 (한국어 부제)`
- `## 핵심 질문`
- 번호가 매겨진 주요 섹션: `## 1.`, `## 2.`, ...
- 코드 예시: 나쁜 예와 좋은 예를 대비한다.
- `> **핵심 통찰**:` blockquote로 핵심 인사이트를 강조한다.
- 비교 테이블을 적절히 사용한다.
- `## 요약`
- `## 다른 챕터와의 관계`: 챕터 간 연결이 약한 책은 생략 가능하다.

## 용어 설명 스타일

부가 설명이 필요한 용어는 처음 등장하는 곳에서 한 번만 인라인 이탤릭으로 설명한다.

형식:

```md
용어(*Term - 설명*)
```

예:

```md
테스트 스위트(*Test Suite - 프로젝트에 속한 모든 테스트의 집합*)
```

이후에는 한국어 용어만 사용한다.

## origin.md 파일 분리 작업

PDF에서 추출한 `origin.md` 원본 파일은 내용 수정 없이 라인 기반으로 분리한다.

### 파일 네이밍 규칙

| 파일명 | 내용 |
|--------|------|
| `0.origin.md` | 서문, 대상 독자, 책 구성, 집필 동기, 감사의 글, 목차 등 앞부분 |
| `1.origin.md` ~ `N.origin.md` | 챕터 본문. 챕터 번호와 파일 번호를 맞춘다. |
| `N+1.origin.md` ~ | 내용이 있는 부록 |
| `99.origin.md` | 색인/찾아보기 등 학습 노트 작성에 쓸모가 낮은 뒷부분 |

### 작업 절차

1. `rg`로 챕터 제목 패턴을 검색해 목차와 본문 시작 라인을 구분한다.
2. 각 챕터의 시작/끝 라인을 확인한다.
3. 라인 범위 기준으로 분리한다.
4. 분리된 파일을 순서대로 이어붙였을 때 `origin.md`와 같은지 검증한다.

검증은 단순 줄 수 합계보다 바이트 단위 일치 확인을 우선한다.

## Notion 업로드 규칙

Claude 프로젝트에는 `.claude/agents/notion-uploader.md`가 있지만, Codex에서는 자동 sub-agent로 동작한다고 가정하지 않는다. Codex에서 Notion 업로드를 요청받으면 아래 규칙을 직접 따른다.

- 원본 내용을 요약, 압축, 생략하지 않는다.
- 로컬 마크다운 파일의 원본 내용을 Notion에 그대로 업로드한다.
- 파일이 커서 한 번에 올릴 수 없으면 사용 가능한 도구 범위에서 나눠 올리되, 내용을 누락하지 않는다.
- 업로드 전 `<sub>` 태그 각주는 Notion에서 렌더링되지 않으므로 `용어¹`를 `용어(*Term - 설명*)` 형태로 인라인 변환하고, `<sub>...</sub><br>` 각주 블록은 제거한다.
- 마크다운 테이블을 Notion HTML table 형태로 변환해야 할 때는 `<th>`를 쓰지 않고 `<td>`만 사용하며, `<table header-row="true">`를 지정한다.
- 현재 Codex 세션에 Notion MCP 도구가 없으면 업로드는 불가능하다고 명확히 보고하고, 로컬 변환 파일까지만 준비한다.

## 작업 시 유의사항

- 기존 파일의 사용자 변경을 되돌리지 않는다.
- OCR 원문 파일을 분리할 때는 내용을 교정하거나 정리하지 않는다.
- 학습 노트를 작성할 때는 단순 요약이 아니라 개념 이해, 예시, 비교, 실전 적용까지 포함한다.
- 사용자가 특정 책/챕터 스타일을 지정하면 기존 책별 스타일보다 사용자 지시를 우선한다.
